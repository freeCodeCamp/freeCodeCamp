## Description
<section id="description"> <strong>Objetivo:</strong> crear una aplicación <a href="https://codepen.io" target="_blank">CodePen.io</a> que sea funcionalmente similar a esta: <a href="https://codepen.io/freeCodeCamp/full/bgpXyK" target="_blank">https://codepen.io/freeCodeCamp/full/bgpXyK</a> . Cumple las siguientes <a href="https://en.wikipedia.org/wiki/User_story" target="_blank">historias de usuario</a> y haz que pasen todas las pruebas. Dale tu propio estilo personal. Puede utilizar HTML, JavaScript, CSS y la biblioteca de visualización basada en svg D3. Las pruebas requieren que se generen ejes utilizando la propiedad del eje D3, que genera automáticamente tics a lo largo del eje. Estas marcas son necesarias para pasar las pruebas D3 porque sus posiciones se usan para determinar la alineación de los elementos graficados. Encontrará información sobre la generación de ejes en <a href="https://github.com/d3/d3/blob/master/API.md#axes-d3-axis" target="_blank">https://github.com/d3/d3/blob/master/API.md#axes-d3-axis</a> . Los elementos DOM (no virtuales) requeridos se consultan en el momento de cada prueba. Si usa un marco de frontend (como Vue, por ejemplo), los resultados de la prueba pueden ser inexactos para el contenido dinámico. Esperamos acomodarlos eventualmente, pero estos marcos no están actualmente soportados para proyectos D3. <strong>Historia de usuario n. ° 1:</strong> Puedo ver un elemento de título que tiene un <code>id=&quot;title&quot;</code> . <strong>Historia de usuario n. ° 2:</strong> Puedo ver un eje x que tiene una <code>id=&quot;x-axis&quot;</code> correspondiente <code>id=&quot;x-axis&quot;</code> . <strong>Historia de usuario n. ° 3:</strong> Puedo ver un eje y que tiene una <code>id=&quot;y-axis&quot;</code> correspondiente <code>id=&quot;y-axis&quot;</code> . <strong>Historia de usuario n. ° 4:</strong> Puedo ver puntos, cada uno con una clase de <code>dot</code> , que representan los datos que se trazan. <strong>Historia de usuario n. ° 5:</strong> Cada punto debe tener las propiedades <code>data-xvalue</code> y <code>data-yvalue</code> contienen sus valores x e y correspondientes. <strong>Historia de usuario n. ° 6:</strong> El valor de <code>data-xvalue</code> y el valor de <code>data-yvalue</code> de cada punto deben estar dentro del rango de los datos reales y en el formato de datos correcto. Para <code>data-xvalue</code> , los enteros (años completos) o los objetos Date son aceptables para la evaluación de la prueba. Para <code>data-yvalue</code> (minutos), use objetos Date. <strong>Historia de usuario n. ° 7:</strong> el valor <code>data-xvalue</code> y su punto correspondiente deben alinearse con el punto / valor correspondiente en el eje x. <strong>Historia de usuario n. ° 8:</strong> el <code>data-yvalue</code> y del <code>data-yvalue</code> y su punto correspondiente deben alinearse con el punto / valor correspondiente en el eje y. <strong>Historia de usuario n. ° 9:</strong> Puedo ver varias etiquetas de tic en el eje y con el formato de hora <code>%M:%S</code> <strong>Historia de usuario n. ° 10:</strong> Puedo ver varias etiquetas de tick en el eje x que muestran el año. <strong>Historia de usuario n. ° 11:</strong> Puedo ver que el rango de las etiquetas del eje x está dentro del rango de los datos reales del eje x. <strong>Historia de usuario n. ° 12:</strong> Puedo ver que el rango de las etiquetas del eje y está dentro del rango de los datos reales del eje y. <strong>Historia de usuario n. ° 13:</strong> Puedo ver una leyenda que contiene texto descriptivo que tiene <code>id=&quot;legend&quot;</code> . <strong>Historia de usuario n. ° 14:</strong> puedo pasar el mouse sobre un área y ver una información sobre herramientas con la correspondiente <code>id=&quot;tooltip&quot;</code> que muestra más información sobre el área. <strong>Historia de usuario n. ° 15:</strong> Mi información sobre herramientas debe tener una propiedad de <code>data-year</code> que se corresponda con el valor de <code>data-xvalue</code> del área activa.
