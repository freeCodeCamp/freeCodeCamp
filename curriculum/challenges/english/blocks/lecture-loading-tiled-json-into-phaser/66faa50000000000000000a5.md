# --interactive--

Once `preload` has both the JSON and the image, three calls in `create` turn them into game objects you can render and collide against.

```js
create() {
  const map = this.make.tilemap({ key: "room1" });
  const tileset = map.addTilesetImage("dungeon", "tiles");
  const floor = map.createLayer("floor", tileset, 0, 0);
  const walls = map.createLayer("walls", tileset, 0, 0);
}
```

Step by step:

- `make.tilemap({ key })` reads the parsed JSON out of the cache and returns a `Tilemap` object. Nothing renders yet — this is just the data structure.
- `addTilesetImage(<tilesetName>, <imageKey>)` binds Tiled's tileset (named in the JSON's `tilesets[]`) to the image you loaded. The first argument is the **`name` field of the tileset inside the JSON**, *not* the file name. The second argument is the cache key from `load.image`.
- `createLayer(<layerName>, tileset, x, y)` is the call that actually creates a `TilemapLayer` game object on screen. It pulls the named layer (`"floor"`, `"walls"`) out of the JSON, walks the data array, and draws the tiles. The `x, y` is where the top-left of the layer goes — almost always `(0, 0)`.

:::interactive_editor

```js
function create() {
  const map = this.make.tilemap({ key: "room1" });
  const tileset = map.addTilesetImage("dungeon", "tiles");
  const floor = map.createLayer("floor", tileset, 0, 0);
  const walls = map.createLayer("walls", tileset, 0, 0);

  console.log("map size:", map.width, "x", map.height, "tiles");
  console.log("floor layer:", floor && floor.layer.name);
  console.log("walls layer:", walls && walls.layer.name);
}
```

:::

The `tileset` variable returned by `addTilesetImage` is the same object you pass to every `createLayer` call. If your map references multiple tilesets — say, a separate `"props"` tileset — you call `addTilesetImage` once per tileset and pass an *array* of tilesets to `createLayer`. Tiny map, single tileset is the usual case and the one this chapter sticks to.

A common pitfall: the first argument to `addTilesetImage` mismatches the JSON. If your tileset JSON has `"name": "dungeon"` but you write `addTilesetImage("tileset-dungeon", "tiles")`, you will silently get back `null` and every layer renders blank. Open the JSON, find `tilesets[0].name`, and copy it exactly.
