# --interactive--

Stereo pan tells the mixer how loud a sound is in the left ear vs the right ear. `setPan` takes one number from `-1` (full left) to `+1` (full right), with `0` being centered.

```js
const enemyShout = this.sound.add("sfx-enemy-shout");
enemyShout.setPan(-0.6);
enemyShout.play();
```

`-0.6` means roughly 80% left, 20% right. Headphone players hear the enemy off to their left. With speakers the effect is weaker but still present.

The most common production use is **directional audio**: an enemy off-screen to the player's left should sound from the left. The math is just normalized horizontal distance.

```js
function panFor(soundX, listenerX, halfWidth) {
  return Phaser.Math.Clamp((soundX - listenerX) / halfWidth, -1, 1);
}

const pan = panFor(enemyX, this.cameras.main.scrollX + this.cameras.main.width / 2, 320);
shout.setPan(pan);
```

`halfWidth` is how far in either direction is "full pan". A reasonable value is half the camera width, so an enemy at the very left edge of the screen pans to `-1` and an enemy in the center pans to `0`.

:::interactive_editor

```js
function panFor(soundX, listenerX, halfWidth) {
  const raw = (soundX - listenerX) / halfWidth;
  return Math.max(-1, Math.min(1, raw));
}

console.log("dead center:", panFor(400, 400, 320));
console.log("offscreen-left:", panFor(0, 400, 320));
console.log("right edge:", panFor(720, 400, 320));
console.log("far past right:", panFor(2000, 400, 320));
```

:::

A few notes on what `setPan` does and doesn't do.

**It's stereo, not surround.** Phaser exposes a `StereoPanner` node — left vs right only. There's no "front vs back". For 3D positional audio you'd want the Web Audio `PannerNode` or one of the Phaser-on-top-of-WebAudio plugins; that's out of scope here.

**It's instance-scoped.** Calling `this.sound.setPan` on the manager doesn't exist. Every sound's pan is on the `Sound` instance. If you have a 2D engine sound that should pan with the player's position, hold the instance and update its `pan` every frame.

**It can be tweened.** Pan is a normal numeric property, so:

```js
this.tweens.add({
  targets: thunder,
  pan: { from: -1, to: 1 },
  duration: 2000
});
```

…sweeps thunder from far-left to far-right over 2 seconds, perfect for a "boss enters" sting.

**Mono sources only have horizontal information.** If you imported a stereo `OGG` and call `setPan(-0.5)`, the existing stereo image is squashed into the new pan position. For directional SFX, ship mono masters and let the engine pan them. For ambience and music, ship stereo and don't pan.

A subtle bug: `setPan(-1)` and immediately `setPan(0)` on the same frame doesn't tween — it snaps. If the listener moves quickly, the audible jump can feel jarring. Either tween the pan with a short duration (50–100 ms) or low-pass the position update with `pan += (target - pan) * 0.2`.
