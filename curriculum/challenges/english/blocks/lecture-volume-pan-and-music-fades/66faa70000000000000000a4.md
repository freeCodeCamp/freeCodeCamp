# --interactive--

Three knobs control how a `Sound` instance feels: **volume**, **detune**, and **rate**. They're independent, set on the `Sound` reference, and can be changed any time — including mid-playback.

```js
const jump = this.sound.add("sfx-jump");
jump.setVolume(0.5);   // 0 = silent, 1 = full, no clamp above 1
jump.setDetune(200);   // cents; 100 cents = 1 semitone, 1200 = 1 octave up
jump.setRate(1.5);     // 1 = normal, 2 = double speed, 0.5 = half speed
jump.play();
```

`setVolume(level)` is the loudness multiplier. The mixer rolls everything together: the instance's volume, the global `this.sound.volume`, and the bus volume if you've set one. So the *effective* loudness is `instance × global × bus`. Set `instance` for clip-specific tweaks (the boss roar should be louder); set `global` for "mute" toggles; introduce buses (Music vs SFX) yourself if you want sliders for both.

`setDetune(cents)` shifts pitch without changing playback speed. 100 cents = 1 semitone. To play a sound a semitone higher, `setDetune(100)`. An octave higher, `setDetune(1200)`. Phaser snaps to the nearest cent, but the perceptual change is smooth. Detune is the cleanest way to add variety to a repeated SFX — every time a coin pickup fires, randomize detune by `±50` cents and the player no longer hears the same sound on repeat.

`setRate(multiplier)` shifts pitch *and* speed together. `2.0` plays a sound twice as fast and an octave higher. `0.5` plays it twice as long and an octave lower. Use rate for stylized effects (slow-mo cassette tape on death, sped-up coin sound on a combo); use detune when you want a pitch tweak with no duration change.

A real trick: combining `setRate(0.7)` with `setDetune(700)` cancels the pitch shift of the rate change, leaving you with **time-stretching at the original pitch**. Phaser doesn't have proper time-stretch, but for short SFX the trick is good enough.

:::interactive_editor

```js
function variety(base, options = {}) {
  const detuneRange = options.detuneRange ?? 50;
  const rateRange = options.rateRange ?? 0;
  const detune = (Math.random() * 2 - 1) * detuneRange;
  const rate = 1 + (Math.random() * 2 - 1) * rateRange;
  return { ...base, detune, rate };
}

console.log(variety({ key: "sfx-coin", volume: 0.6 }));
console.log(variety({ key: "sfx-jump", volume: 0.5 }, { detuneRange: 200 }));
console.log(variety({ key: "sfx-hit", volume: 0.7 }, { detuneRange: 100, rateRange: 0.1 }));
```

:::

The pattern above is the **3-line cure for repeated SFX boredom**. You have one `sfx-coin.ogg`. You play it 60 times in a level. With raw playback the player notices it's the same clip and tunes it out. With `±50` cents of randomized detune, every coin sounds slightly different and the brain stays engaged.

Volume gotcha: setting `instance.setVolume(2.0)` does *not* clip; it just outputs above unity gain. On Web Audio that means the buffer is amplified, which can produce harsh distortion. If you need "louder than the source", remaster the source — don't push past 1.0 at the instance level.

Pitch gotcha: extreme detune (`±2000` cents) starts to sound like aliasing because Web Audio's pitch shifter is naive. For sound design, stay within ±300 cents.

# --questions--

## --text--

You want to randomize the pitch of a coin SFX so 60 plays don't sound identical, but the duration must stay constant. Which API do you use?

## --answers--

`setDetune(±50)` — detune shifts pitch with no duration change.

---

`setRate(0.95–1.05)` — rate gives a similar effect.

### --feedback--

`setRate` shifts pitch *and* duration. The pickups would each take a different amount of time.

---

`setVolume(0.95–1.05)` — small loudness changes mask repetition.

### --feedback--

That changes loudness, not timbre. Repetition still bores the player.

---

Reload the audio file with a new key and play that.

### --feedback--

You'd be reloading the same buffer for no reason. Detune is the right tool.

## --text--

You want a slow-motion death effect: the same hit SFX should play half as fast and an octave lower. Which call?

## --answers--

`setRate(0.5)` — rate halves both speed and pitch (one octave is a rate of 2 or 0.5).

---

`setDetune(-1200)` alone.

### --feedback--

That drops the pitch but keeps the duration. The clip would still play at full speed.

---

`setVolume(0.5)`.

### --feedback--

Volume is loudness. Speed and pitch don't change.

---

`setRate(0.5)` followed by `setDetune(-1200)`.

### --feedback--

Adding `-1200` detune on top of `setRate(0.5)` *cancels* the pitch drop, giving you slow audio at the original pitch — not the goal here.

## --text--

You set `instance.setVolume(2.0)` to make the boss roar "louder than full." The result sounds harsh and clippy. Why?

## --answers--

Volumes above `1.0` apply unity-plus gain on Web Audio, amplifying the buffer beyond the source's headroom and producing distortion. The fix is to remaster the source louder, then play at `<= 1.0`.

---

Phaser silently caps volume at `1.0`.

### --feedback--

It does not cap. The signal really is amplified, and the distortion is real.

---

`setVolume` doesn't accept floats above `1`.

### --feedback--

It accepts the value; the consequence is the distortion you observed.

---

Web Audio is set to "low" output mode by default.

### --feedback--

There is no such mode. Web Audio always uses linear gain.
