# --interactive--

Phaser exposes the sprite-sheet pipeline through one method on the loader: `this.load.spritesheet`. You call it inside a scene's `preload`, where Phaser collects every load instruction before any of them actually run.

Here is the canonical signature:

```js
this.load.spritesheet(key, url, frameConfig);
```

- **`key`** is the string you'll use everywhere else to refer to the texture — `"hero"`, `"coin"`, whatever. Pick something short and unambiguous.
- **`url`** is the path to the PNG. In a Phaser game served from `/learn/...`, an absolute path like `/curriculum-assets/phaser/spritesheets/hero-walk.png` resolves against the site origin and just works.
- **`frameConfig`** describes how to slice the sheet. The two required keys are `frameWidth` and `frameHeight`.

A complete preload for a 64×64 hero sheet split into 16 frames of 16×16 looks like this:

```js
preload() {
  this.load.spritesheet(
    "hero",
    "/curriculum-assets/phaser/spritesheets/hero-walk.png",
    { frameWidth: 16, frameHeight: 16 }
  );
}
```

Phaser computes the frame count automatically by dividing the sheet's pixel dimensions by `frameWidth × frameHeight`. For a 64×64 sheet at 16×16 frames, that's `(64/16) * (64/16) = 16` frames, indexed `0..15`.

You can override the auto-detection with two optional keys when you want to use only a subset of the sheet:

```js
this.load.spritesheet("hero", "...", {
  frameWidth: 16,
  frameHeight: 16,
  startFrame: 0,
  endFrame: 7
});
```

`startFrame` defaults to `0` and `endFrame` defaults to "all of them". You'll reach for these when one PNG holds animations for several characters and you want each character to use only its own frames.

Two non-obvious gotchas:

1. **Frame size mismatches.** If you tell Phaser `frameWidth: 16` but the sheet's frames are actually 24 pixels wide, the slicer happily produces garbage. Each frame ends up showing a sliver of the next one. Always check the artist's frame size before pasting it in.
2. **Padding and margins.** Some art tools export sheets with 1-pixel margins or gutters between frames. `this.load.spritesheet` accepts `margin` and `spacing` keys for that case. The sheets in this certification have neither, so you can ignore them — but they exist when you need them.

Once the sheet is loaded, the texture is in Phaser's cache under your key. The next lecture covers what to do with it: turning a numbered range of frames into a runnable animation.

# --questions--

## --text--

Which arguments are required when calling `this.load.spritesheet`?

## --answers--

A texture key, a URL, and a `frameConfig` with `frameWidth` and `frameHeight`.

---

Just a URL.

### --feedback--

Phaser needs a key to identify the texture afterward, plus the frame size to slice it.

---

A key, a URL, and a frame count.

### --feedback--

Phaser computes the frame count from the sheet's pixel dimensions; you pass the frame size, not the count.

---

A key and a base64 image string.

### --feedback--

`load.spritesheet` accepts a URL; embedded base64 is a separate, less common path.

## --text--

When would you pass `startFrame` and `endFrame` to `this.load.spritesheet`?

## --answers--

When the sheet contains frames you don't want — for example, several characters share one PNG and each should only see its own range.

---

Always — they are required arguments.

### --feedback--

Both are optional with sensible defaults.

---

When the frames are not the same size.

### --feedback--

A sprite sheet's frames must all be the same size; varying sizes is what atlases are for.

---

When loading audio sprites instead of image sprites.

### --feedback--

Audio uses a different loader (`this.load.audio`).
