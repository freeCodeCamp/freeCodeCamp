# --interactive--

Loading a sprite sheet hands Phaser the raw frames. To turn those frames into a playing animation you register one with the **animation manager**, available on every scene as `this.anims`.

The minimum animation definition looks like this:

```js
this.anims.create({
  key: "walk-down",
  frames: this.anims.generateFrameNumbers("hero", { start: 0, end: 3 }),
  frameRate: 8,
  repeat: -1
});
```

Four properties matter:

- **`key`** is the global name for this animation. You'll use it later when you tell a sprite to `anims.play("walk-down")`. Names live in a single shared namespace, so prefix them when collisions are likely (`hero-walk-down` vs just `walk-down`).
- **`frames`** is an array of frame descriptors. The helper `this.anims.generateFrameNumbers(textureKey, { start, end })` builds that array for you from a numeric range; the texture key matches the one you registered in `preload`.
- **`frameRate`** is frames per second. 8 FPS is a good default for a 4-frame walk cycle on a 16×16 character. 12–15 FPS reads as faster motion. Below 6 FPS the animation looks stilted.
- **`repeat`** controls how many times to play after the first pass. `0` plays once and stops. `-1` loops forever. `2` plays the original pass plus 2 more, so 3 total.

Two bonus properties earn their place often:

- **`yoyo: true`** plays the frames forward, then reverses. A 4-frame coin-spin played as a yoyo shows 8 frames per cycle without authoring an explicit reverse sequence.
- **`hideOnComplete: true`** hides the sprite after the animation ends. Useful for one-shot effects like an explosion or a pickup sparkle.

You define each animation **once per scene**, typically in `create`. After that, any sprite of the matching texture can play it. A scene with one hero and ten coins still defines `coin-spin` only once; the same animation key drives every coin.

If you mistype a frame range — say, `start: 0, end: 99` on a 16-frame sheet — Phaser silently includes the missing frames as `undefined`. The sprite blinks empty for those frames. Always cross-check `start` and `end` against your sheet layout.

A subtlety with `generateFrameNumbers`: the range is inclusive on both ends. `{ start: 0, end: 3 }` produces frames `[0, 1, 2, 3]` — four frames, not three.

# --questions--

## --text--

What does `repeat: -1` do in an animation config?

## --answers--

Loops the animation forever — every cycle starts again at the first frame after the last completes.

---

Plays the animation in reverse.

### --feedback--

That's `yoyo`, not `repeat: -1`.

---

Disables the animation.

### --feedback--

A negative repeat value isn't a kill switch; it's the "infinite" sentinel.

---

Plays the animation once and prints a warning.

### --feedback--

Phaser doesn't warn on `repeat: -1`; it loops.
