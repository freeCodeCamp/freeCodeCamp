# --interactive--

A body can be wired into as many colliders and overlaps as you like. Each `physics.add.collider` and `physics.add.overlap` call registers an independent rule, and the engine evaluates all of them every frame. There is no "one collider per body" cap.

This matters because real games stack rules. A platformer player typically participates in:

- a collider against the ground tilemap (so they land);
- an overlap against pickups (so coins fire a callback);
- an overlap against enemies (so contact damages them);
- a collider against moving platforms (so they ride along);
- an overlap against trigger zones (so cutscenes fire on entry).

Five rules, one body. Each rule has its own callbacks and process callback; they do not interfere with each other.

:::interactive_editor

```js
function create() {
  this.ball = this.add.circle(120, 30, 8, 0xffaa44);
  this.physics.add.existing(this.ball);
  this.ball.body.setBounce(0.7);
  this.ball.body.setCollideWorldBounds(true);

  const ground = this.add.rectangle(120, 170, 240, 12, 0x66ddff);
  this.physics.add.existing(ground, true);
  this.physics.add.collider(this.ball, ground);

  const wind = this.add.rectangle(60, 100, 40, 80, 0x88ccff, 0.2);
  this.physics.add.existing(wind, true);
  this.physics.add.overlap(this.ball, wind, (b) => b.body.setVelocityX(80));
}
```

:::

The ball collides with the ground (solid), overlaps with the wind zone (callback only). One body, two rules, no conflict.

The order matters when callbacks mutate shared state. Phaser fires colliders in **registration order** every frame. If two overlaps both try to set the ball's velocity, the second one wins. Register the rule with the higher priority *last*.

The flip side: if you want a body to ignore everything except a single specific other body, do not invent an exclusion list — use `physics.add.collider(specificA, specificB)` and skip the broad-group collider entirely. Selectivity comes from *which pairs you register*, not from filter callbacks.

The mental model: every `physics.add.collider`/`overlap` is one row in a rules table the engine evaluates each frame. Add and remove rows freely. The body itself stays simple.

# --questions--

## --text--

How many colliders or overlaps can a single body participate in at the same time?

## --answers--

As many as you register — there is no cap.

---

One per body — you must remove the previous one to add a new one.

### --feedback--

There is no per-body cap. Each `physics.add.collider`/`overlap` is independent.

---

Up to four — Phaser caches a fixed-size list internally.

### --feedback--

There is no built-in numeric cap.

---

Two: one collider and one overlap.

### --feedback--

You can have any number of each.

## --text--

Two overlap callbacks both call `ball.body.setVelocityX(...)` on the same frame. Which one wins?

## --answers--

The one registered later — Phaser fires callbacks in registration order, and the last write wins.

---

The one registered earlier — earlier registrations have priority.

### --feedback--

It is the opposite: later registrations write last, and the last write wins.

---

Phaser averages the two values.

### --feedback--

Phaser does not blend competing writes; it just fires both callbacks in order.

---

Phaser throws because two callbacks may not write the same property.

### --feedback--

Multiple callbacks freely mutate the same body; conflicts are the developer's responsibility.

## --text--

You want the player to collide with the world tilemap but ignore one specific NPC's body. What is the cleanest way?

## --answers--

Register the player–tilemap collider; do not register a player–NPC collider at all.

---

Register a player–NPC collider with a process callback that always returns `false`.

### --feedback--

That works but pays per-frame cost for a contact you could just skip by not registering.

---

Move the NPC into a different scene.

### --feedback--

Heavy-handed. Selectivity belongs at the rule registration level.

---

Set `npc.body.enable = false`.

### --feedback--

That disables the NPC's physics entirely, including its own colliders.
