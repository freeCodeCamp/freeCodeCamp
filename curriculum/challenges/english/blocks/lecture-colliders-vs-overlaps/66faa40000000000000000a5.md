# --interactive--

Both `collider` and `overlap` accept a fourth argument: the **process callback**. It runs *before* the collide/overlap callback and decides whether the contact should count at all. Return `true` to allow it, `false` to skip it for this frame.

```js
this.physics.add.collider(
  ball,
  platform,
  collideCallback,
  processCallback,
  this
);
```

The classic use case is **one-way platforms**. The player should pass through them when jumping up but land on them when falling. The process callback inspects velocities and only allows the collision when the player is moving down:

:::interactive_editor

```js
function create() {
  this.player = this.add.rectangle(60, 30, 16, 16, 0xffffff);
  this.physics.add.existing(this.player);

  const platform = this.add.rectangle(120, 90, 80, 6, 0x66ddff);
  this.physics.add.existing(platform, true);

  this.physics.add.collider(
    this.player,
    platform,
    null,
    (player, plat) =>
      player.body.velocity.y > 0 && player.body.bottom <= plat.body.top + 4
  );
}
```

:::

Trampolines are the same idea in reverse. An overlap with a process callback fires the boost only when the player is moving down — once they are launched up, the same overlap stops triggering until they fall again.

```js
this.physics.add.overlap(
  this.ball,
  this.trampoline,
  (ball) => ball.body.setVelocityY(-600),
  (ball) => ball.body.velocity.y > 0
);
```

The process callback also lets you write **per-frame skip logic** without touching state. "Ignore collisions while the player is invulnerable" → return `false` while a flag is set. "Skip collisions for the first 200 ms after spawn" → return `false` while the body's `body.embedded` flag is true. The collide/overlap callbacks would have to do this with extra branches; the process callback keeps the rule in one place.

Trap to avoid: the process callback runs on **every frame** the bodies are in contact, so keep it cheap. No allocations, no async, no traversing the scene graph. A handful of comparisons and a boolean is the right shape.
