# --interactive--

The reverse process is where the learning happens. A neural network is trained to undo each forward diffusion step — to predict what a slightly less noisy image looks like given the current noisy one. This learned reverse process is what enables generation.

### What the Network Predicts

There are two equivalent ways to frame what the network predicts:

1. **Predict the noise** (`ε-prediction`) — given the noisy image `x_t` and the timestep `t`, predict the Gaussian noise `ε` that was added to get from `x_0` to `x_t`. This is the most common formulation, used in DDPM and Stable Diffusion.

2. **Predict the clean image** (`x_0`-prediction) — given `x_t` and `t`, directly predict what the original clean image `x_0` looked like.

Both are mathematically equivalent (you can derive one from the other using the closed-form formula), but noise prediction tends to be more numerically stable in practice.

### The Training Objective

The training loop is surprisingly simple:

1. Sample a real image `x_0` from the training dataset
2. Sample a random timestep `t` uniformly from 1 to T
3. Sample Gaussian noise `ε`
4. Create the noisy image: `x_t = √ᾱ_t * x_0 + √(1-ᾱ_t) * ε`
5. Pass `x_t` and `t` to the neural network to predict the noise
6. Compute loss: mean squared error between predicted noise and actual noise
7. Update network weights

```python
import tensorflow as tf

def train_step(model, x0, alphas_cumprod, T, optimizer):
    batch_size = tf.shape(x0)[0]

    # Sample random timesteps — one per image in the batch
    t = tf.random.uniform(
        shape=(batch_size,), minval=0, maxval=T, dtype=tf.int32
    )

    # Sample Gaussian noise
    noise = tf.random.normal(shape=tf.shape(x0))

    # Create noisy images using the closed-form formula
    alpha_t    = tf.gather(alphas_cumprod, t)
    alpha_t    = tf.reshape(alpha_t, [-1, 1, 1, 1])  # broadcast over image dims
    x_noisy    = tf.sqrt(alpha_t) * x0 + tf.sqrt(1 - alpha_t) * noise

    with tf.GradientTape() as tape:
        # Predict the noise that was added
        noise_pred = model([x_noisy, t], training=True)
        # Loss: how well did we predict the actual noise?
        loss = tf.reduce_mean(tf.square(noise - noise_pred))

    grads = tape.gradient(loss, model.trainable_variables)
    optimizer.apply_gradients(zip(grads, model.trainable_variables))
    return loss
```

### The Reverse Sampling Step

During generation (inference), the model runs the reverse process. Starting from pure Gaussian noise `x_T`, the model repeatedly denoises:

```python
def reverse_step(model, x_t, t, betas, alphas_cumprod):
    """Perform one step of the reverse process (DDPM sampling)"""
    beta_t         = betas[t]
    alpha_t        = 1 - beta_t
    alpha_cumprod_t = alphas_cumprod[t]

    # Predict the noise
    t_tensor   = tf.constant([t])
    noise_pred = model([x_t, t_tensor], training=False)

    # Compute the predicted x_{t-1}
    coeff      = beta_t / tf.sqrt(1 - alpha_cumprod_t)
    mean       = (1 / tf.sqrt(alpha_t)) * (x_t - coeff * noise_pred)

    if t > 0:
        noise  = tf.random.normal(shape=tf.shape(x_t))
        x_prev = mean + tf.sqrt(beta_t) * noise
    else:
        x_prev = mean  # no noise added at the final step

    return x_prev

def generate(model, betas, alphas_cumprod, T, shape):
    x = tf.random.normal(shape)           # start from pure noise
    for t in reversed(range(T)):          # run T denoising steps
        x = reverse_step(model, x, t, betas, alphas_cumprod)
    return x
```

### Why Timestep Conditioning Matters

The network receives the current timestep `t` as an input alongside the noisy image. This is essential: the amount and character of noise present in `x_t` depends entirely on `t`. Without knowing `t`, the network cannot know how much noise to remove — a step-1 image (almost clean) requires a very different denoising operation than a step-900 image (almost pure noise).

### Key Points

- The neural network is trained to predict the noise `ε` added to an image, given the noisy image and the timestep
- The training loss is the mean squared error between the predicted noise and the actual noise added
- During generation, the reverse process starts from pure noise and applies T denoising steps
- The timestep `t` is always provided as an input so the network knows how noisy the current image is
- No noise is added on the final step (t=0) to produce the clean output

What loss function is used to train the diffusion model's denoising network?

## --answers--

Binary cross-entropy between real and fake image classifications

### --feedback--

Binary cross-entropy on real/fake classification describes a GAN discriminator's loss. Diffusion models have a different and simpler training objective.

---

Categorical cross-entropy over the vocabulary of possible pixel values

### --feedback--

Categorical cross-entropy over pixel values describes autoregressive pixel prediction. The diffusion model's loss is a simpler continuous-valued function.

---

Mean squared error between the predicted noise and the actual noise that was added

---

KL divergence between the generated distribution and the real data distribution

### --feedback--

KL divergence on distributions is used in VAE training. The diffusion model uses a much simpler direct comparison between predicted and actual noise values.
