# --interactive--

Each emitter property accepts one of three shapes:

1. **Number** — every particle uses this exact value: `speed: 200`.
2. **Range** — `{ min, max }` — Phaser picks uniformly per particle: `speed: { min: 100, max: 300 }`.
3. **Tween** — `{ start, end }` — interpolates over the particle's lifespan: `scale: { start: 1, end: 0 }`.

The five most-used properties:

```js
this.add.particles(x, y, key, {
  speed: { min: 100, max: 300 },     // initial speed in px/s, random per particle
  lifespan: { min: 300, max: 600 },  // how long each particle lives
  scale: { start: 1, end: 0 },       // shrinks 1 → 0 over its life
  rotate: { start: 0, end: 360 },    // spins 0 → 360° over its life
  alpha: { start: 1, end: 0 },       // fades 1 → 0 over its life
  quantity: 1
});
```

`scale.end: 0` and `alpha.end: 0` are the foundation of "graceful particle death" — particles vanish smoothly instead of popping out of existence. Without them you get sharp clipping at the lifespan boundary that reads as a glitch.

Speed direction is controlled by `angle` (in degrees, 0 = right):

```js
{ angle: { min: 0, max: 360 } }      // omnidirectional explosion
{ angle: { min: -120, max: -60 } }   // upward cone
{ angle: 90 }                         // every particle goes straight down
```

For "blood spatter" or "spark on metal" use a tight cone facing away from the impact. For an "explosion" use the full 360°. For "rising smoke" use a narrow upward cone.

Gravity:

```js
{ gravityY: 200 }   // pulls particles downward at 200 px/s²
{ gravityX: -50, gravityY: 100 }  // wind + gravity
```

Combined with a 360° angle, that gives you a hit burst that *settles* — particles spray out, then fall under gravity. Reads way more "physical" than pure radial.

:::interactive_editor

```js
function recipe(name) {
  const recipes = {
    "fire": {
      speed: { min: 50, max: 150 },
      lifespan: { min: 400, max: 800 },
      scale: { start: 0.6, end: 0 },
      alpha: { start: 1, end: 0 },
      angle: { min: -120, max: -60 },
      gravityY: -50,
      quantity: 2
    },
    "sparks": {
      speed: { min: 200, max: 500 },
      lifespan: { min: 200, max: 400 },
      scale: { start: 0.5, end: 0 },
      angle: { min: 0, max: 360 },
      gravityY: 300,
      frequency: -1
    },
    "snow": {
      speed: { min: 20, max: 60 },
      lifespan: { min: 4000, max: 8000 },
      scale: { start: 0.4, end: 0.4 },
      alpha: { start: 0.7, end: 0 },
      angle: 90,
      gravityY: 30,
      quantity: 1
    }
  };
  return recipes[name];
}

console.log("fire   :", recipe("fire"));
console.log("sparks :", recipe("sparks"));
console.log("snow   :", recipe("snow"));
```

:::

The fire recipe: narrow upward cone, slight negative gravity (so flames rise faster than ambient), small scale that fades out, fast inner-loop quantity. Continuous.

The sparks recipe: 360° spray, high speed, strong downward gravity, one-shot via `frequency: -1`. Use with `.explode(20)` on every hit.

The snow recipe: straight down, slow, very long lifespan, slight horizontal drift via `speedX`. Continuous on a top-edge emitter.

A common authoring mistake: forgetting `alpha: { start: 1, end: 0 }`. Particles abruptly disappear, which reads as bad collision detection or a render bug. Always fade your particles.

Another: setting `lifespan: 5000` on a one-shot. The particles are gone visually after 1s but the emitter keeps tracking 20 invisible particles for 4 more seconds. Cap `lifespan` at the longest visible duration; if you can't see them anymore, kill them.

# --questions--

## --text--

You want particles to fade out smoothly and shrink as they die, instead of popping. Which two config keys?

## --answers--

`alpha: { start: 1, end: 0 }` and `scale: { start: 1, end: 0 }` — both interpolate over each particle's lifespan.

---

`alpha.fadeOut: true` and `scale.shrink: true`.

### --feedback--

Those keys don't exist; the API uses `{ start, end }` ranges.

---

Set `lifespan: 0` to instantly kill particles.

### --feedback--

That makes them never appear, not fade gracefully.

---

`onDestroy: () => particle.alpha = 0`.

### --feedback--

There's no `onDestroy` per-particle hook; tween via `{ start, end }` instead.

## --text--

For a hit-spark burst that flies in all directions and falls under gravity, what `angle` and `gravityY` would you pick?

## --answers--

`angle: { min: 0, max: 360 }` for omnidirectional spray, `gravityY: 200` (or higher) so particles arc back down.

---

`angle: 0` and `gravityY: -200`.

### --feedback--

That fires every particle to the right and pulls them up. Wrong directions on both.

---

`angle: 90` and `gravityY: 0`.

### --feedback--

That fires straight down with no gravity — they don't spray.

---

`angle: -45` and `gravityY: 0`.

### --feedback--

Fires only up-and-right with no settling. Not omnidirectional.

## --text--

Why is `lifespan: 5000` on a one-shot hit-explosion a performance trap?

## --answers--

The 20 particles linger invisibly for 5 seconds after their visual fade. The emitter tracks 20 quads per hit × number of hits per second. On a 5/sec brawler that's 500 invisible quads at all times — pure overhead. Cap lifespan at the visible-fade duration.

---

Phaser doesn't allow `lifespan` over 1000.

### --feedback--

It allows any positive number.

---

Long-lifespan particles trigger garbage collection.

### --feedback--

GC isn't the issue; the issue is per-frame transform updates on every alive particle.

---

5000 caps your frame rate at 30 FPS automatically.

### --feedback--

There's no such cap.
