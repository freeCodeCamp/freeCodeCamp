# --interactive--

Neural networks only understand numbers. Before an LSTM or GRU can process text, that text must be converted into a numerical representation. This lesson covers the standard pipeline for preparing text data.

### The Pipeline

```
Raw text → Tokenization → Integer encoding → Padding → Embedding
```

Each step transforms the text into a form the network can process more effectively.

### Step 1: Tokenization

Tokenization splits text into individual units called **tokens**. The most common approach is word-level tokenization — splitting on whitespace and punctuation.

```python
sentence = "The cat sat on the mat"
tokens = sentence.lower().split()
# ['the', 'cat', 'sat', 'on', 'the', 'mat']
```

More sophisticated tokenizers handle contractions ("don't" → ["do", "n't"]), punctuation, and special characters. Subword tokenizers (used in transformers) can split rare words into smaller known pieces ("unhappiness" → ["un", "happiness"]).

### Step 2: Building a Vocabulary

A vocabulary maps each unique token to an integer ID. Common practice:

- Reserve ID 0 for padding
- Reserve ID 1 for unknown words (words not in the vocabulary)
- Sort words by frequency — most common words get lower IDs

```python
import tensorflow as tf

texts = [
    "the cat sat on the mat",
    "the dog sat on the log"
]

tokenizer = tf.keras.preprocessing.text.Tokenizer(num_words=1000)
tokenizer.fit_on_texts(texts)

print(tokenizer.word_index)
# {'the': 1, 'sat': 2, 'on': 3, 'cat': 4, 'mat': 5, 'dog': 6, 'log': 7}
```

### Step 3: Integer Encoding

Convert each text to a sequence of integer IDs using the vocabulary:

```python
sequences = tokenizer.texts_to_sequences(texts)
print(sequences)
# [[1, 4, 2, 3, 1, 5], [1, 6, 2, 3, 1, 7]]
```

### Step 4: Padding

Sequences in a batch must all be the same length. Shorter sequences are padded with zeros; longer ones are truncated.

```python
padded = tf.keras.preprocessing.sequence.pad_sequences(
    sequences,
    maxlen=10,
    padding='post',    # add zeros at the end
    truncating='post'  # cut from the end if too long
)
print(padded)
# [[1 4 2 3 1 5 0 0 0 0]
#  [1 6 2 3 1 7 0 0 0 0]]
```

### Step 5: The Embedding Layer

Integer IDs cannot be used directly — the number 4 is not "closer" to 5 than to 100 in any meaningful sense. An **Embedding layer** converts each integer ID into a dense vector of learnable numbers.

```python
# 10000 words in vocabulary, each represented as a 64-dimensional vector
embedding = tf.keras.layers.Embedding(input_dim=10000, output_dim=64)
```

The embedding layer learns that "king" and "queen" should have similar vector representations, and that "cat" and "feline" should be close together in the 64-dimensional space. These relationships emerge from training on the task.

### A Complete Text Preprocessing Example

```python
import tensorflow as tf
import numpy as np

texts  = ["I loved this film", "This film was terrible", "Great movie overall"]
labels = [1, 0, 1]

tokenizer = tf.keras.preprocessing.text.Tokenizer(num_words=1000, oov_token='')
tokenizer.fit_on_texts(texts)
sequences = tokenizer.texts_to_sequences(texts)
padded    = tf.keras.preprocessing.sequence.pad_sequences(sequences, maxlen=8, padding='post')

print(padded)
# Each row is one text, padded to length 8
```

### Key Points

- Text must be tokenized, integer-encoded, and padded before a neural network can process it
- A vocabulary maps each unique word to an integer ID
- Padding ensures all sequences in a batch have the same length
- The Embedding layer converts integer IDs into dense learnable vectors
- Embeddings capture semantic relationships — similar words end up with similar vectors

### --feedback--

Padding does not affect embedding quality. It serves a structural purpose related to how batches of sequences are processed.

---

All sequences in a batch must be the same length for the network to process them together

---

It prevents the network from memorising short sequences

### --feedback--

Padding is not a regularization technique. It is a practical requirement for batch processing.

---

It converts words to lowercase before tokenization

### --feedback--

Lowercasing is a text preprocessing step done before tokenization. Padding is applied after integer encoding to standardise sequence lengths.

The Embedding layer does not change sequence length. It changes the representation of each element from an integer to a dense vector.

---

It converts integer word IDs into dense learnable vectors

### --feedback--

### --feedback--

That describes the tokenizer and vocabulary mapping steps, which happen before the Embedding layer. The Embedding layer takes integer IDs as input and outputs vectors.

## --text--

Why can we not feed raw integer word IDs directly into an LSTM without an Embedding layer?

## --answers--

Integer IDs are too large for the LSTM to process without causing overflow

### --feedback--

Numerical overflow is not the issue. The problem is about the meaning (or lack thereof) encoded in raw integer IDs.

---

The LSTM requires inputs to be floating point, and integers cannot be converted

### --feedback--

Integers can easily be cast to floats. The issue is not data type conversion but the meaningfulness of the representation.

---

Raw integer IDs carry no semantic meaning — word 4 is not more similar to word 5 than to word 100

---

The LSTM can only process one word at a time, so all words must be encoded together first

### --feedback--

LSTMs do process one time step at a time — but that is their normal operation, not a reason to avoid integer IDs. The embedding is needed for a different reason related to representation quality.
