# --interactive--

Text classification is one application of recurrent networks. Time series forecasting is another — predicting future numerical values based on historical sequences. This lesson builds a model to forecast temperature using past readings.

### What is Time Series Forecasting?

A time series is a sequence of values recorded at regular time intervals: hourly temperature, daily stock prices, monthly sales figures. Forecasting means predicting one or more future values given a window of past values.

The core idea: use the last N time steps as input to predict the next value (or the next K values).

```
Input:  [T-9, T-8, T-7, ..., T-1, T]   (10 past values)
Output: [T+1]                           (next value)
```

### Preparing Time Series Data

Unlike text, time series data does not need tokenization. But it does need:

1. **Normalisation** — scale values to a standard range
2. **Windowing** — create input/output pairs by sliding a window through the series

```python
import numpy as np
import tensorflow as tf

# Generate a synthetic sine wave time series
time    = np.arange(0, 1000)
series  = np.sin(0.1 * time) + 0.1 * np.random.randn(1000)

# Normalise
mean, std = series[:800].mean(), series[:800].std()
series_norm = (series - mean) / std

# Create windowed dataset
def make_dataset(series, window_size=30, batch_size=32):
    dataset = tf.data.Dataset.from_tensor_slices(series)
    dataset = dataset.window(window_size + 1, shift=1, drop_remainder=True)
    dataset = dataset.flat_map(lambda w: w.batch(window_size + 1))
    dataset = dataset.map(lambda w: (w[:-1], w[-1]))  # last value is target
    return dataset.shuffle(1000).batch(batch_size).prefetch(1)

train_ds = make_dataset(series_norm[:800])
val_ds   = make_dataset(series_norm[800:])
```

### Building the Forecasting Model

```python
model = tf.keras.Sequential([
    # Input shape: (window_size, 1) — one feature per time step
    tf.keras.layers.LSTM(64, input_shape=(30, 1), return_sequences=True),
    tf.keras.layers.LSTM(32),
    tf.keras.layers.Dense(16, activation='relu'),
    tf.keras.layers.Dense(1)   # no activation — regression output
])

model.compile(
    optimizer='adam',
    loss='mean_squared_error',
    metrics=['mae']
)
```

Note the output layer uses no activation function — this is a regression task (predicting a continuous number), not classification.

### Reshaping for LSTM Input

LSTM layers expect 3D input: `(batch_size, time_steps, features)`. A univariate time series (one value per time step) needs to be reshaped to add the features dimension:

```python
# Univariate series: shape (800,) → (800, 1)
series_3d = series_norm[:800].reshape(-1, 1)
```

### Multi-step Forecasting

Forecasting a single next step is the simplest case. For **multi-step forecasting** (predicting the next 7 days, for example), change the output layer:

```python
tf.keras.layers.Dense(7)   # predict next 7 time steps
```

And adjust the target preparation to include 7 future values.

### RNNs vs Other Forecasting Approaches

RNNs are one of several approaches for time series forecasting:

| Approach | Strengths | Weaknesses |
|---|---|---|
| ARIMA (classical) | Interpretable, fast | Linear only, manual configuration |
| LSTM / GRU | Captures non-linear patterns | Needs more data, slower |
| 1D CNN | Fast, parallelisable | Less natural for long-range dependencies |
| Transformer | Best for long sequences | Requires large datasets |

### Key Points

- Time series forecasting predicts future values from a window of past values
- Data must be normalised and windowed before training
- LSTM input requires 3D shape: `(batch_size, time_steps, features)`
- The output layer for regression has no activation function; use MSE as the loss
- LSTMs capture non-linear temporal patterns that classical methods like ARIMA cannot

The LSTM's tanh output is an internal computation. The final Dense layer's activation is a separate design choice that depends on the task type.

### --feedback--

Vanishing gradients are a concern in deep hidden layers, not specifically in the output layer. The reason for no activation here is about the nature of the prediction target.

### --feedback--

The optimiser choice does not dictate the output activation. Linear (no activation) output is chosen because the target is a continuous value, not a probability.

## --text--

What input shape does an LSTM layer expect?

## --answers--

`(batch_size, features)` — a 2D tensor

### --feedback--

A 2D tensor is the input shape for a Dense layer. LSTM layers require an additional dimension to represent the sequence.

---

`(time_steps, features)` — a 2D tensor without the batch dimension

### --feedback--

TensorFlow always processes data in batches. The full input shape includes the batch dimension as the first axis.

---

`(batch_size, time_steps, features)` — a 3D tensor

---

`(batch_size, time_steps, features, channels)` — a 4D tensor

### --feedback--

4D tensors are used for image data (height × width × channels). LSTM input is 3D — one dimension for the sequence and one for the features at each step.
