# --interactive--

In this lesson you will build a complete sentiment analysis model using an LSTM. The task: classify movie reviews from the IMDB dataset as positive or negative.

### The IMDB Dataset

The IMDB dataset contains 50,000 movie reviews — 25,000 for training and 25,000 for testing — labelled as positive (1) or negative (0). It is built directly into TensorFlow and comes pre-tokenized with words already replaced by integer IDs.

```python
import tensorflow as tf
import numpy as np

VOCAB_SIZE = 10000
MAX_LEN    = 256

(x_train, y_train), (x_test, y_test) = tf.keras.datasets.imdb.load_data(
    num_words=VOCAB_SIZE
)

print(x_train.shape)   # (25000,) — each element is a variable-length list
print(x_train[0][:5])  # e.g., [1, 14, 22, 16, 43]
print(y_train[0])      # 1 (positive)
```

### Step 1: Pad the Sequences

```python
x_train = tf.keras.preprocessing.sequence.pad_sequences(
    x_train, maxlen=MAX_LEN, padding='post', truncating='post'
)
x_test = tf.keras.preprocessing.sequence.pad_sequences(
    x_test, maxlen=MAX_LEN, padding='post', truncating='post'
)

print(x_train.shape)  # (25000, 256)
```

### Step 2: Build the Model

```python
model = tf.keras.Sequential([
    # Convert integer IDs to 64-dimensional vectors
    tf.keras.layers.Embedding(input_dim=VOCAB_SIZE, output_dim=64,
                               input_length=MAX_LEN),

    # Bidirectional LSTM — reads the review forward and backward
    tf.keras.layers.Bidirectional(tf.keras.layers.LSTM(64, return_sequences=True)),

    # Second LSTM layer
    tf.keras.layers.LSTM(32),

    # Classification head
    tf.keras.layers.Dense(32, activation='relu'),
    tf.keras.layers.Dropout(0.3),
    tf.keras.layers.Dense(1, activation='sigmoid')   # binary output
])

model.summary()
```

### Step 3: Compile and Train

```python
model.compile(
    optimizer='adam',
    loss='binary_crossentropy',
    metrics=['accuracy']
)

early_stop = tf.keras.callbacks.EarlyStopping(
    monitor='val_loss', patience=3, restore_best_weights=True
)

history = model.fit(
    x_train, y_train,
    epochs=10,
    batch_size=128,
    validation_split=0.1,
    callbacks=[early_stop]
)
```

### Step 4: Evaluate

```python
test_loss, test_acc = model.evaluate(x_test, y_test)
print(f"Test accuracy: {test_acc:.4f}")
# Typically around 86-89%
```

### Step 5: Predict on New Reviews

```python
# The IMDB tokenizer uses a specific word-to-index mapping
word_index = tf.keras.datasets.imdb.get_word_index()

def encode_review(text):
    words  = text.lower().split()
    ids    = [word_index.get(w, 2) + 3 for w in words]  # +3 for reserved IDs
    padded = tf.keras.preprocessing.sequence.pad_sequences(
                 [ids], maxlen=MAX_LEN, padding='post')
    return padded

review = "This film was absolutely brilliant. I loved every moment of it."
encoded = encode_review(review)
score   = model.predict(encoded)[0][0]
print(f"Sentiment: {'Positive' if score > 0.5 else 'Negative'} ({score:.2f})")
```

### Why Bidirectional?

A standard LSTM reads a review from left to right. When it reaches the word "disappointing" near the end, it has already processed "despite a promising start" — but it can only look backward. A bidirectional LSTM adds a second pass reading right to left, so every word is informed by both its past and future context in the sentence.

### Key Points

- IMDB is a standard binary sentiment classification benchmark built into TensorFlow
- Reviews are padded to a fixed length before being fed into the model
- A Bidirectional LSTM reads the sequence in both directions, capturing fuller context
- Stacking LSTM layers allows the model to learn hierarchical sequential patterns
- The final sigmoid output represents the probability of a positive review

### --feedback--

Training speed is not the reason for post-padding. The choice of pre- vs post-padding affects what the RNN sees last, not its speed.

---

It ensures all sequences in a batch have the same length, with zeros added at the end

---

It removes stopwords like "the" and "a" from the end of reviews

### --feedback--

Padding adds zeros — it does not remove words. Stopword removal is a separate text preprocessing step not shown here.

---

It encodes unknown words as zeros at the end of each sequence

### --feedback--

Unknown words are typically encoded as a special OOV (out-of-vocabulary) token ID, not added at the end. Padding zeros are structural placeholders.

Why might a Bidirectional LSTM outperform a standard (unidirectional) LSTM for sentiment classification?

## --answers--

It processes the sequence twice as fast because it uses parallel computation

### --feedback--

Bidirectional LSTMs actually take longer to train because they run two passes. The advantage is about information quality, not speed.

---

It has twice as many parameters, which always leads to better performance

### --feedback--

More parameters do not automatically guarantee better performance — they can cause overfitting. The advantage of bidirectional is about capturing context from both directions.

---

It captures context from both future and past words at every position in the sequence

---

It eliminates the need for an Embedding layer

### --feedback--

The Embedding layer is always needed to convert integer IDs to meaningful dense vectors. Bidirectionality does not change this requirement.
