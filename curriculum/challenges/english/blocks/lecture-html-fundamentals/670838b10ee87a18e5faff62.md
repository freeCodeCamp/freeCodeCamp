# --interactive--

Forward propagation makes a prediction. Backpropagation figures out how to improve it. Together, these two processes form the complete training loop of a neural network.

### The Core Idea

After forward propagation computes a prediction and a loss function measures the error, the network needs to know which weights were responsible for that error and by how much. Backpropagation answers this question by computing the **gradient** of the loss with respect to every weight in the network.

The gradient tells you two things:
- **Direction**: should this weight increase or decrease to reduce the loss?
- **Magnitude**: how sensitive is the loss to changes in this weight?

### The Chain Rule

Backpropagation is an efficient application of the **chain rule** from calculus. The chain rule says that if one quantity depends on another, which depends on another, you can compute the overall rate of change by multiplying the individual rates.

In a neural network, the loss depends on the output layer, which depends on the hidden layers, which depend on the weights. Backpropagation starts at the output layer and works backward, computing gradients one layer at a time using the chain rule.

You do not need to implement this yourself — TensorFlow handles it automatically. But understanding the concept helps you debug and design better networks.

### Step by Step

1. **Forward pass** — compute the prediction and the loss
2. **Compute output gradient** — how does the loss change as the output changes?
3. **Propagate backward** — use the chain rule to compute gradients for each layer, moving from output to input
4. **Update weights** — adjust every weight in the direction that reduces the loss (this is done by the optimiser, covered in the next lesson)

### A Simple Analogy

Imagine you are adjusting the ingredients in a recipe and the dish came out too salty. You want to know which ingredient was most responsible.

- Salt was added directly → high responsibility
- The stock was salty, and you used a lot of it → medium responsibility  
- The vegetables had trace sodium → very low responsibility

Backpropagation does the mathematical equivalent: it traces the error backward through every step that contributed to it, assigning each weight a gradient that reflects its share of responsibility.

### What Backpropagation Does Not Do

A common misconception is that backpropagation updates the weights itself. It does not. Backpropagation only **computes the gradients**. A separate component called the **optimiser** uses those gradients to actually update the weights. This distinction matters because different optimisers use gradients in different ways.

### In TensorFlow

TensorFlow computes gradients automatically using a system called automatic differentiation. You rarely need to call backpropagation explicitly:

```python
import tensorflow as tf

x = tf.Variable(3.0)
with tf.GradientTape() as tape:
    y = x ** 2   # y = x²

gradient = tape.gradient(y, x)
print(gradient)  # Output: 6.0 (because dy/dx = 2x = 2*3 = 6)
```

During model training, `model.fit()` handles the gradient tape, loss computation, and weight updates automatically.

### Key Points

- Backpropagation computes the gradient of the loss with respect to every weight in the network
- It uses the chain rule from calculus to propagate gradients from output to input
- Backpropagation only computes gradients — the optimiser uses them to update weights
- TensorFlow handles backpropagation automatically via its automatic differentiation system
