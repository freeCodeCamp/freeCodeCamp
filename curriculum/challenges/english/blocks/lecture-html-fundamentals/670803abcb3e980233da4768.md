# --interactive--

If a neural network only multiplied inputs by weights and added biases, it would only be capable of learning straight-line relationships. Activation functions are what give neural networks the ability to learn complex, curved patterns.

### Why Activation Functions are Necessary

A weighted sum is a linear operation — no matter how many layers you stack, if each layer only does `output = weights * inputs + bias`, the whole network collapses into a single linear equation. It cannot learn curves, spirals, or any pattern that requires bending.

Activation functions introduce **non-linearity** by transforming the weighted sum before passing it to the next layer. This is what allows deep networks to approximate virtually any function.

### Sigmoid

The sigmoid function squashes any number into a range between 0 and 1:

```
sigmoid(x) = 1 / (1 + e^(-x))
```

Very large positive numbers become close to 1. Very large negative numbers become close to 0. Values near zero land around 0.5.

Sigmoid is useful for the output layer of a binary classification model (e.g., "is this email spam?") because the output can be interpreted as a probability. However, sigmoid is rarely used in hidden layers of modern networks because it suffers from the **vanishing gradient problem** — gradients shrink to near zero for extreme input values, making training very slow.

### Tanh

The tanh (hyperbolic tangent) function is similar to sigmoid but maps values to the range -1 to 1:

```
tanh(x) = (e^x - e^(-x)) / (e^x + e^(-x))
```

Because its outputs are centred around zero, tanh tends to train faster than sigmoid. It still suffers from vanishing gradients for extreme values, so it is also rarely used in deep hidden layers today.

### ReLU

ReLU (Rectified Linear Unit) is the most widely used activation function in modern deep learning:

```
relu(x) = max(0, x)
```

If the input is positive, the output equals the input. If the input is negative or zero, the output is zero. This simple rule has two powerful advantages: it is computationally fast, and it does not suffer from vanishing gradients for positive values.

ReLU is the default choice for hidden layers in most convolutional and fully connected networks.

### Leaky ReLU

A small variation on ReLU that allows a tiny gradient for negative inputs:

```
leaky_relu(x) = x if x > 0, else 0.01 * x
```

This solves the "dying ReLU" problem, where neurons that always receive negative input become permanently stuck outputting zero.

### Softmax

Softmax is used in the output layer of multi-class classification problems. It converts a vector of raw scores into a probability distribution — all outputs sum to 1.

```python
# Example: raw scores for 3 classes
scores = [2.0, 1.0, 0.1]
# After softmax: probabilities that sum to 1
# → [0.659, 0.242, 0.099]
```

### Choosing the Right Activation Function

| Layer | Recommended activation |
|---|---|
| Hidden layers (most networks) | ReLU |
| Hidden layers (recurrent networks) | Tanh |
| Output — binary classification | Sigmoid |
| Output — multi-class classification | Softmax |
| Output — regression (predicting a number) | None (linear) |

### Key Points

- Without activation functions, stacked layers collapse into a single linear equation
- Activation functions introduce non-linearity, allowing networks to learn complex patterns
- ReLU is the most common choice for hidden layers due to speed and resistance to vanishing gradients
- Sigmoid is used for binary classification outputs; Softmax for multi-class classification outputs
