# --interactive--

After forward propagation produces a prediction, the network needs to know how wrong it was. A loss function measures the gap between the predicted value and the correct answer. This measurement is the signal that drives all learning.

### What is a Loss Function?

A loss function (also called a cost function) takes two inputs — the network's prediction and the true label — and returns a single number called the **loss**. A high loss means the prediction was far off. A loss of zero means the prediction was perfect.

The goal of training is to find the weights and biases that make the loss as small as possible across the entire dataset.

### Mean Squared Error (MSE)

Mean Squared Error is the most common loss function for **regression** problems — tasks where the output is a continuous number, like predicting house prices or tomorrow's temperature.

```
MSE = (1/n) * Σ (predicted - actual)²
```

The squaring does two things: it makes all errors positive (so negative and positive errors do not cancel each other out), and it penalises large errors more heavily than small ones.

```python
# Example: predicting house prices
predicted = [250000, 310000, 190000]
actual    = [240000, 300000, 210000]

errors = [10000, 10000, -20000]
squared = [100000000, 100000000, 400000000]
mse = (100000000 + 100000000 + 400000000) / 3
    = 200000000
```

### Binary Cross-Entropy

Binary Cross-Entropy is used for **binary classification** problems — tasks where the output is one of two classes (e.g., spam or not spam, cat or dog).

```
loss = -[y * log(p) + (1-y) * log(1-p)]
```

Where `y` is the true label (0 or 1) and `p` is the predicted probability.

This function heavily penalises confident wrong predictions. If the correct label is 1 and the model predicts 0.01 (almost certain it is class 0), the loss is very large. If the model predicts 0.99 (almost certain it is class 1), the loss is nearly zero.

### Categorical Cross-Entropy

Categorical Cross-Entropy is the extension of binary cross-entropy for **multi-class classification** — tasks with more than two possible labels (e.g., classifying images as cat, dog, bird, fish, or rabbit).

```python
import tensorflow as tf

loss_fn = tf.keras.losses.CategoricalCrossentropy()

true_label = [0, 0, 1, 0, 0]   # bird
predicted   = [0.1, 0.1, 0.6, 0.1, 0.1]  # 60% confident it's bird

loss = loss_fn(true_label, predicted)
```

### Choosing the Right Loss Function

| Problem type | Loss function |
|---|---|
| Regression (predicting a number) | Mean Squared Error |
| Binary classification (2 classes) | Binary Cross-Entropy |
| Multi-class classification (3+ classes) | Categorical Cross-Entropy |

### Loss vs Accuracy

It is easy to confuse loss and accuracy:

- **Loss** is a continuous number computed directly from the raw predictions. The optimiser minimises this.
- **Accuracy** is a percentage calculated from rounded predictions. It is easy to understand but cannot be directly optimised because it is not differentiable.

A model's accuracy can stay the same across several training steps while its loss continues to decrease. This is normal — the model is becoming more confident in its correct answers even if the count of correct answers has not changed yet.

### Key Points

- A loss function measures the gap between a prediction and the correct answer
- Mean Squared Error is used for regression; cross-entropy variants are used for classification
- A loss of zero is a perfect prediction; higher loss means larger error
- The goal of training is to minimise the loss across the full dataset
- Loss and accuracy measure different things — the optimiser minimises loss, not accuracy
