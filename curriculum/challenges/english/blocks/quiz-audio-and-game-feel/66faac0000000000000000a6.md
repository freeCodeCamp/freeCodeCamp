### --question--

#### --text--

Why do `this.load.audio` calls usually pass an array of two URLs (one `.ogg` and one `.mp3`)?

#### --distractors--

To preload both formats so the same SFX can be played twice at once.

---

To halve the file size by streaming both formats in parallel.

---

To enable stereo playback (left channel from one file, right channel from the other).

#### --answer--

Browsers disagree on supported audio formats — Phaser tries each URL in order and uses the first one the runtime can decode.

### --question--

#### --text--

What's the difference between `this.sound.play(key)` and `this.sound.add(key)`?

#### --distractors--

`play` mutes globally and `add` plays through speakers.

---

`play` works only for music and `add` works only for SFX.

---

`play` returns the source `AudioBuffer`; `add` returns a `URL`.

#### --answer--

`play` is fire-and-forget for one-shots; `add` returns a reusable `Sound` instance you keep around (to call `pause`, `setVolume`, `setRate`, etc.).

### --question--

---

Changes duration only — pitch is preserved.

---

Changes the stereo balance, with `1` centred and `0.5` panned hard left.

#### --answer--

Changes playback speed AND pitch together — there's no independent pitch control in Web Audio without third-party DSP.

### --question--

#### --text--

What's the cleanest Phaser pattern to crossfade from one music track to another?

#### --distractors--

Stop the old track immediately, then `play` the new one with a `tween` on `volume` from 1 to 0.

---

Set `oldMusic.rate = 0` and `newMusic.rate = 2` simultaneously.

---

Use `this.sound.setMasterVolume(0)` for one frame before swapping tracks.

#### --answer--

Start the new track at `volume: 0` and `play` it; tween its volume up while tweening the old track's volume down; on the old track's complete callback, call `oldMusic.stop()`.

### --question--

#### --text--

What does "hit-pause" do, and how is it implemented in Phaser?

#### --distractors--

Calls `this.tweens.pauseAll()` for 40ms; tweens carry the squash animation through the freeze.

---

Sets `time.timeScale = 0` for a fixed duration; restores via `time.delayedCall`.

---

Calls `this.cameras.main.flash(40, 0, 0, 0)` to overlay a black flash that masks frame motion.

#### --answer--

Calls `this.physics.world.pause()` and schedules `this.physics.world.resume()` via `this.time.delayedCall(40, ...)` — physics stops, but tweens, time events, and the time-event scheduling continue.

### --question--

#### --text--

Why is hit-pause typically 40ms (not 200ms)?

#### --distractors--

Because Phaser's physics step is fixed at 40ms.

---

Because Web Audio drops samples below 25 Hz, which corresponds to 40ms.

---

Because tweens auto-pause for 40ms whenever the physics world stops.

#### --answer--

Because the brain's motion-perception threshold sits around 30–50ms — shorter feels invisible, longer starts feeling like lag rather than impact.

### --question--

#### --text--

What's the squash & stretch tween config that produces a 1.3× horizontal squash, 0.7× vertical squish, and snaps back to 1×1?

#### --distractors--

`{ targets: enemy, scaleX: 1.3, scaleY: 0.7, duration: 60, repeat: 1 }` — `repeat: 1` plays the tween twice.

---

`{ targets: enemy, scaleX: 1.3, scaleY: 0.7, duration: 60, ease: "Power2" }` — Phaser auto-yoyos `scaleX/Y` tweens.

---

`{ targets: enemy, scale: { from: 1, to: 1.3 }, duration: 120, hold: 60 }`.

#### --answer--

`{ targets: enemy, scaleX: 1.3, scaleY: 0.7, duration: 60, yoyo: true, ease: "Power2" }` — `yoyo: true` runs the tween forward then reverse, returning to `scale: 1`.

### --question--

#### --text--

Three valid forms of `speed:` on a particle emitter config?

#### --distractors--

`speed: 100`, `speed: [50, 200]`, `speed: { random: 0.5 }`.

---

`speed: 100`, `speed: { from: 50, to: 200 }`, `speed: function() { return 200; }`.

---

`speed: "fast"`, `speed: "fast..slow"`, `speed: { ease: "Power2" }`.

#### --answer--

A plain number (`100`), a min/max range (`{ min: 50, max: 200 }`), and a tween over the particle's lifetime (`{ start: 200, end: 0 }`).

### --question--

#### --text--

What's the difference between a "burst" emitter and a "continuous" emitter in Phaser 3.60+?

#### --distractors--

Burst emitters are 2D, continuous are 3D — different render pipelines.

---

Burst emitters use `add.particles`; continuous emitters use `add.particleEmitter`.

---

Burst emitters die after one frame; continuous emitters live forever.

#### --answer--

Continuous emitters tick on a `frequency` (ms between particles) and need `emitting: true`; burst emitters set `emitting: false` and only fire when you call `.explode(count, x, y)`.

### --question--

#### --text--

Why is it bad practice to call `this.add.particles(0, 0, "particle", config).explode(20, x, y)` on every hit?

#### --distractors--

It's fine — Phaser deduplicates emitters by config.

---

Phaser's renderer only allows one emitter per scene, so the new ones replace the old.

---

`add.particles` returns `null` after the third call per second.

#### --answer--

Each call creates a fresh `Emitter` game object that lives forever (unless explicitly destroyed) — pre-create one with `emitting: false` and reuse it via `.explode`.

### --question--

#### --text--

In a 2D top-down brawler, why do shake intensities like `cameras.main.shake(80, 0.005)` use 0.005 and not, say, 0.5?

#### --distractors--

Phaser internally divides the second argument by 100 before applying.

---

`shake` ignores values above 0.01 to prevent screen damage.

---

The first argument is in milliseconds; the second is in seconds — `0.005` seconds is 5ms.

#### --answer--

The second argument is intensity as a fraction of the canvas dimensions — `0.005` means a 0.5% offset (about 4px on an 800px-wide canvas), which feels punchy without being disorienting.

### --question--

#### --text--

A clean way to scale every juice magnitude by hit weight ("light hits feel small, heavy hits feel huge")?

#### --distractors--

Pass `weight` to `cameras.main.shake(80, 0.005, false, weight)` — the fourth arg multiplies intensity.

---

Use `tweens.timeline` instead of `tweens.add` — timelines auto-scale by duration weight.

---

Run two cameras and pick which one `shake` is called on.

#### --answer--

Compute a single `impact` multiplier (`heavy ? 2 : 1`) and apply it to every magnitude — `shake(80 * impact, 0.005 * impact)`, `fx.explode(20 * impact, ...)`, etc.

### --question--

#### --text--

What does `this.time.timeScale = 0.4` do, and why is the timer that resets it written `this.time.delayedCall(120, () => { this.time.timeScale = 1; })`?

#### --distractors--

Slows physics only — tweens keep running at full speed; the 120ms is real time.

---

Slows audio by 60% — visuals are unaffected; the timer is in real audio frames.

---

Slows the framerate from 60 fps to ~24 fps; the 120ms is in frames, not milliseconds.

#### --answer--

Slows ALL Phaser time (tweens AND delayed calls) by 60% — including the timer that resets `timeScale`, so 120ms scaled = 300ms real time, which is the actual slow-mo duration.

### --question--

#### --text--

Why register a `Phaser.Scenes.Events.SHUTDOWN` listener that destroys emitters and stops tweens?

#### --distractors--

It's optional — Phaser auto-destroys all scene game objects on shutdown.

---

It speeds up the scene reload by avoiding garbage collection.

---

It enables sound to play after the scene shuts down.

#### --answer--

Without it, restarting the scene (e.g., from a "Play Again" button) leaves the previous scene's emitters and tweens dangling — they keep ticking against destroyed targets and leak memory each restart.

# --explanation--

Audio loading uses array fallbacks so browsers pick a format they support, the first user gesture must call `sound.unlock()` to satisfy autoplay policy, and `this.sound.add(key)` returns a reusable instance for music while `this.sound.play(key)` is fire-and-forget for SFX. The juice stack stacks audio, squash/stretch yoyo tweens, tint flashes, camera shake (intensity is a screen-fraction), 40ms hit-pause via `physics.world.pause/resume`, and pre-created burst emitters fired with `.explode(count, x, y)`. Scale every magnitude by an `impact` multiplier so heavy hits feel different, suppress slow-mo when `prefers-reduced-motion: reduce` is set, and always wire a `SHUTDOWN` listener to destroy emitters and stop tweens.
