# --description--

Now that you know Python variables are references, you can see why simply writing `b = a` does not create an independent copy of the object. When you want a real copy, Python gives you two options: a **shallow copy** and a **deep copy**.

Understanding the difference is especially important when your data structures contain nested objects, like a list of lists.

## Assignment – No Copy at All

When you assign one variable to another, both names point to the exact same object. There is no copying:

```python
a = [[1, 2], [3, 4]]
c1 = a  # assignment – c1 and a share the same object

c1[0].append(99)

print(a)   # [[1, 2, 99], [3, 4]] – a is affected!
print(c1)  # [[1, 2, 99], [3, 4]]
```

## Shallow Copy – One Level Deep

A shallow copy creates a **new** top-level object, but the elements inside it are still shared references to the original nested objects.

You can make a shallow copy of a list in several equivalent ways:

```python
import copy

a = [[1, 2], [3, 4]]
c2 = copy.copy(a)    # shallow copy
# equivalent alternatives: a.copy()  |  list(a)  |  a[:]
```

Because the top-level list is new, adding or removing elements from `c2` does not affect `a`. But because the inner lists are still shared, mutating a nested object affects both:

```python
c2.append([5, 6])  # only c2 grows – top-level is independent
print(a)           # [[1, 2], [3, 4]]
print(c2)          # [[1, 2], [3, 4], [5, 6]]

c2[0].append(99)   # inner list is shared – both are affected
print(a)           # [[1, 2, 99], [3, 4]]
print(c2)          # [[1, 2, 99], [3, 4], [5, 6]]
```

## Deep Copy – Fully Independent

A deep copy creates a completely independent copy of the original object **and all objects nested within it**. Modifying the copy will never affect the original, no matter how deeply nested the objects are:

```python
import copy

a = [[1, 2], [3, 4]]
c3 = copy.deepcopy(a)

c3[0].append(99)

print(a)   # [[1, 2], [3, 4]] – completely unaffected
print(c3)  # [[1, 2, 99], [3, 4]]
```

## Summary

Here is a quick comparison of all three approaches:

| Approach | Top-level object | Nested objects |
|---|---|---|
| Assignment (`c = a`) | Shared | Shared |
| Shallow copy (`copy.copy(a)`) | New (independent) | Shared |
| Deep copy (`copy.deepcopy(a)`) | New (independent) | New (independent) |

Choose the approach that matches your needs:
- Use **assignment** when you intentionally want two names for the same object.
- Use a **shallow copy** when your data has no nested mutable objects, or when sharing nested objects is acceptable.
- Use a **deep copy** when you need a fully independent duplicate.
