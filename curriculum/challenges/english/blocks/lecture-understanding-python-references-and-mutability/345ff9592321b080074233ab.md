# --description--

In Python, a variable is not a container that holds a value directly. Instead, it is a **name** that refers, or points, to an object stored in memory.

When you write `x = 10`, Python creates an integer object with the value `10` in memory, and then the name `x` is bound to that object:

```python
x = 10
print(x)  # 10
```

You can use Python's built-in `id()` function to see the unique memory address of any object:

```python
x = 10
print(id(x))  # e.g., 140234567891234 (the exact value will differ each run)
```

## Assigning One Variable to Another

When you assign one variable to another, both names point to the **same** object in memory. No copy is made:

```python
a = [1, 2, 3]
b = a

print(id(a) == id(b))  # True – both point to the same list
```

This is called a **shared reference**.

## Mutability and Shared References

Whether a shared reference causes problems depends on whether the object is **mutable** or **immutable**.

- **Immutable** objects (like `int`, `float`, `str`, `tuple`) cannot be changed after they are created. If you reassign one name, the other is unaffected:

```python
a = 5
b = a

a = 10  # a now points to a new object; b is unchanged

print(a)  # 10
print(b)  # 5
```

- **Mutable** objects (like `list`, `dict`, `set`) can be changed in place. Because both names share the same object, a change made through one name is visible through the other:

```python
a = [4, 3, 2]
b = a

b.append(1)  # modifies the shared list

print(a)  # [4, 3, 2, 1] – a is also affected!
print(b)  # [4, 3, 2, 1]
```

This is a common source of bugs for Python beginners: after writing `b = a`, you might expect `b` to be an independent copy, but it is not.

## Function Parameters Are New References

When you pass a variable to a function, Python creates a new name inside the function that points to the **same** object:

```python
def add_item(lst):
    lst.append(99)

my_list = [1, 2, 3]
add_item(my_list)

print(my_list)  # [1, 2, 3, 99] – the original list was modified
```

The parameter `lst` inside `add_item` is a new name, but it refers to the same list object. Any in-place modification made inside the function will be visible outside.

However, reassigning the parameter inside the function does **not** affect the original variable, because reassignment just makes the local name point to a different object:

```python
def replace_list(lst):
    lst = [10, 20, 30]  # creates a new local reference; original is untouched

my_list = [1, 2, 3]
replace_list(my_list)

print(my_list)  # [1, 2, 3] – unchanged
```
