# --interactive--

Training a transformer from scratch requires millions of dollars of compute. Hugging Face's `transformers` library gives you access to hundreds of pre-trained models — BERT, GPT-2, T5, RoBERTa, and more — in a few lines of Python. This lesson shows you how to use them for real NLP tasks.

### Installing the Library

```bash
pip install transformers tensorflow datasets
```

### The Pipeline API: Fastest Way to Get Results

The `pipeline` function wraps a pre-trained model and tokenizer into a single object that handles everything:

```python
from transformers import pipeline

# Sentiment analysis
classifier = pipeline('sentiment-analysis')
result = classifier("This course on deep learning is fantastic!")
print(result)
# [{'label': 'POSITIVE', 'score': 0.9998}]

# Text generation
generator = pipeline('text-generation', model='gpt2')
output = generator("Neural networks learn by", max_new_tokens=30)
print(output[0]['generated_text'])

# Question answering
qa = pipeline('question-answering')
answer = qa(
    question="What is backpropagation?",
    context="Backpropagation is an algorithm that computes gradients by applying the chain rule from output to input layers."
)
print(answer['answer'])  # "an algorithm that computes gradients"

# Named entity recognition
ner = pipeline('ner', aggregation_strategy='simple')
entities = ner("Yann LeCun works at Meta in New York.")
print(entities)
# [{'entity_group': 'PER', 'word': 'Yann LeCun'}, {'entity_group': 'ORG', 'word': 'Meta'}, ...]
```

### Tokenizing Text

Behind the pipeline, the tokenizer converts text to token IDs. You can use it directly when you need more control:

```python
from transformers import BertTokenizer

tokenizer = BertTokenizer.from_pretrained('bert-base-uncased')

text   = "Deep learning is revolutionising AI."
tokens = tokenizer(text, return_tensors='tf', padding=True, truncation=True, max_length=128)

print(tokens['input_ids'])      # integer token IDs
print(tokens['attention_mask']) # 1 for real tokens, 0 for padding
```

BERT tokenizers automatically add special tokens:
- `[CLS]` — at the start of every input (used for classification)
- `[SEP]` — separates sentences or marks the end

### Fine-tuning BERT for Text Classification

```python
from transformers import BertTokenizer, TFBertForSequenceClassification
from datasets import load_dataset
import tensorflow as tf

# Load dataset
dataset   = load_dataset('imdb')
tokenizer = BertTokenizer.from_pretrained('bert-base-uncased')

def tokenize(batch):
    return tokenizer(batch['text'], padding=True, truncation=True, max_length=256)

tokenized = dataset.map(tokenize, batched=True)
tokenized.set_format('tensorflow', columns=['input_ids', 'attention_mask', 'label'])

# Prepare TensorFlow datasets
def to_tf_dataset(split):
    features = {
        'input_ids':      tokenized[split]['input_ids'],
        'attention_mask': tokenized[split]['attention_mask']
    }
    labels = tokenized[split]['label']
    return tf.data.Dataset.from_tensor_slices((features, labels)).batch(16)

train_ds = to_tf_dataset('train')
test_ds  = to_tf_dataset('test')

# Load pre-trained BERT with a classification head
model = TFBertForSequenceClassification.from_pretrained(
    'bert-base-uncased', num_labels=2
)

model.compile(
    optimizer=tf.keras.optimizers.Adam(learning_rate=2e-5),
    loss=tf.keras.losses.SparseCategoricalCrossentropy(from_logits=True),
    metrics=['accuracy']
)

model.fit(train_ds, validation_data=test_ds, epochs=3)
```

A learning rate of `2e-5` is typical for fine-tuning transformers — low enough to avoid destroying pre-trained weights.

### Choosing the Right Model

| Task | Recommended model |
|---|---|
| Text classification / sentiment | `bert-base-uncased`, `distilbert-base-uncased` |
| Text generation | `gpt2`, `facebook/opt-350m` |
| Question answering | `deepset/roberta-base-squad2` |
| Translation | `Helsinki-NLP/opus-mt-en-fr` |
| Summarisation | `facebook/bart-large-cnn` |
| Zero-shot classification | `facebook/bart-large-mnli` |

All available at `huggingface.co/models`.

### Key Points

- The Hugging Face `transformers` library provides hundreds of pre-trained models with a simple API
- The `pipeline` function is the fastest way to run sentiment analysis, generation, QA, and NER
- Tokenizers handle text-to-token-ID conversion and add special tokens like `[CLS]` and `[SEP]`
- Fine-tuning BERT requires a very low learning rate (around 2e-5) to preserve pre-trained knowledge
- The Hugging Face Model Hub at huggingface.co/models hosts thousands of community fine-tuned models

What are the `[CLS]` and `[SEP]` tokens that BERT tokenizers add?

## --answers--

Tokens that mark incorrect grammar so BERT can skip those positions

### --feedback--

`[CLS]` and `[SEP]` are structural tokens with specific roles in the input format. They are not related to grammatical correctness.

---

Random noise tokens added to prevent the model from memorising input positions

### --feedback--

These are deterministic special tokens with defined roles. `[CLS]` is used for classification output; `[SEP]` separates sentence segments.

---

Special tokens where `[CLS]` marks the start (used for classification) and `[SEP]` separates sentences or marks the end

---

Tokens used only during pre-training that are removed before fine-tuning

### --feedback--

`[CLS]` and `[SEP]` are present during both pre-training and fine-tuning. The `[CLS]` token's representation is specifically used as the input to the classification head.
