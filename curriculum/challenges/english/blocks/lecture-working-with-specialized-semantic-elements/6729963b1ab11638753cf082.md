# --interactive--

Every model you have built so far has been a discriminative model — it learns to classify or predict from existing data. Generative models learn something fundamentally different: the underlying distribution of the data itself, so they can create entirely new examples.

### Discriminative vs Generative Models

A **discriminative model** learns the boundary between classes. Given an input X, it predicts a label Y. It answers the question: "Given this image, is it a cat or a dog?"

A **generative model** learns the distribution of the data itself. It answers a different question: "What does a cat image look like?" and can produce new cat images that never existed before.

| | Discriminative | Generative |
|---|---|---|
| Learns | Boundary between classes | Distribution of the data |
| Input → Output | Image → Label | Noise/condition → New data |
| Examples | CNN classifier, BERT | GAN, VAE, diffusion model |
| Use case | Classification, regression | Image synthesis, data augmentation |

### Why Generative Models Matter

Generative models have practical applications across many domains:

- **Data augmentation** — generate additional training examples when real data is scarce
- **Image synthesis** — create photorealistic faces, artwork, or product images
- **Drug discovery** — generate novel molecular structures with desired properties
- **Super resolution** — generate high-resolution details from low-resolution images
- **Inpainting** — fill in missing or damaged regions of an image
- **Style transfer** — redraw an image in the style of a painting
- **Text-to-image** — generate images from natural language descriptions

### The Core Challenge: Learning a Distribution

Real-world data like images occupies a tiny region of all possible pixel combinations. A 64×64 colour image has 64 × 64 × 3 = 12,288 pixel values, each ranging from 0 to 255. The number of possible images is astronomically large, but the set of images that look like real photographs is vanishingly small within that space.

A generative model must learn to navigate this space — to find the manifold where real data lives and generate new points on it.

### Major Generative Architectures

**Variational Autoencoders (VAEs)** — encode data into a compressed latent space and decode samples back into data. Tend to produce blurry images but are stable to train.

**Generative Adversarial Networks (GANs)** — two networks compete: one generates fake data, the other distinguishes real from fake. Produces sharp, realistic outputs but can be unstable to train.

**Diffusion Models** — gradually add noise to data and learn to reverse the process. The basis of Stable Diffusion and DALL-E 2. Currently state of the art for image quality.

**Autoregressive Models** — generate data one element at a time (pixel by pixel, or token by token). GPT is an autoregressive generative model for text.

### Key Points

- Discriminative models predict labels; generative models learn data distributions and create new examples
- Generative models are used for image synthesis, data augmentation, drug discovery, and more
- The core challenge is learning to navigate the high-dimensional space of real data
- Major generative architectures include VAEs, GANs, diffusion models, and autoregressive models
- GANs produce sharp realistic outputs and are the focus of this block

## --text--

What is the fundamental difference between a discriminative model and a generative model?

## --answers--

Discriminative models use supervised learning; generative models use unsupervised learning exclusively

### --feedback--

Both types can involve supervision. GANs, for example, use a form of supervision through the discriminator's feedback. The key difference is what each model learns to do.

---

Discriminative models learn boundaries between classes; generative models learn the underlying data distribution and can create new examples

---

Discriminative models are used for images; generative models are used for text

### --feedback--

Both types can be applied to images, text, audio, and other data types. The distinction is about what the model learns, not the data modality.

---

Discriminative models require more training data than generative models

### --feedback--

Generative models typically require large amounts of data to learn realistic distributions. Data requirements are not the key distinction between the two types.

---

### --feedback--

Sorting by existing ratings is a retrieval or ranking task. Generative models create new content rather than organise existing content.
