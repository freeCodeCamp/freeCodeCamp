# --interactive--

GPT (Generative Pre-trained Transformer) is a family of decoder-only transformer models developed by OpenAI. While BERT was designed for understanding tasks, GPT was designed for generation — producing coherent, contextual text by predicting one token at a time.

### The Architecture: Decoder-Only

GPT uses only the decoder portion of the transformer, but without the cross-attention layer (since there is no encoder to attend to). Each layer has:

1. **Masked multi-head self-attention** — each token can only attend to previous tokens (causal masking)
2. **Feed-forward network** — applied independently to each position

The causal masking means GPT is fundamentally left-to-right: when generating token at position N, it has only seen positions 1 through N-1. This makes it naturally suited for text generation.

### Pre-training: Causal Language Modelling

GPT is pre-trained with a simple objective: predict the next token given all previous tokens.

```
Input:  "The cat sat on the"
Target: "mat"

Input:  "The cat sat on"
Target: "the"

Input:  "The cat sat"
Target: "on"
```

This is an entirely self-supervised task — any large text corpus provides unlimited training examples with no human labelling required. GPT-3 was trained on roughly 500 billion tokens of text from the internet, books, and Wikipedia.

### The GPT Family

| Model | Year | Parameters | Key development |
|---|---|---|---|
| GPT-1 | 2018 | 117M | Proved pre-train/fine-tune for generation |
| GPT-2 | 2019 | 1.5B | "Too dangerous to release" — later released fully |
| GPT-3 | 2020 | 175B | Few-shot learning without fine-tuning |
| GPT-4 | 2023 | ~1T (estimated) | Multimodal, dramatically improved reasoning |

### Few-Shot and Zero-Shot Learning

One of the remarkable properties that emerged in GPT-3 was **in-context learning** — the ability to perform new tasks simply by providing a few examples in the prompt, without updating any weights.

```
Prompt:
Translate English to French:
English: sea → French: mer
English: sky → French: ciel
English: mountain → French:

GPT-3 output: montagne
```

This is called **few-shot learning**. GPT-3 was never explicitly trained to translate — it inferred the task from the pattern in the prompt. This emergent capability was a major surprise and pointed toward the power of very large language models.

### Generating Text with GPT

```python
from transformers import GPT2Tokenizer, TFGPT2LMHeadModel
import tensorflow as tf

tokenizer = GPT2Tokenizer.from_pretrained('gpt2')
model     = TFGPT2LMHeadModel.from_pretrained('gpt2')

prompt  = "Deep learning is transforming the field of"
inputs  = tokenizer(prompt, return_tensors='tf')

outputs = model.generate(
    inputs['input_ids'],
    max_new_tokens=50,
    temperature=0.7,     # lower = more focused, higher = more creative
    do_sample=True
)

print(tokenizer.decode(outputs[0], skip_special_tokens=True))
```

### Temperature and Sampling

When generating text, the model produces a probability distribution over the vocabulary at each step. How the next token is selected from this distribution controls the quality and creativity of the output:

- **Greedy decoding** — always pick the most probable token. Fast but repetitive.
- **Temperature sampling** — divide logits by temperature before softmax. Low temperature (0.1) makes output more focused; high temperature (1.5) makes it more random.
- **Top-k sampling** — only sample from the top-k most probable tokens.
- **Top-p (nucleus) sampling** — sample from the smallest set of tokens whose cumulative probability exceeds p.

### Key Points

- GPT is a decoder-only transformer pre-trained to predict the next token
- Causal masking ensures each position can only attend to previous positions
- GPT-3 demonstrated that very large models can perform new tasks from examples in the prompt without any weight updates
- Temperature, top-k, and top-p sampling control the creativity and coherence of generated text
- The GPT family grew from 117M parameters (GPT-1) to an estimated ~1 trillion (GPT-4)

It makes the output longer and more detailed

### --feedback--

Temperature controls the randomness of token selection, not the length of the output. Length is controlled by the `max_new_tokens` parameter.

---

It makes the model's output more focused and deterministic by sharpening the probability distribution

---

### --feedback--

Increasing diversity by flattening the distribution is the effect of a high temperature. Low temperature has the opposite effect.

---

### --feedback--

GPT always generates tokens from its vocabulary, which it learned from training data regardless of temperature. Temperature controls the shape of the probability distribution over those tokens.
