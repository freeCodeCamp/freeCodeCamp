## HTML Form Elements and Attributes

### Form Structure

Use the `<form>` element to create a form. Key attributes:


```html
<form method="POST" action="/submit-form">
  <!-- form elements go here -->
</form>
```

- `action` - URL where form data will be sent
- `method` - HTTP method (`GET` or `POST`)

### Input Elements

#### Text Input


```html
<!-- Text input -->
<input 
  type="text"
  id="username"
  name="username"
  placeholder="Enter your username" 
  size="20"
  minlength="3"
  maxlength="20"
  required
/>
```

**Attributes:**

- `type` - input type (`text`, `email`, `number`, `password`, etc.)
- `placeholder` - hint text shown in the field
- `name` - key used when form is submitted
- `size` - number of visible characters
- `minlength` / `maxlength` - minimum and maximum character count
- `required` - field must be filled before submission
- `disabled` - field is disabled
- `readonly` - field is read-only

#### Number Input


```html
<input 
  type="number"
  id="age"
  name="age"
  min="18"
  max="100"
  value="25"
/>
```

`min` and `max` attributes set boundaries for number fields.

#### Radio Buttons


```html
<fieldset>
  <legend>Select your gender:</legend>
  
  <label for="male">
    <input id="male" type="radio" name="gender" value="male" />
    Male
  </label>
  
  <label for="female">
    <input id="female" type="radio" name="gender" value="female" />
    Female
  </label>
</fieldset>
```

**Note:** Radio buttons with the same `name` attribute form a group, and only one can be selected.

#### Checkboxes


```html
<fieldset>
  <legend>Select your interests:</legend>
  
  <label for="html">
    <input type="checkbox" id="html" name="html" value="html" />
    HTML
  </label>
  
  <label for="css">
    <input type="checkbox" id="css" name="css" value="css" />
    CSS
  </label>
  
  <label for="js">
    <input type="checkbox" id="js" name="js" value="javascript" />
    JavaScript
  </label>
</fieldset>
```

### Label Elements

Two ways to associate labels with inputs:

**Implicit association:**


```html
<label>
  Email:
  <input type="email" name="email" />
</label>
```

**Explicit association:**


```html
<label for="user-email">Email:</label>
<input type="email" id="user-email" name="email" />
```

The `for` attribute must match the input's `id`.

### Button Elements


```html
<!-- Submit button - submits the form -->
<button type="submit">Submit</button>

<!-- Reset button - resets form to initial values -->
<button type="reset">Reset</button>

<!-- Regular button - no default behavior -->
<button type="button">Click Me</button>
```

You can also use `<input type="button">`:


```html
<input type="button" value="Show Alert" />
<input type="submit" value="Send Form" />
```

### Fieldset and Legend

Group related inputs together:


```html
<fieldset>
  <legend>Personal Information</legend>
  
  <label for="firstname">First Name:</label>
  <input type="text" id="firstname" name="firstname" />
  
  <label for="lastname">Last Name:</label>
  <input type="text" id="lastname" name="lastname" />
</fieldset>
```

### Complete Form Example


```html
<form method="POST" action="/register">
  <fieldset>
    <legend>Registration Form</legend>
    
    <label for="email">Email:</label>
    <input 
      type="email" 
      id="email" 
      name="email" 
      placeholder="user@example.com"
      required 
    />
    
    <label for="password">Password:</label>
    <input 
      type="password" 
      id="password" 
      name="password" 
      minlength="8"
      required 
    />
    
    <label for="age">Age:</label>
    <input 
      type="number" 
      id="age" 
      name="age" 
      min="18" 
      max="120"
    />
    
    <fieldset>
      <legend>Newsletter subscription:</legend>
      
      <label for="yes">
        <input type="radio" id="yes" name="newsletter" value="yes" />
        Yes
      </label>
      
      <label for="no">
        <input type="radio" id="no" name="newsletter" value="no" />
        No
      </label>
    </fieldset>
    
    <button type="submit">Register</button>
    <button type="reset">Clear Form</button>
  </fieldset>
</form>
```

## HTML Tables

### Basic Table Structure


```html
<table>
  <caption>Student Grades</caption>
  
  <thead>
    <tr>
      <th>Name</th>
      <th>Subject</th>
      <th>Grade</th>
    </tr>
  </thead>
  
  <tbody>
    <tr>
      <td>John Doe</td>
      <td>Math</td>
      <td>95</td>
    </tr>

    <tr>
      <td>Jane Smith</td>
      <td>Math</td>
      <td>88</td>
    </tr>
  </tbody>
  
  <tfoot>
    <tr>
      <td colspan="2">Average Grade</td>
      <td>91.5</td>
    </tr>
  </tfoot>
</table>
```

### Table Elements

- `<table>` - creates the table
- `<caption>` - adds a title to the table
- `<thead>` - groups header content
- `<tbody>` - groups body content
- `<tfoot>` - groups footer content
- `<tr>` - creates a table row
- `<th>` - creates a header cell
- `<td>` - creates a data cell

### The colspan Attribute

Use `colspan` to make a cell span multiple columns:


```html
<table>
  <thead>
    <tr>
      <th>Product</th>
      <th>Q1</th>
      <th>Q2</th>
      <th>Total</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>Laptops</td>
      <td>150</td>
      <td>200</td>
      <td>350</td>
    </tr>
    <tr>
      <td colspan="3">Total Sales</td>
      <td>350</td>
    </tr>
  </tbody>
</table>
```

### Practical Table Example


```html
<table>
  <caption>Monthly Sales Report</caption>
  
  <thead>
    <tr>
      <th>Month</th>
      <th>Units Sold</th>
      <th>Revenue</th>
    </tr>
  </thead>
  
  <tbody>
    <tr>
      <td>January</td>
      <td>1,200</td>
      <td>$48,000</td>
    </tr>
    <tr>
      <td>February</td>
      <td>1,450</td>
      <td>$58,000</td>
    </tr>
    <tr>
      <td>March</td>
      <td>1,680</td>
      <td>$67,200</td>
    </tr>
  </tbody>
  
  <tfoot>
    <tr>
      <td>Total</td>
      <td>4,330</td>
      <td>$173,200</td>
    </tr>
  </tfoot>

```

## HTML Development Tools

### HTML Validator

Checks your HTML syntax for errors. You can use:

- W3C Markup Validation Service (validator.w3.org)
- Browser extensions
- IDE plugins

**Why validate?**

- Ensures cross-browser compatibility
- Catches syntax errors early
- Improves accessibility
- Maintains web standards

### DOM Inspector

Allows you to inspect and modify HTML structure in real-time. Available in browser DevTools.

**How to access:**

- Chrome/Edge: F12 or Right-click → Inspect
- Firefox: F12 or Right-click → Inspect Element
- Safari: Enable DevTools in Preferences → Right-click → Inspect Element

### Browser DevTools

Built-in tools for debugging and analyzing web pages:

**Elements/Inspector tab:**

- View and edit HTML structure
- Modify CSS styles in real-time
- See computed styles
- Check element box model

**Console tab:**

- View JavaScript errors
- Execute JavaScript code
- Debug scripts

**Network tab:**

- Monitor HTTP requests
- Check load times
- Analyze performance

## Best Practices

### Forms

1. Always use labels with inputs for accessibility
2. Use appropriate input types (`email`, `tel`, `url`, etc.)
3. Add `required` attribute for mandatory fields
4. Use `placeholder` for helpful hints, not essential information
5. Group related inputs with `fieldset` and `legend`
6. Provide clear error messages
7. Use meaningful `name` attributes for server-side processing

### Tables

1. Always use `<thead>`, `<tbody>`, and `<tfoot>` for better structure
2. Add `<caption>` to describe table content
3. Use `<th>` for headers, not `<td>` with bold text
4. Keep tables simple and avoid nested tables
5. Use CSS for styling, not HTML attributes
6. Make tables responsive for mobile devices
7. Use `colspan` and `rowspan` sparingly

### Accessibility

1. Associate labels with inputs properly
2. Provide descriptive button text
3. Use semantic HTML elements
4. Add ARIA attributes when necessary
5. Ensure forms can be navigated with keyboard
6. Test with screen readers
7. Maintain proper heading hierarchy

## Common Mistakes to Avoid

**Forms:**

- Forgetting to add `name` attributes (data won't be submitted)
- Not associating labels with inputs
- Using placeholder as a replacement for labels
- Missing `type` attribute on buttons (defaults to submit)

**Tables:**

- Not using `<thead>`, `<tbody>`, `<tfoot>` structure
- Using tables for layout instead of CSS
- Forgetting to close tags properly
- Missing `<caption>` for accessibility

## Testing Checklist

Before deploying your forms and tables:

- [ ] Validate HTML with W3C validator
- [ ] Test form submission with various inputs
- [ ] Check required field validation
- [ ] Test with keyboard navigation only
- [ ] Verify responsive design on mobile
- [ ] Check accessibility with screen reader
- [ ] Test in multiple browsers
- [ ] Inspect with DevTools for errors
- [ ] Verify table data displays correctly
- [ ] Check form data reaches server correctly
