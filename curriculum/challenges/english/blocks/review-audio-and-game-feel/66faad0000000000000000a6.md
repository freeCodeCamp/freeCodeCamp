## Loading audio with format fallbacks

Web audio is a format zoo. Safari/iOS prefer AAC/MP3, Chrome and Firefox have native Ogg, and the safe move is to ship both. Phaser accepts an array of URLs and the runtime picks the first format the browser supports.

:::interactive_editor

```js
function preload() {
  this.load.audio("sfx-hit", [
    "/curriculum-assets/phaser/audio/sfx-hit.ogg",
    "/curriculum-assets/phaser/audio/sfx-hit.mp3"
  ]);
  this.load.audio("music-ambient", [
    "/curriculum-assets/phaser/audio/music-ambient.ogg",
    "/curriculum-assets/phaser/audio/music-ambient.mp3"
  ]);
}
```

:::

Most browsers block audio until a user gesture happens. Always have a "tap to enable" overlay or treat the first click on the canvas as the unlock event — `if (this.sound.locked) this.sound.unlock();`.

## Playing one-shots vs holding instances

`this.sound.play(key)` is fire-and-forget. Use it for SFX where you don't care about the resulting `Sound` instance.

`this.sound.add(key)` returns a `Sound` you can keep — call `.play()`, `.pause()`, `.resume()`, `.setVolume()`, `.setRate()`, `.setLoop()`, `.stop()`. Use it for music and any SFX you'll re-tune at runtime.

:::interactive_editor

```js
function create() {
  this.sound.play("sfx-coin", { volume: 0.5 });

  this.music = this.sound.add("music-ambient", {
    loop: true,
    volume: 0.3
  });
  this.music.play();
}
```

:::

`{ volume, rate, detune, loop, mute, seek }` is the config object accepted by both `play(key, config)` and `add(key, config)`.

## Volume, pan, and rate

Each `Sound` instance has `setVolume(0..1)`, `setPan(-1..1)`, and `setRate(0.5..2)`. Pan is stereo position (left ear/right ear); rate is playback speed AND pitch tied together — there's no independent pitch knob in Web Audio.

:::interactive_editor

```js
function create() {
  const ping = this.sound.add("sfx-ping");

  this.input.on("pointerdown", (pointer) => {
    const pan = (pointer.x / 600) * 2 - 1;
    const rate = 0.7 + (1 - pointer.y / 480) * 0.6;
    ping.play({ pan, rate });
  });
}
```

:::

For UI-style audio, dropping the volume by 10-20% on click feedback (`{ volume: 0.4 }` instead of `1`) helps the SFX bed under voice/music so it never feels punchy.

## Two-volume buses: SFX vs music

Most production games expose two sliders: one for SFX, one for music. Track each as a property and apply it on every play call.

:::interactive_editor

```js
function create() {
  this.sfxVolume = parseFloat(localStorage.getItem("sfx-vol") ?? "0.7");
  this.musicVolume = parseFloat(localStorage.getItem("music-vol") ?? "0.3");

  this.playSfx = (key) =>
    this.sound.play(key, { volume: this.sfxVolume });

  this.music = this.sound.add("music-ambient", {
    loop: true,
    volume: this.musicVolume
  });
  this.music.play();
}
```

:::

Persist both volumes to `localStorage` so the player's tuning survives reloads. Always `parseFloat` with a default — `localStorage.getItem` returns `null` for missing keys.

## Music fades and crossfades

A hard cut between tracks sounds amateur. `this.tweens.add({ targets: music, volume: 0, duration: 800 })` fades a track to silence; chain it with `onComplete: () => music.stop()` to free the slot.

A crossfade plays both tracks at once for a beat: ramp the outgoing music down while ramping the incoming music up.

:::interactive_editor

```js
function crossfadeTo(scene, oldMusic, newKey) {
  const next = scene.sound.add(newKey, { loop: true, volume: 0 });
  next.play();
  scene.tweens.add({
    targets: oldMusic,
    volume: 0,
    duration: 800,
    onComplete: () => oldMusic.stop()
  });
  scene.tweens.add({
    targets: next,
    volume: scene.musicVolume,
    duration: 800
  });
  return next;
}
```

:::

The receiving track must `.play()` before the tween starts — Phaser tween targets must already be playing for the volume property to be heard.

## prefers-reduced-motion as an audio default

Players who set `prefers-reduced-motion: reduce` are often sensitive to loud transients too. A reasonable default is to start music muted (player can toggle on) and to clamp SFX volume.

:::interactive_editor

```js
function create() {
  const reduced = window.matchMedia(
    "(prefers-reduced-motion: reduce)"
  ).matches;

  this.sfxVolume = reduced ? 0.3 : 0.7;
  this.music = this.sound.add("music-ambient", {
    loop: true,
    volume: 0
  });
  this.music.play();
  if (!reduced) {
    this.tweens.add({
      targets: this.music,
      volume: 0.3,
      duration: 1500
    });
  }
}
```

:::

Reduced-motion users can still use the mute/unmute toggle — we just don't auto-fade music in. The default assumes "less stimulating".

## Anti-double-play guard

Rapid mash on the same SFX button stacks instances of the same sound on the same frame, which sounds harsh. Gate every play through a per-key timestamp.

:::interactive_editor

```js
function create() {
  this.lastPlayed = new Map();

  this.playSfx = (key) => {
    const now = this.time.now;
    const last = this.lastPlayed.get(key) ?? -Infinity;
    if (now - last < 50) return;
    this.lastPlayed.set(key, now);
    this.sound.play(key, { volume: this.sfxVolume });
  };
}
```

:::

50ms is the sweet spot for "human can't tell if a tap registered twice". Tune up to 80-100ms for percussive SFX, down to 20-30ms for chimes and pings where layering is fine.

## The six-stack juice recipe

Game feel on a hit comes from layering — no single ingredient is the magic. The recipe:

1. **Audio cue** (`sfx-hit` at hit time) — fastest signal to the brain.
2. **Squash & stretch** (`tweens.add` with `scaleX/scaleY`, yoyo) — proves the impact has body.
3. **Tint flash** (`setTint(0xffffff)` then `clearTint` on a delayed call) — the briefest "I got hit" tell.
4. **Camera shake** (`cameras.main.shake(80, 0.005)`) — moves the world, not just the actor.
5. **Hit-pause** (`physics.world.pause()` + `time.delayedCall(40, () => physics.world.resume())`) — buys 40ms of "stop" so the player's brain catches up.
6. **Particles** (`add.particles().explode(20, x, y)`) — the visual exclamation point.

:::interactive_editor

```js
function onHit(enemy) {
  if (!enemy.active) return;

  this.sound.play("sfx-hit", { volume: 0.5 });
  this.cameras.main.shake(80, 0.005);
  this.physics.world.pause();
  this.time.delayedCall(40, () => this.physics.world.resume());
  this.fx.explode(20, enemy.x, enemy.y);

  enemy.setFillStyle(0xffffff);
  this.time.delayedCall(80, () => {
    if (enemy.active) enemy.setFillStyle(0xff5252);
  });
  this.tweens.add({
    targets: enemy,
    scaleX: 1.3,
    scaleY: 0.7,
    duration: 60,
    yoyo: true,
    ease: "Power2"
  });
}
```

:::

Order matters less than presence — try removing any single ingredient and you'll feel the loss. The audio cue and the shake are the two with the most punch per millisecond.

## Hit-pause without freezing tweens

Hit-pause is the most under-used juice ingredient because devs don't realise it's just `physics.world.pause()`. Crucially, **pausing physics doesn't pause tweens or time events** — they live in different managers. So the squash tween keeps animating during the pause and the resume timer still fires.

:::interactive_editor

```js
function freezeFrame(scene) {
  scene.physics.world.pause();
  scene.time.delayedCall(40, () => scene.physics.world.resume());
}
```

:::

40ms is the magic number. Below 20ms the brain misses it; above 80ms the game starts feeling laggy. Tune by impact magnitude — heavy hits earn a longer pause (60-100ms), light hits get 20-30ms.

## Particle emitters in Phaser 3.60+

Phaser 3.60 introduced a single `add.particles(x, y, key, config)` factory that returns an `Emitter` directly. The emitter ships its own `explode(count, x, y)` for one-shot bursts and an `emitting: true/false` flag for continuous streams.

:::interactive_editor

```js
function create() {
  const g = this.add.graphics();
  g.fillStyle(0xffffff, 1);
  g.fillCircle(4, 4, 4);
  g.generateTexture("particle", 8, 8);
  g.destroy();

  this.fx = this.add.particles(0, 0, "particle", {
    speed: { min: -200, max: 200 },
    lifespan: 500,
    scale: { start: 1, end: 0 },
    gravityY: 200,
    emitting: false
  });

  this.input.on("pointerdown", (p) => this.fx.explode(20, p.x, p.y));
}
```

:::

`Graphics.generateTexture` builds a texture from a runtime drawing — handy for prototypes when you don't have a particle sprite. For final assets, use a real PNG.

## Emitter properties: speed, lifespan, scale, rotate, alpha

Most emitter properties accept three forms:

* A plain number: `lifespan: 500`.
* A range: `speed: { min: -200, max: 200 }`.
* A tween: `scale: { start: 1, end: 0 }` — interpolates over the particle's lifetime.

`angle` shapes the spread cone (`{ min: -45, max: 45 }` for a 90° fan). `gravityY` pulls particles downward. `rotate` spins them around their centre. `alpha` fades them; combined with `scale: { end: 0 }` you get a graceful death.

:::interactive_editor

```js
function create() {
  const g = this.add.graphics();
  g.fillStyle(0xffffff, 1);
  g.fillCircle(4, 4, 4);
  g.generateTexture("particle", 8, 8);
  g.destroy();

  this.fountain = this.add.particles(400, 500, "particle", {
    speed: { min: 200, max: 350 },
    angle: { min: -100, max: -80 },
    gravityY: 600,
    lifespan: 1500,
    scale: { start: 0.8, end: 0 },
    alpha: { start: 1, end: 0 },
    rotate: { min: 0, max: 360 }
  });
}
```

:::

The pattern: `speed` + `angle` aim it, `gravityY` curves it, `lifespan` tells it when to die, `scale`/`alpha`/`rotate` pretty it up over time.

## Bursts vs continuous emitters

Two emitter modes:

* **Continuous**: emits over time at a configured `frequency` (ms between particles). `emitting: true` (default).
* **Burst**: only fires when you call `.explode(count, x, y)`. Set `emitting: false` so it stays idle.

:::interactive_editor

```js
function create() {
  const g = this.add.graphics();
  g.fillStyle(0xffffff, 1);
  g.fillCircle(2, 2, 2);
  g.generateTexture("dust", 4, 4);
  g.destroy();

  this.smoke = this.add.particles(400, 300, "dust", {
    speed: 30,
    lifespan: 1000,
    scale: { start: 0.5, end: 0 },
    frequency: 60
  });

  this.fx = this.add.particles(0, 0, "dust", {
    speed: { min: -200, max: 200 },
    lifespan: 400,
    scale: { start: 1, end: 0 },
    emitting: false
  });

  this.input.on("pointerdown", (p) => this.fx.explode(40, p.x, p.y));
}
```

:::

Burst emitters are the default for impact effects — pre-create one in `create()` and call `.explode(n, x, y)` from anywhere. Don't `add.particles().explode(...)` on every hit; you'll leak emitters.

## Restraint: scaling juice by impact magnitude

The trap with juice is "more is better". It isn't — players adapt to whatever the *baseline* is. Scale every magnitude by an `impact` factor so heavy hits feel different from light ones:

:::interactive_editor

```js
function onHit(enemy) {
  const isHeavy = this.heavyMode;
  const isCrit = Math.random() < 0.2;
  const impact = (isHeavy ? 2 : 1) * (isCrit ? 1.5 : 1);

  this.cameras.main.shake(80 * impact, 0.005 * impact);
  this.fx.explode(20 * impact, enemy.x, enemy.y);
  if (isHeavy) {
    this.time.timeScale = 0.4;
    this.time.delayedCall(120, () => {
      this.time.timeScale = 1;
    });
  }
}
```

:::

Slow-mo (`time.timeScale = 0.4`) is the most expensive juice ingredient — reserve it for once-per-encounter moments like a heavy crit or a boss death. Sprinkling slow-mo on every hit makes the game feel like wading through syrup.

## Honoring `prefers-reduced-motion`

`window.matchMedia("(prefers-reduced-motion: reduce)").matches` exposes the OS-level accessibility flag. When set, scale every juice magnitude down (we use `0.2` in the workshop). Suppress slow-mo entirely. Particles stay (they're informational, not motion-heavy in tiny counts).

:::interactive_editor

```js
function create() {
  this.juice = window.matchMedia(
    "(prefers-reduced-motion: reduce)"
  ).matches
    ? 0.2
    : 1;
}

function onHit(enemy) {
  const impact = (this.heavyMode ? 2 : 1) * this.juice;
  this.cameras.main.shake(80 * impact, 0.005 * impact);
  this.fx.explode(Math.max(2, 20 * impact), enemy.x, enemy.y);

  if (this.heavyMode && this.juice >= 0.8) {
    this.time.timeScale = 0.4;
    this.time.delayedCall(120, () => {
      this.time.timeScale = 1;
    });
  }
}
```

:::

The mechanics still work; the visual aggression dials back. Players who didn't ask for accessibility get full juice; players who did get a game they can play without motion sickness.

## Cleanup on scene shutdown

Emitters, sounds, and tweens leak across scene restarts unless you tear them down. Hook the `SHUTDOWN` event:

:::interactive_editor

```js
function create() {
  this.events.on(Phaser.Scenes.Events.SHUTDOWN, () => {
    if (this.fx) this.fx.destroy();
    this.tweens.killAll();
    this.sound.stopAll();
  });
}
```

:::

`SHUTDOWN` fires when the scene stops. Without this, clicking "Play Again" on a Game Over screen leaves the old emitter dangling — drawing forever to a destroyed scene, leaking memory each restart.
