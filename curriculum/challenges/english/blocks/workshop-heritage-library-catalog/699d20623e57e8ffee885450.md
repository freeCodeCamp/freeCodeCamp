# --description--

The rest of the catalog cards have been added for you. Notice the three entries near the end with missing fields: one has a blank author, one has a blank year, and one has a blank location. Real-world data often has gaps like these, and you will handle them in a later step.

Right now, each book is a single long string. That makes it difficult to work with. How would you search by author, or sort by year?

The `parseCard` function will solve this. Its job is to take a raw catalog string and return a structured JavaScript object with four clear properties: `title`, `author`, `year`, and `location`. Once a card is parsed, you can access `book.author` or `book.year` directly. This structured format will make searching, grouping, and validating entries much easier.

Create a function named `parseCard` that takes a parameter named `rawString`.
