# --description--

Some catalog cards have missing fields. Entry 16 in `rawCatalogCards`, `"Unknown Title |  | 1975 | Shelf X1"`, has a blank where the author should be. Right now, `parseCard` returns an empty string for that field.

In JavaScript, certain values are considered *falsy*; they behave like `false` in a boolean context. The falsy values are `false`, `0`, `""` (empty string), `null`, `undefined`, and `NaN`. Everything else is truthy.

The logical OR operator (`||`) lets you provide a fallback: `a || b` evaluates to `a` when `a` is truthy, and to `b` otherwise. Since an empty string is falsy, `"" || "Unknown"` evaluates to `"Unknown"`. This matters for data quality; instead of letting empty strings silently propagate through your program, you replace them with a meaningful placeholder.

Update the `title`, `author`, and `location` properties in the return statement to use `|| "Unknown"` as a fallback. Leave `year` as-is for now; you will handle that in the next step.

To verify the fallback works, you can log `parseCard(rawCatalogCards[16])`. The `author` property should now show `"Unknown"` instead of an empty string.
