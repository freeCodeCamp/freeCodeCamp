## --quiz--

### --question--

#### --text--

Which Phaser API fires the same callback whether the user clicks with a mouse or taps with a finger?

#### --distractors--

`this.input.on("mousedown", callback)`

---

`this.input.keyboard.on("keydown", callback)`

---

`window.addEventListener("touchstart", callback)`

#### --answer--

`this.input.on("pointerdown", callback)`

### --question--

#### --distractors--

The browser window's top-left corner.

---

The center of the canvas.

---

The currently focused HTML element.

#### --answer--

The scene's coordinate system, already adjusted for the camera.

### --question--

#### --text--

What does `pointer.id` represent?

#### --distractors--

The DOM `id` attribute of the element under the pointer.

---

A counter that increments on every `pointerdown`.

---

The keycode of the most recent key pressed alongside the pointer.

#### --answer--

A stable id Phaser uses to track each individual finger across multi-touch events.

### --question--

#### --text--

---

Only the global listener — per-object handlers require `setInteractive(false)`.

---

The order is undefined and varies per browser.

#### --answer--

The global scene listener, then the per-object listener.

### --question--

#### --text--

What does `setInteractive()` do when you call it on a `Sprite`?

It enables a physics body on the sprite.

#### --answer--

It opts the sprite into the input pipeline so per-object pointer events are dispatched to it.

### --question--

### --question--

#### --text--

Why does the action-map pattern (`actions = { left: ['LEFT', 'A'] }`) scale better than checking `cursors.left.isDown` directly inside `update`?

#### --distractors--

It avoids the cost of allocating a new `cursors` object every frame.

---

It is required for Phaser's physics engine to recognize input.

---

It bypasses the browser's keyboard accessibility filter.

#### --answer--

It decouples gameplay from any specific device, so adding WASD, gamepad, or remappable keys does not require changing `update`.

### --question--

#### --text--

You want a game to ship with both arrow keys and WASD without an `if`-tree in `update`. What's the right shape of the input layer?

#### --distractors--

Two `update` bodies — one keyed on `cursors`, one keyed on `keys` — and a `cursors || keys` fallback.

---

Two scenes, one for arrow keys and one for WASD, swapped on a settings screen.

---

A `keydown` event listener that mutates a global `keyboardState` object every frame.

#### --answer--

One `ACTIONS` map listing every key name that should drive each action, plus an `isDown(action)` helper that resolves the list.

### --question--

#### --text--

#### --distractors--

A cookie set on every keystroke.

---

A `localStorage` entry written from `update` every frame.

---

A query-string parameter on the page URL.

#### --answer--

A single `localStorage` entry written when the player confirms the new mapping and read once on boot.

### --question--

#### --text--

Why is a virtual D-pad usually built as its own `Scene.launch`'d scene rather than as game objects added to `MainScene`?

#### --distractors--

Phaser does not allow rectangles to be added to `MainScene`.

---

Game objects in `MainScene` cannot be made interactive on touch.

---

`MainScene` cannot run while another scene is paused.

#### --answer--

It keeps the controls layer above gameplay, isolates pointer handlers from `MainScene`'s listeners, and lets `MainScene` restart without rebuilding the buttons.

### --question--

#### --text--

A `ControlsScene` button needs to tell `MainScene` "the player is pressing left." What's the canonical wiring?

#### --distractors--

Mutate a global `window.controls.left = true` flag.

---

Call `this.scene.start("MainScene")` on every `pointerdown`.

---

Use `localStorage.setItem("left", "true")` and have `MainScene` poll on each `update`.

#### --answer--

Emit a custom event on `MainScene`'s emitter via `this.scene.get("MainScene").events.emit("action:left")` and listen for it inside `MainScene.create`.
