# --interactive--

Once you have a second camera, the missing piece is *which* objects each camera renders. Phaser cameras default to **see everything** — every game object you create is drawn by every camera in the scene. That's almost never what you want.

The tool is `Camera#ignore`:

```js
this.cameras.main.ignore(hud);
```

`ignore` accepts a single game object, a `Group`, or an array. After this call, the main camera skips `hud` entirely on render. Anything you add to `hud` later is also ignored — `Group` membership is checked at render time.

The standard pattern is **two ignore lists, one per camera**:

```js
const world = this.add.container();
const ui = this.add.container();

world.add([this.player, this.platforms, this.enemies]);
ui.add([this.scoreText, this.heartIcons]);

this.uiCamera = this.cameras.add(0, 0, 480, 320);

this.cameras.main.ignore(ui);
this.uiCamera.ignore(world);
```

`Container` is convenient because every object you add to it inherits the ignore. You can also use a `Group` for the world and a `Group` for the UI; the API is the same. Whichever shape you pick, the rule is: **add objects to a parent, then ignore the parent on the wrong camera.**

A subtle gotcha: `ignore` is the *opposite* of how you normally think. You don't tell a camera what to *show* — you tell it what to *hide*. The result is the same, but the mental model is inverted from layer-based engines like Unity. If you need to add a new HUD element three steps into the game, just add it to the UI container. You don't need to update the cameras — they already trust the container.

:::interactive_editor

```js
const world = ["player", "platform", "enemy"];
const ui = ["score", "heart", "pause"];

function renderableBy(camera, allObjects, ignored) {
  return allObjects.filter(o => !ignored.includes(o));
}

const all = [...world, ...ui];
console.log("main camera renders:", renderableBy("main", all, ui));
console.log("ui camera renders:", renderableBy("ui", all, world));
```

:::

Ignoring works at the camera level, not the scene. If you have *three* cameras (main + minimap + UI), each one has its own ignore list. A common pattern with a minimap is:

```js
this.minimap.ignore([this.uiContainer, this.particleEmitter, this.player]);
```

The minimap shows the world but skips the UI, the particle effects, and (sometimes) the player itself — replacing the player with a brighter marker drawn only for the minimap. That's how Module 1's camera playground works.

Common bug: you `ignore(group)` early in `create()` but later add objects directly to the scene with `this.add.something(...)` — those objects bypass the group and are seen by both cameras. Either always add to the container/group, or maintain an explicit ignore list and update it when you add new objects.

# --questions--

## --text--

What does `this.cameras.main.ignore(hud)` do?

## --answers--

The main camera skips the `hud` object (or every member of the `hud` group/container) on render.

---

Disables input on the HUD.

### --feedback--

`ignore` is about rendering, not input. Use `setInteractive(false)` for input.

---

Hides the HUD from the entire scene.

### --feedback--

It only hides the HUD from the *main* camera. Other cameras still render it.

---

Removes the HUD from the scene entirely.

### --feedback--

That would be `hud.destroy()`. `ignore` only suppresses rendering for one camera.

## --text--

You add `this.scoreText` to a UI container, set the main camera to ignore the UI container, and ignore the world container on the UI camera. Then you add `this.healthBar = this.add.image(...)` directly to the scene (no container). What happens at render time?

## --answers--

Both cameras render the health bar — it bypassed both ignore lists.

---

Only the UI camera renders it.

### --feedback--

The UI camera ignores the world container, not "everything except the UI container." A direct add is not in either container.

---

Only the main camera renders it.

### --feedback--

Same reason — neither ignore list excludes a directly-added object.

---

Phaser throws an error.

### --feedback--

It does not throw. The bug is silent: the object is rendered twice.

## --text--

What's the difference between `ignore` and `setVisible(false)`?

## --answers--

`ignore` is per-camera (the object stays visible on other cameras); `setVisible(false)` hides the object on every camera in the scene.

---

There is no difference; they're aliases.

### --feedback--

They're different. One is camera-scoped; the other is object-scoped.

---

`ignore` removes the object from physics; `setVisible(false)` does not.

### --feedback--

`ignore` only affects rendering. Physics state is untouched by both.

---

`setVisible(false)` is faster because Phaser skips the geometry; `ignore` is slower.

### --feedback--

Both skip rendering work. Performance is comparable; the difference is *scope*.
