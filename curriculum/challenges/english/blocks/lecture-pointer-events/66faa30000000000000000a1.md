# --interactive--

The web has spent decades layering input APIs on top of each other: `mousedown`, `touchstart`, `MSPointerDown`. Phaser papers over all of it with one **pointer** API. The same listener handles a mouse click on desktop, a finger tap on mobile, and a stylus press on a tablet ‚Äî without you writing any branching logic.

The core events live on `this.input` inside any scene:

```js
this.input.on("pointerdown", pointer => {
  console.log("pressed at", pointer.x, pointer.y);
});

this.input.on("pointermove", pointer => {
  this.cursor.setPosition(pointer.x, pointer.y);
});

this.input.on("pointerup", pointer => {
  console.log("released");
});
```

Every callback receives a `Pointer` object with the fields you actually need:

- `pointer.x`, `pointer.y` ‚Äî current position in **scene coordinates** (already adjusted for the camera; you do not have to convert from page coordinates).
- `pointer.isDown` ‚Äî `true` while a button or finger is held.
- `pointer.button` ‚Äî which mouse button was used (`0` left, `1` middle, `2` right). On touch this is always `0`.
- `pointer.id` ‚Äî a stable id Phaser uses to track multi-touch fingers.

The single most useful pattern is "follow the pointer." Inside `create`, attach the listener; inside the listener, drive a game object's position:

```js
create() {
  this.cursor = this.add.circle(0, 0, 8, 0xffffff);
  this.input.on("pointermove", pointer => {
    this.cursor.setPosition(pointer.x, pointer.y);
  });
}
```

That's the whole "draw with your cursor" demo. Replace `setPosition` with `setVelocity` and you've built a paddle that chases your finger. You'll do exactly that in the workshop.

There are two listeners worth knowing besides the obvious three:

- `pointerdownoutside` / `pointerupoutside` ‚Äî fires when the pointer event happens outside the canvas. Useful if you have HTML UI overlapping your game.
- `gameout` / `gameover` ‚Äî fires when the cursor enters/leaves the game canvas itself.

A common trap: registering a global `pointerdown` listener and a per-object `pointerdown` listener at the same time. Both fire when the user clicks the object, in that order ‚Äî the global one first. If you only want the per-object one, check `event.stopPropagation` or rely on the per-object handler that the next lecture covers.

# --questions--

## --text--

Which Phaser API fires the same callback whether the user clicks with a mouse or taps with a finger?

## --answers--

`this.input.on("pointerdown", callback)`

---

`this.input.on("mousedown", callback)`

### --feedback--

Phaser exposes `pointer*` events specifically so you do not have to branch on input device. `mousedown` is the raw DOM name and is not the public Phaser API.

---

`this.input.keyboard.on("keydown", callback)`

### --feedback--

That's the keyboard plugin. It does not fire on a mouse click or a touch.

---

`window.addEventListener("touchstart", callback)`

### --feedback--

That bypasses Phaser's input pipeline, leaks listeners between scenes, and only handles touch.

## --text--

Inside a `pointermove` callback, where does `pointer.x` measure from?

## --answers--

The scene's coordinate system, already adjusted for the camera.

---

The browser window's top-left corner.

### --feedback--

`window` coordinates would force you to subtract the canvas offset by hand. Phaser does that for you.

---

The center of the canvas.

### --feedback--

The origin is the canvas's top-left, not the center.

---

The currently focused HTML element.

### --feedback--

Phaser has nothing to do with HTML focus.
