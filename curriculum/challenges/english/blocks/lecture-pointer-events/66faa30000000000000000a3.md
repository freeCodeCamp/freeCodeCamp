# --interactive--

By default Phaser tracks two pointers: one mouse cursor and one touch finger. That's enough for almost every desktop game. The moment you ship to mobile and want a "left thumb steers, right thumb fires" control scheme, you need more.

The opt-in is one line:

```js
create() {
  this.input.addPointer(2);
}
```

`addPointer(n)` adds *n* additional pointer slots on top of the default. After this call, four pointers exist in total: the mouse and three touch slots. You access them by index:

```js
update() {
  const left = this.input.pointer1;
  const right = this.input.pointer2;

  if (left.isDown && left.x < 240) {
    this.player.setVelocityX(-160);
  }
  if (right.isDown && right.x >= 240) {
    this.fireBullet();
  }
}
```

`pointer1` is the first finger that touched the screen, `pointer2` the second, and so on. They keep their identity across `pointermove` events — finger 1 stays finger 1 until you release it. That is what lets you do "thumbstick on the left, button on the right" without crossed wires.

Three useful properties on each pointer:

- `pointer.isDown` — currently pressed.
- `pointer.x`, `pointer.y` — current position (only valid while down or just-released).
- `pointer.justDown`, `pointer.justUp` — true for exactly one frame after the state changed.

The default mouse pointer is always at `this.input.activePointer`, which Phaser updates to whichever pointer is currently most relevant — useful for "show the most recent touch as a HUD cursor."

A common pitfall: forgetting to call `addPointer` before listening to `pointer3` and silently getting `undefined`. The other pitfall is the opposite: requesting too many pointers (anything above 4 is unusual) and burning per-frame work hit-testing slots that will never be touched.

For pinch and two-finger gestures, you compare the positions of two pointers each frame:

```js
if (this.input.pointer1.isDown && this.input.pointer2.isDown) {
  const dx = this.input.pointer1.x - this.input.pointer2.x;
  const dy = this.input.pointer1.y - this.input.pointer2.y;
  const distance = Math.sqrt(dx * dx + dy * dy);
}
```

Track `distance` over time and you have pinch-zoom — the basis of every map and image-viewer touch control.

# --questions--

## --text--

How many pointers does Phaser track by default in a fresh scene?

## --answers--

2 — one mouse cursor and one touch finger.

---

1 — only the mouse cursor.

### --feedback--

Phaser tracks both the mouse and one touch slot out of the box.

---

10 — Phaser pre-allocates slots for every finger.

### --feedback--

That would be wasteful. Phaser starts at two and lets you opt in to more.

---

0 — pointer tracking is opt-in.

### --feedback--

Pointer tracking is on by default; only *additional* slots are opt-in.

## --text--

You want to support a "left thumb steers, right thumb fires" two-finger control scheme on mobile. What's the first line of `create` you need?

## --answers--

`this.input.addPointer(1);`

---

`this.input.enableMultiTouch();`

### --feedback--

There is no `enableMultiTouch` method on the input plugin.

---

`navigator.maxTouchPoints = 2;`

### --feedback--

That's a read-only browser property and would not affect Phaser anyway.

---

`this.input.touch.start(2);`

### --feedback--

That's not a Phaser API. The opt-in is `addPointer`.

## --text--

After calling `this.input.addPointer(2)`, the user touches the screen with three fingers in quick succession. Which pointer object holds the third finger's position?

## --answers--

`this.input.pointer3`

---

`this.input.pointer2`

### --feedback--

`pointer2` holds the *second* finger; the third finger goes one slot further.

---

`this.input.pointer[2]`

### --feedback--

Phaser exposes pointers as named properties (`pointer1`, `pointer2`, …), not as an array index.

---

`this.input.activePointer`

### --feedback--

`activePointer` is whichever pointer Phaser is currently treating as primary, not specifically the third.
