# --interactive--

The Hugging Face `diffusers` library provides a clean, high-level API for running state-of-the-art diffusion models. In this lesson you will use it to generate images from text prompts, control generation parameters, and perform image-to-image translation — all with a few lines of code.

### Installing the Library

```bash
pip install diffusers transformers accelerate torch
```

Note: the `diffusers` library primarily supports PyTorch. A GPU is strongly recommended — generation on CPU is very slow.

### Text-to-Image Generation

```python
from diffusers import StableDiffusionPipeline
import torch

# Load the pipeline — downloads model weights on first run (~4GB)
pipe = StableDiffusionPipeline.from_pretrained(
    "runwayml/stable-diffusion-v1-5",
    torch_dtype=torch.float16   # use float16 for faster GPU inference
)
pipe = pipe.to("cuda")

# Generate an image
prompt = "a photorealistic tabby cat sitting on a windowsill at golden hour, bokeh background"

image = pipe(
    prompt,
    num_inference_steps=50,    # number of denoising steps
    guidance_scale=7.5,        # how strongly to follow the prompt
    height=512,
    width=512
).images[0]

image.save("generated_cat.png")
```

### Controlling Reproducibility with Seeds

Setting a random seed makes generation reproducible — the same seed and prompt always produce the same image:

```python
import torch

generator = torch.Generator("cuda").manual_seed(42)

image = pipe(
    prompt,
    num_inference_steps=50,
    guidance_scale=7.5,
    generator=generator
).images[0]
```

This is essential when iterating on prompts — changing only the prompt while keeping the seed constant lets you compare the effect of prompt changes directly.

### Negative Prompts

A negative prompt tells the model what to avoid in the generated image. This is one of the most effective ways to improve output quality:

```python
image = pipe(
    prompt          = "a photorealistic tabby cat on a windowsill",
    negative_prompt = "blurry, low quality, cartoon, deformed, ugly, watermark",
    num_inference_steps=50,
    guidance_scale=7.5
).images[0]
```

Common negative prompt terms: `blurry`, `low quality`, `deformed hands`, `extra fingers`, `watermark`, `text`, `cartoon`.

### Image-to-Image Generation

Starting from an existing image rather than pure noise — useful for style transfer, refinement, or variation generation:

```python
from diffusers import StableDiffusionImg2ImgPipeline
from PIL import Image

img2img = StableDiffusionImg2ImgPipeline.from_pretrained(
    "runwayml/stable-diffusion-v1-5",
    torch_dtype=torch.float16
).to("cuda")

# Load a starting image
init_image = Image.open("my_sketch.jpg").convert("RGB").resize((512, 512))

image = img2img(
    prompt     = "a detailed oil painting of a mountain landscape",
    image      = init_image,
    strength   = 0.75,    # 0=keep original, 1=ignore original completely
    guidance_scale = 7.5
).images[0]
```

The `strength` parameter controls how much the model deviates from the input image. At 0.75, the model starts at 75% of the way through the noising process, keeping the rough composition of the input.

### Choosing a Model

Thousands of fine-tuned models are available on the Hugging Face Hub at `huggingface.co/models`:

| Model | Best for |
|---|---|
| `runwayml/stable-diffusion-v1-5` | General purpose, fastest |
| `stabilityai/stable-diffusion-2-1` | Higher resolution (768px) |
| `stabilityai/stable-diffusion-xl-base-1.0` | Best quality, needs more VRAM |
| `prompthero/openjourney` | Midjourney-like artistic style |
| `hakurei/waifu-diffusion` | Anime-style illustration |

### Memory Optimisation

If you have limited GPU memory:

```python
# Enable attention slicing — trades speed for memory
pipe.enable_attention_slicing()

# Or use CPU offloading
pipe.enable_model_cpu_offload()
```

### Key Points

- The Hugging Face `diffusers` library provides high-level pipelines for text-to-image, image-to-image, and inpainting
- `num_inference_steps` controls quality vs speed — 20–50 steps is typical
- `guidance_scale` controls prompt adherence — 7.5 is a commonly used default
- Negative prompts tell the model what to avoid and can significantly improve output quality
- The `strength` parameter in image-to-image controls how much the model deviates from the input image

### --feedback--

Strength does not directly measure pixel similarity. It controls how far into the noising process the model starts, which affects how much the output can deviate from the input.

The model starts from 75% through the noising process, allowing significant deviation from the input while preserving rough composition

### --feedback--

Strength and guidance scale are independent parameters. Strength controls the starting point in the diffusion process relative to the input image.

The entire input image is used as conditioning. Strength controls the starting noise level, not which pixels are used.
