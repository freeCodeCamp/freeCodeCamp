# --interactive--

The neural network at the heart of most diffusion models is the U-Net. Originally designed for medical image segmentation in 2015, the U-Net is now the standard denoising backbone in systems like DDPM and Stable Diffusion. Its architecture is perfectly suited to the task of predicting noise at every pixel.

### Why U-Net for Denoising?

The denoising task requires a network that:

1. **Understands global context** — knowing that an image contains a face helps predict how to denoise the eye region
2. **Preserves fine spatial detail** — the noise prediction must be spatially precise, down to individual pixels
3. **Produces output the same size as its input** — the noise map has exactly the same dimensions as the noisy image

Standard convolutional networks progressively shrink spatial dimensions, losing fine detail. Fully connected networks ignore spatial structure entirely. The U-Net solves all three requirements with its distinctive encoder-decoder structure and skip connections.

### The U-Net Structure

The U-Net has three main parts:

**Encoder (downsampling path)** — a series of convolutional blocks that progressively reduce spatial dimensions while increasing the number of feature channels. Each block captures increasingly abstract, global features.

**Bottleneck** — the deepest part of the network, where the representation is most compressed. In modern diffusion U-Nets, this often includes self-attention layers to capture long-range dependencies.

**Decoder (upsampling path)** — a series of blocks that progressively restore spatial dimensions using transposed convolutions, mirroring the encoder.

**Skip connections** — at every resolution level, the encoder's feature maps are concatenated directly into the corresponding decoder layer. This gives the decoder access to fine-grained spatial information from the encoder.

```
Input (noisy image)
      ↓
[Encoder block 1] ──────────────────────────────→ [Decoder block 4]
      ↓                                                    ↑
[Encoder block 2] ──────────────────────────────→ [Decoder block 3]
      ↓                                                    ↑
[Encoder block 3] ──────────────────────────────→ [Decoder block 2]
      ↓                                                    ↑
      [Bottleneck (attention)]                   [Decoder block 1]
                                                           ↑
                                               Output (predicted noise)
```

### Timestep Embedding

The U-Net must know the current timestep `t` to perform appropriate denoising. The timestep is encoded using sinusoidal positional embeddings (similar to those in transformers) and injected into every residual block in the network:

```python
import tensorflow as tf
import numpy as np

def get_timestep_embedding(t, dim=256):
    """Sinusoidal embedding for timestep t"""
    half = dim // 2
    freqs = tf.exp(
        -np.log(10000) * tf.cast(tf.range(half), tf.float32) / half
    )
    args  = tf.cast(t, tf.float32)[:, None] * freqs[None]
    embed = tf.concat([tf.sin(args), tf.cos(args)], axis=-1)
    return embed  # shape: (batch_size, dim)
```

This embedding is added to the feature maps inside each residual block, conditioning every layer on the current noise level.

### A Simplified U-Net Block

```python
def ResBlock(x, temb, filters):
    """Residual block with timestep conditioning"""
    h = tf.keras.layers.GroupNormalization(groups=8)(x)
    h = tf.keras.layers.Activation('swish')(h)
    h = tf.keras.layers.Conv2D(filters, 3, padding='same')(h)

    # Add timestep embedding
    temb_proj = tf.keras.layers.Dense(filters)(
        tf.keras.layers.Activation('swish')(temb)
    )
    h = h + temb_proj[:, None, None, :]  # broadcast over spatial dims

    h = tf.keras.layers.GroupNormalization(groups=8)(h)
    h = tf.keras.layers.Activation('swish')(h)
    h = tf.keras.layers.Conv2D(filters, 3, padding='same')(h)

    # Residual connection
    if x.shape[-1] != filters:
        x = tf.keras.layers.Conv2D(filters, 1)(x)
    return x + h
```

### Attention in the Bottleneck

Modern diffusion U-Nets add self-attention layers at the bottleneck (and sometimes at intermediate resolutions). This allows the network to model long-range relationships across the image — for example, ensuring that two sides of a generated face are consistent with each other.

### Key Points

- The U-Net is the standard neural network architecture for diffusion model noise prediction
- Its encoder-decoder structure with skip connections captures both global context and fine spatial detail
- Skip connections concatenate encoder feature maps into the decoder at every resolution, preserving spatial information
- Timestep embeddings condition every residual block on the current noise level
- Self-attention at the bottleneck enables long-range spatial consistency in generated images

What problem do skip connections in the U-Net solve?

## --answers--

They allow the network to process multiple images simultaneously in a single forward pass

### --feedback--

Batch processing is handled by the batch dimension, not skip connections. Skip connections serve a different purpose related to information flow within the network.

---

They prevent the encoder from updating its weights during training

### --feedback--

Skip connections do not freeze weights — all parts of the U-Net are trained normally. They are about information flow, not gradient blocking.

---

They provide the decoder with direct access to fine-grained spatial information from the encoder, preventing loss of detail

---

They replace the need for batch normalisation in the decoder layers

### --feedback--

Skip connections and normalisation are independent components. Skip connections carry spatial information from encoder to decoder; they do not replace the normalisation function.

## --text--

How is the timestep `t` incorporated into the U-Net?

## --answers--

It is concatenated to the input image as an extra channel before the first layer

### --feedback--

Appending `t` as a pixel channel would be a very coarse signal that gets processed like image data. The U-Net uses a more sophisticated approach to distribute timestep information throughout the network.

---

It is encoded as a sinusoidal embedding and added to feature maps inside every residual block

---

It is used only in the bottleneck layer and not in the encoder or decoder

### --feedback--

Limiting timestep information to the bottleneck would mean the encoder and decoder layers cannot adapt their behaviour to the noise level. The embedding is injected into every residual block.

---

It sets the number of downsampling steps in the encoder to match the current timestep

### --feedback--

The U-Net architecture is fixed — the number of encoder blocks does not change with `t`. The timestep is incorporated through learned embeddings, not architectural changes.

## --text--

Why is self-attention added at the U-Net bottleneck in modern diffusion models?

## --answers--

To allow the network to process the timestep embedding more efficiently

### --feedback--

Timestep embeddings are incorporated through residual blocks, not through attention. The bottleneck attention serves a different purpose related to spatial relationships in the image.

---

To reduce the number of parameters in the bottleneck layer

### --feedback--

Self-attention adds parameters, not reduces them. It is included for its ability to capture a specific type of relationship in the compressed representation.

---

To enable the network to model long-range spatial relationships, ensuring consistency across distant parts of the image

---

To compress the image representation into a single vector for the decoder

### --feedback--

Compressing to a single vector is what a standard bottleneck does in an autoencoder. Self-attention maintains spatial structure — it operates on a sequence of spatial feature vectors, not a single compressed vector.
