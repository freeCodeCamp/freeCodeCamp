# --interactive--

An unconditional diffusion model generates random images from the learned data distribution. Text-to-image models add a conditioning signal — a text prompt — that steers the generation toward a specific description. This lesson explains how text conditioning works and how Classifier-Free Guidance (CFG) makes it much more effective.

### How Text Conditioning Works

The text prompt is converted into a sequence of embedding vectors using a pre-trained text encoder. In Stable Diffusion, this is a CLIP text encoder — the same model that can measure similarity between images and text descriptions.

```python
from transformers import CLIPTokenizer, TFCLIPTextModel

tokenizer    = CLIPTokenizer.from_pretrained("openai/clip-vit-large-patch14")
text_encoder = TFCLIPTextModel.from_pretrained("openai/clip-vit-large-patch14")

prompt  = "a photorealistic cat sitting on a window sill at sunset"
tokens  = tokenizer(prompt, return_tensors='tf', padding='max_length',
                    max_length=77, truncation=True)
text_embeddings = text_encoder(tokens).last_hidden_state
# Shape: (1, 77, 768) — 77 tokens, each a 768-dimensional vector
```

These text embeddings are fed into the U-Net via **cross-attention layers**. At each cross-attention layer, the spatial feature maps of the U-Net act as queries, and the text embeddings act as keys and values. This allows every spatial location in the denoising process to attend to the most relevant parts of the text description.

```
U-Net spatial features (queries) ←→ Text embeddings (keys & values)
        → weighted combination conditioned on text
```

### Why Cross-Attention for Text Conditioning?

Cross-attention is the natural mechanism for conditioning on a sequence of varying length. Unlike class labels (a single integer), a text prompt is a sequence of up to 77 tokens with complex semantic relationships. Cross-attention allows the U-Net to selectively attend to the most relevant tokens for each spatial region — "cat" for the left side of the image, "sunset" for the background colours.

### Classifier-Free Guidance (CFG)

Text conditioning alone produces images that match the prompt but may not match it strongly — the model hedges toward the most common outputs. Classifier-Free Guidance amplifies the influence of the text prompt to produce sharper, more prompt-adherent images.

The idea: during training, the text conditioning is randomly dropped 10–20% of the time (replaced with an empty embedding). This trains the model to generate both conditional outputs (given a prompt) and unconditional outputs (no prompt).

At inference, the model runs two forward passes per step:
1. Conditioned on the text prompt: `noise_pred_cond`
2. Unconditioned (empty prompt): `noise_pred_uncond`

The final noise prediction is:

```
noise_pred = noise_pred_uncond + guidance_scale × (noise_pred_cond - noise_pred_uncond)
```

```python
guidance_scale = 7.5   # typical value — higher = more prompt-adherent

noise_pred_uncond = unet(latent, t, text_embeddings=empty_embeddings)
noise_pred_cond   = unet(latent, t, text_embeddings=text_embeddings)

noise_pred = noise_pred_uncond + guidance_scale * (noise_pred_cond - noise_pred_uncond)
```

### Understanding the Guidance Scale

The guidance scale is a slider between "follows the learned data distribution" and "follows the text prompt literally":

- **guidance_scale = 1.0** — standard conditional generation, no extra guidance
- **guidance_scale = 7.5** — balanced, used in most Stable Diffusion defaults
- **guidance_scale = 15+** — very strong prompt adherence, but images may become over-saturated or artefact-prone

### Key Points

- Text prompts are encoded into embeddings using a pre-trained CLIP text encoder
- The U-Net receives text embeddings through cross-attention layers, allowing spatial features to attend to relevant text tokens
- Classifier-Free Guidance runs two forward passes — conditioned and unconditioned — and amplifies the difference
- The guidance scale controls how strongly the output follows the text prompt
- Higher guidance scales produce more prompt-adherent but potentially over-saturated images

How does the U-Net incorporate the text prompt during the denoising process?

## --answers--

The text is converted to a single number that is added to the timestep embedding

### --feedback--

Reducing a rich text description to a single number would discard almost all semantic information. The U-Net incorporates text through a mechanism that preserves its sequence structure.

---

The text embedding is concatenated to the noisy image as extra channels before the first U-Net layer

### --feedback--

Text has a variable length sequence structure that cannot be simply concatenated as image channels. The U-Net uses a different mechanism specifically designed for sequence-to-spatial conditioning.

---

The spatial feature maps attend to the text embeddings via cross-attention layers at multiple points in the U-Net

---

The text is used only to select which pre-trained U-Net weights to load from a model library

### --feedback--

The U-Net weights are fixed after training — the text does not select different weights. Text conditioning happens dynamically through attention during each forward pass.

What is the purpose of randomly dropping the text conditioning during training in Classifier-Free Guidance?

## --answers--

To save GPU memory by avoiding the text encoder forward pass on some batches

### --feedback--

The motivation is not computational efficiency. The random dropping of conditioning is an intentional training strategy that enables a specific capability at inference time.

---

To train the model to generate both conditional (text-guided) and unconditional outputs, enabling guidance at inference

---

To prevent the model from overfitting to specific text prompts in the training data

### --feedback--

Overfitting prevention is a separate concern. The specific reason for random conditioning dropout is to enable a two-pass inference technique.

---

To reduce the maximum prompt length the model needs to handle

### --feedback--

The maximum prompt length is fixed by the tokenizer configuration (77 tokens for CLIP), not by the dropout rate. Random conditioning dropout serves a different purpose.

## --text--

What happens when the guidance scale is set very high (e.g., 20+) in Classifier-Free Guidance?

## --answers--

The model ignores the text prompt entirely and generates random images

### --feedback--

Ignoring the text prompt would be the effect of a guidance scale of 0 or negative. A very high scale has the opposite effect.

---

Generation becomes faster because fewer denoising steps are needed

### --feedback--

The guidance scale affects the direction of denoising, not the number of steps. Generation speed is controlled by the number of sampling steps, not the guidance scale.

---

The output closely follows the text prompt but may become over-saturated or develop artefacts

---

The model switches from latent space to pixel space diffusion automatically

### --feedback--

The diffusion space (latent or pixel) is fixed by the model architecture. The guidance scale only affects how the noise predictions from conditioned and unconditioned passes are combined.
