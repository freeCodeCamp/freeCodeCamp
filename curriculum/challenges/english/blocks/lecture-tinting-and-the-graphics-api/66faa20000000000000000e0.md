# --interactive--

The features stack: sprite for the art, tint for the recolor, graphics for the runtime ring, blend mode for the glow, depth for the layering, tween for the motion. Stitching them together is what makes a polished effect.

Take a "shake-on-hit" pattern that pieces together everything from this module:

```js
function flashOnHit(scene, target) {
  target.setTintFill(0xffffff);

  const ring = scene.add.graphics();
  ring.setBlendMode(Phaser.BlendModes.ADD);
  ring.setDepth(50);
  ring.lineStyle(3, 0xffeeaa, 1);
  ring.strokeCircle(target.x, target.y, 28);

  scene.tweens.add({
    targets: ring,
    alpha: 0,
    scale: 1.6,
    duration: 200,
    onComplete: () => ring.destroy()
  });

  scene.time.delayedCall(80, () => target.clearTint());
  scene.cameras.main.shake(80, 0.005);
}
```

What's happening:

1. **Sprite tint flash.** `setTintFill(0xffffff)` makes the target read as a white silhouette for the duration of the hit-stop.
2. **Graphics shock-ring.** A new `Graphics` is added at the foreground depth (`50`), with `ADD` blend so it lays light on top of the scene — a glowy halo, not a sticker.
3. **Tween out.** The ring fades and scales up over 200 ms, then destroys itself in `onComplete`. The destroy on complete prevents the scene from leaking graphics objects on every hit.
4. **Tint reset.** A delayed callback clears the tint after 80 ms — short enough that the white silhouette feels like impact rather than a freeze.
5. **Camera shake.** A short, gentle shake — `0.005` is a few pixels — that registers physical weight without making the player nauseous.

Five techniques, five lines each, one cohesive moment. Each one alone is a draft. Stacked together, they're a hit.

A few rules that keep stacked effects readable:

- **Allocate when you fire, free in `onComplete`.** Effect graphics are short-lived. Create them when the effect starts; destroy them when the tween finishes. The scene's graphics-object count should return to baseline between hits.
- **Tune the durations.** Hit-stop should be shorter than the ring fade. Camera shake should be the longest of all so the visual rest of the effect plays inside it. Roughly: 60–100 ms for tint, 150–250 ms for ring, 80–150 ms for shake — all under a third of a second.
- **Test against the gameplay tempo.** A great hit effect for a slow turn-based game looks like seizure-bait in an arcade brawler. Match the effect to how often it fires.

You'll wire most of this stack — tint, graphics ring, blend mode, tween, depth — in the next workshop.

## --text--

In the `flashOnHit` pattern, why does the ring `Graphics` get destroyed in the tween's `onComplete` callback?

## --answers--

So scenes don't leak `Graphics` objects every time a hit fires; the count returns to baseline between hits.

---

Because Phaser garbage-collects unused tweens, but not their targets.

### --feedback--

Phaser does not auto-destroy tween targets; you have to do it yourself.

---

So the ring restarts cleanly the next time the function is called.

### --feedback--

Even without destroy, a fresh call would create a new ring; the leak is the real problem.

---

Because `Graphics` objects can't be moved between scenes once tweened.

### --feedback--

That's not a real Phaser constraint.

## --text--

You stack tint flash + shock ring + camera shake on every enemy hit. The result feels overwhelming. Which adjustment is the right first lever to pull?

## --answers--

Shorten each effect's duration so the whole stack fires inside ~250 ms instead of ~800 ms.

---

Remove the tint flash entirely — three effects is always too many.

### --feedback--

The number isn't the problem; the duration is. Three short effects feel "punchy"; three long ones feel "long."

---

Switch from `BlendModes.ADD` to `BlendModes.NORMAL`.

### --feedback--

That removes the glow but doesn't fix the timing — the effect still feels long.

---

Reduce the ring's depth from 50 to 0.

### --feedback--

Depth changes the layering, not how long the effect lasts.

## --text--

Why is `setTintFill` (rather than `setTint`) the right call for a hit-flash silhouette?

## --answers--

`setTintFill` replaces every pixel with the tint color. The result is a flat silhouette — exactly what a hit-flash wants.

---

`setTintFill` is the only one that works in WebGL.

### --feedback--

Both work in WebGL; the difference is multiplicative tint vs. silhouette.

---

`setTintFill` runs faster than `setTint`.

### --feedback--

Performance is comparable; the difference is visual.

---

`setTint` is deprecated.

### --feedback--

`setTint` is current and the right call for color modulation; it just isn't the right call for a silhouette.
