# --description--

Node.js does not have a built-in WebSocket server. Developers typically use external packages such as `ws` to implement the WebSocket protocol and handle real-time communication between clients and servers.

The `ws` library is lightweight, fast, and easy to use. It implements the WebSocket protocol so you can create and manage WebSocket connections.

You can install it with npm:

```bash
npm install ws
```

Once installed, create a WebSocket server by importing the `ws` library, invoking its `Server` method, and passing in a port:

```js
const WebSocket = require("ws");

const server = new WebSocket.Server({
  port: 3000,
});
```

This starts a WebSocket server that listens for connections on port `3000`.

When a client connects, the server emits a `connection` event.

```js
server.on("connection", (socket) => {
  console.log("Client connected");
});
```

Think of this event as the server's way of saying "a new client has joined".

The `socket` parameter represents the connection between the server and that client. You use it to send and receive messages.

When a connected client sends data, the server emits a `message` event.

```js
server.on("connection", (socket) => {
  socket.on("message", (message) => {
    console.log(message.toString());
  });
});
```

In this example, the server listens for incoming messages and logs them to the console.

You can send data back to a client using the `send()` method:

```js
socket.send("Hello from the server");
```

This sends a message through the open WebSocket connection.

As long as the connection remains open, both the client and server can continue exchanging messages in real time.

Apart from `connection` and `message`, WebSocket connections also emit the `close` and `error` events.

`close` is triggered when a connection closes:

```js
socket.on("close", () => {
  console.log("Disconnected");
});
```

`error` is triggered when something goes wrong:

```js
socket.on("error", (error) => {
  console.error(error);
});
```

Together, these events form the foundation of most WebSocket applications.

A common requirement in chat applications is broadcasting. Broadcasting means sending a message to multiple connected clients instead of just one.

Imagine a chat room with three users named **John Doe**, **Jane Doe**, and **Wes Smith**.

If John Doe sends a message, the server may need to forward that message to Jane Doe and Wes Smith.

To do this, the server keeps track of connected clients and loops through them when a message arrives.

Before sending a message, the server checks whether the client connection is still open:

```js
if (client.readyState === WebSocket.OPEN) {
  // do something
}
```

Every WebSocket connection has a `readyState` property that describes its current state.

`WebSocket.OPEN` means the connection is active and ready to send and receive data.

By checking for this, the server avoids sending messages to clients that have already disconnected or whose connections are still being established:

```js
server.on("connection", (socket) => {
  socket.on("message", (message) => {
    server.clients.forEach((client) => {
      if (client.readyState === WebSocket.OPEN) {
        client.send(message.toString());
      }
    });
  });
});
```

In this example, every connected client receives the incoming message. This is called broadcasting.

Many real-time applications use broadcasting to deliver data such as chat messages, live notifications, sports updates, and multiplayer game events.

With the `ws` library, it is uncomplicated to build WebSocket servers in Node.js.

All you need to do is create a server, listen for events, exchange messages, and broadcast updates to connected clients.

These building blocks form the foundation of many real-time applications.
