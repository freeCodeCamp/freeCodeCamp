# --interactive--

A real level is not a single grid — it is a **stack** of grids. The floor sits underneath the walls; the walls sit underneath the decorations; the player walks on top of the floor and bumps into the walls. A tilemap holds each of those grids as its own **layer**, and Tiled exports them as a list inside the same JSON file.

Three layers do almost everything you need:

- `floor` — what the player walks on. Decorative; never collidable.
- `walls` — solid geometry. The only layer with collision turned on.
- `decorations` — torches, rugs, banners. Sits above walls; never collidable.

```js
const map = {
  layers: [
    { name: "floor",       data: [/* gids */], collides: false },
    { name: "walls",       data: [/* gids */], collides: true },
    { name: "decorations", data: [/* gids */], collides: false }
  ]
};
```

The order of `layers[]` is the **paint order** — earlier layers render first, later layers render on top. A torch tile in `decorations` paints over the wall tile under it because `decorations` is later in the list. If you reorder the array, you reorder the paint stack.

:::interactive_editor

```js
function paintStack(layers) {
  const result = {};
  for (const layer of layers) {
    for (const cell of layer.cells) {
      const key = `${cell.x},${cell.y}`;
      result[key] = { gid: cell.gid, source: layer.name };
    }
  }
  return result;
}

const layers = [
  { name: "floor",       cells: [{ x: 1, y: 1, gid: 1 }] },
  { name: "walls",       cells: [{ x: 1, y: 1, gid: 13 }] },
  { name: "decorations", cells: [{ x: 1, y: 1, gid: 26 }] }
];

const final = paintStack(layers);
console.log(final["1,1"]);
```

:::

Collision lives on a layer, not on the tilemap as a whole. The `walls` layer flips `collides: true` on the tiles you mark as solid; the `floor` layer leaves it at `false`. When you call `physics.add.collider(player, walls)` later, only the wall layer pushes the player back — the floor layer does nothing physical, even though it is being painted underneath. That is exactly what you want: the player walks freely across `floor`, hits `walls`, and never collides with `decorations`.

There is one more thing layers buy you: **selective rendering**. You can hide the decorations layer for a debug view, or render only the floor for a minimap. Each layer is a `TilemapLayer` game object Phaser builds for you; toggling its `.visible` boolean is enough.

# --questions--

## --text--

Your `layers` array is `["walls", "floor", "decorations"]`. What does the player see?

## --answers--

The floor on top of the walls — paint order follows the array order, and `floor` comes after `walls`, so it covers the walls.

---

Walls on top of the floor — Tiled always paints walls last.

### --feedback--

Tiled has no opinion. Paint order is whatever order you put the layers in.

---

A z-fighting flicker because two grids share the same depth.

### --feedback--

Each layer has its own depth derived from its index in `layers[]`; they never share depth.

---

Nothing at all — the wrong order silently drops the layers.

### --feedback--

Layers always render. The order just decides what sits on top.

## --text--

You add `physics.add.collider(player, layers)` where `layers` is the entire `layers` array. What happens?

## --answers--

It will not work — `physics.add.collider` expects a single layer or group, not the whole list. You collide the player against `walls` specifically.

---

The player collides with every tile in every layer, including the floor.

### --feedback--

Even if the API accepted it, only tiles flagged collidable participate; floor tiles are not flagged.

---

The player collides only with the topmost layer.

### --feedback--

`physics.add.collider` would not pick a "top" — and that is not how layered collision works.

---

The player passes through everything because passing an array is a no-op.

### --feedback--

It is not a no-op; it is a type error. Pass the wall layer.

You want a minimap that shows only the floor and walls. What is the cleanest path?

## --answers--

Render the full tilemap on the main camera and toggle the decorations layer's `.visible = false` on a second camera that draws the minimap.

---

Re-export the JSON without the decorations layer.

### --feedback--

Asset re-exports for view modes are heavy; toggle visibility instead.

---

Loop through every cell of every layer at runtime and rebuild a new minimap layer from scratch.

### --feedback--

That works but is wildly more code than toggling a layer's visibility.

Use a CSS overlay to mask the decorations.

### --feedback--

CSS does not address WebGL pixels rendered into a canvas.
