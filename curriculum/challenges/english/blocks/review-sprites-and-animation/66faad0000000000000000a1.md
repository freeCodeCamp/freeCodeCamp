# --interactive--

## Sprite sheets and atlases

A sprite sheet is one image that holds many frames laid out on a fixed grid. The browser fetches one file, the GPU uploads one texture, and the engine slices out frame `N` by index. Phaser loads them with `this.load.spritesheet(key, url, { frameWidth, frameHeight })`.

:::interactive_editor

```js
function preload() {
  this.load.spritesheet(
    "hero",
    "/curriculum-assets/phaser/spritesheets/hero-walk.png",
    { frameWidth: 16, frameHeight: 16 }
  );
}
```

:::

When frames vary in size — a tall boss alongside short minions — reach for a **texture atlas** instead. The art lives in one PNG, but a JSON manifest tells the engine where each named frame sits. Phaser loads atlases with `this.load.atlas(key, imageUrl, jsonUrl)`.

## Animations

Loading the frames is half the job. Animations stitch them into a sequence and play that sequence back at a target frame rate. You define an animation once in `create` with `this.anims.create({ ... })`, then play it on any sprite that uses the same texture.

:::interactive_editor

```js
function create() {
  this.anims.create({
    key: "walk-down",
    frames: this.anims.generateFrameNumbers("hero", { start: 0, end: 3 }),
    frameRate: 8,
    repeat: -1
  });

  const player = this.add.sprite(120, 120, "hero", 0);
  player.anims.play("walk-down");
}
```

:::

`frameRate` is frames per second. `repeat: -1` loops forever; `repeat: 2` plays the cycle three times (once + 2 repeats). `yoyo: true` plays the cycle forward then backward in a single repeat. Switch animations on the fly by calling `sprite.anims.play("idle", true)` — passing `true` skips the restart if it's already playing that animation.

## Depth and blend modes

By default Phaser paints game objects in creation order. Last-added wins. Override that with `setDepth(n)`: higher numbers paint later (on top). The Chapter 2 layering recipe — `bg=0`, `mid=5`, `entities=10`, `foreground=50`, `hud=100`, `modal=200` — leaves room between layers so future content can slot in without renumbering.

:::interactive_editor

```js
function create() {
  const bg = this.add.rectangle(120, 90, 240, 180, 0x223344);
  const player = this.add.rectangle(120, 90, 16, 16, 0xffeeaa);
  const hud = this.add.text(8, 8, "HUD", { fill: "#ffffff" });

  bg.setDepth(0);
  player.setDepth(10);
  hud.setDepth(100);
}
```

:::

Blend modes change how a new pixel combines with what's already on screen. The default `NORMAL` over-paints; `ADD` brightens (great for glow rings and laser beams); `MULTIPLY` darkens (shadows, mood overlays); `SCREEN` lifts colour without clipping as harshly as `ADD`.

:::interactive_editor

```js
function create() {
  const ring = this.add.graphics();
  ring.setBlendMode(Phaser.BlendModes.ADD);
  ring.lineStyle(2, 0xffffff, 1);
  ring.strokeCircle(120, 90, 24);
}
```

:::

Blend modes only work in the WebGL renderer. The Canvas fallback silently uses `NORMAL` for everything.

## Tinting and Graphics

`setTint(color)` re-colours a sprite on the GPU by multiplying every pixel with the tint. Pass `0xffffff` for "no tint." Pass `0x000000` and the sprite goes black (every channel times zero). Use `setTintFill` to replace every pixel with a solid colour — the classic "got hit" silhouette flash. `clearTint()` puts the original art back.

:::interactive_editor

```js
function create() {
  const enemy = this.add.image(120, 90, "enemy");

  enemy.setTintFill(0xffffff);
  this.time.delayedCall(80, () => enemy.clearTint());
}
```

:::

Tinting works on textured objects (`Sprite`, `Image`, `TileSprite`). It does not apply to `Rectangle`, `Text`, or `Graphics` — those have their own colour APIs.

`Phaser.GameObjects.Graphics` is an immediate-mode canvas you draw into in code. `lineStyle` and `fillStyle` set the current pen and stay until you change them. `lineBetween`, `strokeRect`, `strokeCircle`, `strokeRoundedRect` cover most needs. To redraw every frame, call `g.clear()` first — otherwise the shapes pile up and the canvas turns into a smear.

:::interactive_editor

```js
function create() {
  this.ring = this.add.graphics();
  this.ring.lineStyle(2, 0xffffff, 1);
  this.ring.strokeCircle(0, 0, 24);
  this.ring.setPosition(120, 90);
}

function update() {
  this.ring.setPosition(120 + Math.sin(this.time.now / 200) * 30, 90);
}
```

:::

The pattern that scales: draw at local `(0, 0)` and move the whole graphics object with `setPosition`. No clear-and-redraw, no allocation, just a moving ring.
