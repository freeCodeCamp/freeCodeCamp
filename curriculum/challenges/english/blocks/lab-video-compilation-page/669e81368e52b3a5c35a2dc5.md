# --interactive--

Training a deep CNN from scratch requires millions of images and days of GPU time. Transfer learning lets you skip most of that work by starting from a model that has already learned powerful visual features on a massive dataset.

### The Idea

ImageNet is a dataset of over 14 million labelled images across 1,000 categories. Training a model on ImageNet for weeks produces filters that detect an enormous variety of visual features — edges, textures, shapes, animal parts, vehicle components, and much more.

The key insight of transfer learning: these learned features are not specific to ImageNet categories. A filter that detects "curved edge" or "fur texture" is useful for classifying cats, dogs, flowers, or skin lesions. You can reuse the convolutional base and replace only the final classification layers.

### Feature Extraction vs Fine-tuning

**Feature extraction** — freeze the pre-trained convolutional base (weights are not updated) and train only the new classification head on your data. Fast and works well even with small datasets.

**Fine-tuning** — after training the classification head, unfreeze some of the later convolutional layers and continue training the whole network at a very low learning rate. This adapts the high-level features to your specific task. Requires more data and is slower.

### Using MobileNetV2 for Feature Extraction

```python
import tensorflow as tf

# Load MobileNetV2 pre-trained on ImageNet, without the top classifier
base_model = tf.keras.applications.MobileNetV2(
    input_shape=(96, 96, 3),
    include_top=False,        # exclude the ImageNet classification head
    weights='imagenet'
)

# Freeze the base model — do not update its weights
base_model.trainable = False

# Build new classification head
model = tf.keras.Sequential([
    base_model,
    tf.keras.layers.GlobalAveragePooling2D(),
    tf.keras.layers.Dense(128, activation='relu'),
    tf.keras.layers.Dropout(0.3),
    tf.keras.layers.Dense(10, activation='softmax')
])

model.compile(optimizer='adam',
              loss='sparse_categorical_crossentropy',
              metrics=['accuracy'])
```

### Fine-tuning Example

After training the head for several epochs, unfreeze the top layers of the base model:

```python
# Unfreeze the top layers of the base model
base_model.trainable = True

# Fine-tune from this layer onwards
fine_tune_at = 100

for layer in base_model.layers[:fine_tune_at]:
    layer.trainable = False

# Recompile with a very low learning rate
model.compile(
    optimizer=tf.keras.optimizers.Adam(learning_rate=1e-5),
    loss='sparse_categorical_crossentropy',
    metrics=['accuracy']
)

model.fit(x_train, y_train, epochs=10, validation_split=0.1)
```

A very low learning rate (1e-5) is critical during fine-tuning — large updates would destroy the carefully learned ImageNet features.

### When to Use Transfer Learning

Transfer learning almost always outperforms training from scratch when:

- Your dataset has fewer than ~100,000 images
- Your images are similar in nature to natural photographs
- You have limited GPU resources or time

Training from scratch makes sense only when your data is very different from ImageNet (e.g., medical scans, satellite imagery, microscopy) and you have millions of domain-specific examples.

### Key Points

- Transfer learning reuses the convolutional base of a model trained on ImageNet
- Feature extraction freezes the base and trains only the new classification head
- Fine-tuning unfreezes later layers and trains the full network at a very low learning rate
- Transfer learning almost always outperforms training from scratch on small to medium datasets
- A very low learning rate is essential during fine-tuning to preserve learned features

This is reversed. Feature extraction trains only the new head (base is frozen); fine-tuning additionally trains some of the base model layers.
