# --interactive--

Arcade physics gives you two places to set gravity, and they do different things.

**World gravity** lives on the game config under `physics.arcade.gravity`. It is the constant downward (or upward) acceleration applied to every dynamic body in the scene. Set it once at boot and forget about it.

```js
const config = {
  type: Phaser.AUTO,
  width: 480,
  height: 360,
  physics: { default: "arcade", arcade: { gravity: { y: 600 } } }
};
```

**Body gravity** lives on each individual body via `body.setGravityX` and `body.setGravityY`. It is *added on top of* world gravity, per body. A balloon that should drift up while everything else falls just sets its own gravity to cancel and overshoot the world's:

:::interactive_editor

```js
function create() {
  const balloon = this.add.circle(120, 90, 8, 0xff88aa);
  this.physics.add.existing(balloon);
  balloon.body.setGravityY(-700);
}
```

:::

If world gravity is `+600` and the balloon's body gravity is `-700`, the net acceleration on the balloon is `-100` — a slow upward drift. Everything else in the scene still falls at `+600`.

You can also flip world gravity at runtime — `this.physics.world.gravity.y = -300` mid-game gives you "anti-gravity" pickups, ceiling-walking sequences, or boss arenas where the floor is the ceiling. Live-tuning world gravity is exactly what the bouncing-ball workshop does in the last few steps.

The trap to avoid: setting body gravity when you really wanted world gravity. If every entity in your scene shares the same gravity, set it once on the config. Per-body gravity is for the rare body that wants to *deviate* from the world's pull.

# --questions--

## --text--

Where do you set the gravity that should apply to every dynamic body in the scene?

## --answers--

In the game config under `physics.arcade.gravity`.

---

On every body individually with `body.setGravityY`.

### --feedback--

That works but it duplicates the same value across every body — the config is the single source of truth.

---

In the body's constructor by extending `Phaser.Physics.Arcade.Body`.

### --feedback--

You almost never extend `Body` — the config exposes everything you need.

---

By dispatching a custom event on `this.events`.

### --feedback--

Events do not configure the physics engine.

## --text--

World gravity is `+400` and a body's `setGravityY(-500)` is set on it. What is the net vertical acceleration on that body?

## --answers--

`-100` — the body's gravity is added on top of the world's.

---

`-500` — body gravity replaces world gravity.

### --feedback--

Body gravity is *added* to world gravity, not a replacement.

---

`+400` — `setGravityY` only affects horizontal motion.

### --feedback--

`setGravityY` is the vertical body gravity. `setGravityX` is horizontal.

---

`0` — opposite signs cancel out exactly.

### --feedback--

Only if the magnitudes match. `+400` and `-500` sum to `-100`.

## --text--

You want most enemies to fall normally, but a single "ghost" enemy to ignore gravity entirely. What's the cleanest way?

## --answers--

Leave world gravity alone and call `ghost.body.setAllowGravity(false)` (or `setGravityY(-worldGravity)`).

---

Set world gravity to `0` and add per-body gravity to every other enemy.

### --feedback--

That works but flips the default — every other enemy now needs custom code.

---

Move the ghost into a separate scene with no gravity.

### --feedback--

That's a sledgehammer for one entity.

---

Manually subtract `worldGravity * dt` from `ghost.body.velocity.y` every frame.

### --feedback--

You'd be re-implementing what `setAllowGravity(false)` already does in one line.
