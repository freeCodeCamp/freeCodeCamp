# --interactive--

By default, a Phaser camera sits at the world origin and never moves. A 4000-pixel-wide level with the player out at `x = 2000` is invisible — the camera is staring at the empty `(0, 0)` corner of the world.

The fix is one line:

```js
this.cameras.main.startFollow(this.player);
```

`startFollow` tells the camera to track that game object every frame. The camera centres itself on the target's position, and the rest of the world scrolls past on the canvas. As long as the target stays alive, the camera stays glued.

:::interactive_editor

```js
const camera = { x: 0, y: 0 };
const player = { x: 0, y: 0 };
let following = null;

function startFollow(target) {
  following = target;
}

function update() {
  if (following) {
    camera.x = following.x;
    camera.y = following.y;
  }
}

startFollow(player);
player.x = 2000;
update();
console.log(camera);
```

:::

The camera is now snapped exactly onto the player. That works, but it feels rigid — the camera teleports the moment the player teleports, which during fast movement looks like the world is jittering rather than the player moving. The next two lectures address that with two complementary tools: **lerp** for smoothing and **deadzones** for ignoring small movements.

There are a few `startFollow` knobs worth knowing today:

```js
this.cameras.main.startFollow(target, roundPixels, lerpX, lerpY, offsetX, offsetY);
```

`roundPixels: true` aligns the camera to integer pixels, which avoids subpixel blur on pixel art. `offsetX` and `offsetY` shift where the target appears on screen — `offsetY: -50` keeps the player low on the screen so they can see what's coming up.

To stop following, call `cameras.main.stopFollow()`. The camera holds its last position; you can then move it manually with `setScroll(x, y)` or hand it back to a different target with another `startFollow`.

# --questions--

## --text--

A 4000-pixel-wide level renders correctly in Phaser, but the player walks off-screen and the camera stays put. What's the fix?

## --answers--

Call `this.cameras.main.startFollow(this.player)` after creating the player.

---

Set the camera's `x` and `y` directly each frame in `update`.

### --feedback--

Workable, but `startFollow` is the canonical, declarative API. It also handles lerp, deadzones, and offsets in one call.

---

Increase the canvas width to 4000 pixels.

### --feedback--

That removes the scrolling instead of fixing the camera. The point of a wide world is that it's larger than the camera viewport.

---

Move the player to `x = 0` so the camera sees them.

### --feedback--

That defeats the purpose of a wide world.

## --text--

You want the player low on the screen so they can see what's coming up. Which `startFollow` argument controls that?

## --answers--

`offsetY` — `startFollow(target, true, 1, 1, 0, -50)` shifts the target 50 px above centre, so they appear toward the bottom of the screen.

---

`lerpY` — set it to a small value to slow vertical follow.

### --feedback--

Lerp affects how *fast* the camera reacts, not where on the screen the target is rendered.

---

`roundPixels` — set it to `false` to allow subpixel positioning.

### --feedback--

`roundPixels` controls pixel-perfect alignment, not the on-screen position.

---

`stopFollow` — pause and reposition manually.

### --feedback--

That works, but it's manual and clunky. `offsetY` is the declarative way.

## --text--

After `startFollow`, you call `stopFollow`. What happens to the camera?

## --answers--

It holds its last position. You can move it manually with `setScroll(x, y)` or hand it to a different target with another `startFollow`.

---

It snaps back to `(0, 0)`.

### --feedback--

It does not. The camera is stateful and remembers where it was.

---

It throws an error because nothing is following.

### --feedback--

`stopFollow` is a no-op when nothing is being followed; it does not throw.

---

It scrolls back to wherever the previous `startFollow` target spawned.

### --feedback--

The camera does not remember spawn positions; it remembers its own current position.
