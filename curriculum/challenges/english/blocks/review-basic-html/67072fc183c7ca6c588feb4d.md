# --interactive--

The LSTM was designed to solve the vanishing gradient problem by giving the network a dedicated, protected memory channel. Instead of relying solely on the hidden state to carry information forward, the LSTM maintains a separate **cell state** that can store information across hundreds of time steps.

### Two States Instead of One

A simple RNN has one state: the hidden state `h_t`.

An LSTM has two states:
- **Hidden state `h_t`** — the short-term working memory, passed to the next layer and to the next time step
- **Cell state `c_t`** — the long-term memory highway, protected by gates from being overwritten too easily

The cell state flows from one time step to the next with only minor, controlled modifications. This provides a gradient highway that allows signals to flow backward through time without vanishing.

### The Three Gates

The LSTM controls information flow using three learnable gates. Each gate produces a value between 0 and 1 using a sigmoid activation — 0 means "block everything", 1 means "let everything through".

**Forget gate** — decides how much of the previous cell state to keep:

```
f_t = sigmoid(W_f * [h_{t-1}, x_t] + b_f)
c_t = f_t * c_{t-1}   (element-wise multiplication)
```

If `f_t` is close to 0, the cell state is wiped. If close to 1, it is preserved almost unchanged.

**Input gate** — decides what new information to add to the cell state:

```
i_t = sigmoid(W_i * [h_{t-1}, x_t] + b_i)
g_t = tanh(W_g * [h_{t-1}, x_t] + b_g)    # candidate values
c_t = f_t * c_{t-1} + i_t * g_t
```

**Output gate** — decides what part of the cell state to expose as the hidden state:

```
o_t = sigmoid(W_o * [h_{t-1}, x_t] + b_o)
h_t = o_t * tanh(c_t)
```

### An Intuition for the Gates

Think of the cell state as a conveyor belt carrying important facts through time.

- The **forget gate** is an editor deciding which old facts are no longer relevant and should be discarded
- The **input gate** is a writer deciding which new facts from the current input are worth recording
- The **output gate** is a speaker deciding which stored facts are relevant to share right now

### LSTM in TensorFlow

Using an LSTM in Keras is as simple as swapping `SimpleRNN` for `LSTM`:

```python
import tensorflow as tf

model = tf.keras.Sequential([
    tf.keras.layers.Embedding(input_dim=10000, output_dim=128),
    tf.keras.layers.LSTM(256, return_sequences=True),
    tf.keras.layers.LSTM(128),
    tf.keras.layers.Dense(64, activation='relu'),
    tf.keras.layers.Dropout(0.3),
    tf.keras.layers.Dense(1, activation='sigmoid')
])

model.compile(optimizer='adam',
              loss='binary_crossentropy',
              metrics=['accuracy'])
```

Stacking two LSTM layers (the first with `return_sequences=True`) allows the network to learn hierarchical temporal patterns — similar to how stacking convolutional layers learns hierarchical spatial patterns.

### LSTM Parameter Count

An LSTM has roughly four times the parameters of a simple RNN with the same hidden size, because of the four weight matrices (forget, input, candidate, output). This makes them slower to train but much more capable.

For a hidden size of 128 and input size of 64:
```
SimpleRNN parameters ≈ (64 + 128) × 128 = 24,576
LSTM parameters      ≈ (64 + 128) × 128 × 4 = 98,304
```

### Key Points

- The LSTM has two states: the cell state (long-term memory) and the hidden state (short-term memory)
- Three gates — forget, input, and output — control what information is stored, updated, and exposed
- The cell state provides a gradient highway that allows learning of long-range dependencies
- LSTMs have approximately four times the parameters of simple RNNs of the same hidden size
- Stacking LSTM layers learns hierarchical temporal patterns

What is the key structural difference between a simple RNN and an LSTM?

## --answers--

An LSTM processes sequences in both forward and backward directions simultaneously

### --feedback--

Bidirectional processing is a separate technique that can be applied to both RNNs and LSTMs. It is not the core structural difference.

---

An LSTM uses convolutional filters instead of weight matrices to process sequences

### --feedback--

LSTMs use weight matrices and gates, not convolutional filters. Combining convolutions with sequences is a different architecture called a temporal CNN.

---

An LSTM has a separate cell state that acts as a long-term memory channel, controlled by learnable gates

---

An LSTM processes all time steps in parallel instead of sequentially

### --feedback--

Parallel processing of all time steps is a property of transformers, not LSTMs. LSTMs still process sequences step by step.

## --text--

What does the forget gate in an LSTM decide?

## --answers--

How much of the current input to pass to the next layer

### --feedback--

Passing input to the next layer is the role of the output gate combined with the hidden state. The forget gate operates on existing stored memory.

---

Which neurons should be randomly dropped to prevent overfitting

### --feedback--

Dropout is a separate regularization technique. The forget gate is a learned component of the LSTM that controls memory retention.

---

How much of the previous cell state (long-term memory) to keep or discard

---

Whether to stop training when the loss stops improving

### --feedback--

Early stopping is a training callback, not part of the LSTM architecture. The forget gate operates during every forward pass.

## --text--

Why do LSTMs have approximately four times the parameters of a simple RNN with the same hidden size?

## --answers--

Because LSTMs process sequences four times as fast

### --feedback--

More parameters mean more computation per time step — LSTMs are actually slower than simple RNNs. The extra parameters come from the gating mechanisms.

---

Because LSTMs maintain four separate hidden states

### --feedback--

LSTMs maintain two states (hidden state and cell state), not four. The four-times parameter count comes from the four weight matrices needed for the gates and candidate values.

---

Because LSTMs require four weight matrices for the forget, input, candidate, and output gates

---

Because LSTMs apply the embedding layer four times per time step

### --feedback--

The embedding layer is applied once per sequence. The extra parameters in an LSTM come from the four weight matrices inside the LSTM cell itself.
