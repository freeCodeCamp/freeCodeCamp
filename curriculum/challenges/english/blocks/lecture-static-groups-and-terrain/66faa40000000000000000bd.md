# --interactive--

A static group is the right tool for small, hand-placed terrain — a debug arena, a tutorial level, a single boss room. Once a level grows past about 50 tiles, or once you want to design it in a visual editor, you outgrow the static-group pattern and reach for a **tilemap**.

A tilemap is a 2D grid of indices into a tileset image, plus collision metadata, layered alongside object data (spawn points, doors, enemy positions). Phaser ingests the JSON exported by tools like Tiled and turns it into a `TilemapLayer`, which collides with dynamic bodies the same way a static group does — but it is dramatically more efficient for large grids and trivially editable.

```js
function create() {
  const map = this.make.tilemap({ key: "level1" });
  const tileset = map.addTilesetImage("dungeon", "dungeon-tiles");
  const ground = map.createLayer("ground", tileset, 0, 0);
  ground.setCollisionByProperty({ collides: true });
}
```

:::interactive_editor

```js
function create() {
  this.platforms = this.physics.add.staticGroup();
  for (let x = 0; x < 320; x += 16) {
    this.platforms.create(x + 8, 200, "tile");
  }
  for (let y = 0; y < 200; y += 16) {
    this.platforms.create(8, y + 8, "tile");
  }
}
```

:::

Trade-offs at a glance:

- **Static group**: zero tooling, every tile lives in code, easy to reason about, scales poorly past ~50 tiles, no built-in object metadata.
- **Tilemap**: requires a `.json` exported from Tiled (or hand-written), supports massive levels, collision flags per tile, object layers for spawns and triggers, but adds a tooling step.

The platformer workshop you are about to build uses static groups — it is a single hand-crafted level, the geometry fits in 30 tiles, and the goal is to drill into Arcade physics, not into level-editor tooling. Chapter 5 then revisits all of this with tilemaps and Tiled, building the dungeon explorer.

# --questions--

## --text--

Which scenario is the strongest argument for switching from a static group to a tilemap?

## --answers--

The level grew to several hundred tiles and a designer wants to lay it out visually in Tiled.

---

The level uses 16-pixel tiles instead of 32-pixel tiles.

### --feedback--

Tile size is independent of which container you choose.

---

The player needs to walk on the platforms.

### --feedback--

Both static groups and tilemaps support walking.

---

The game should run at 60 fps.

### --feedback--

Both work fine at 60 fps for a small level. Tilemaps win on large levels.

## --text--

Which feature exists in tilemaps that does *not* exist in static groups?

## --answers--

An object layer that ships spawn points, enemy positions, and door triggers alongside the tile grid.

---

Collision with dynamic bodies.

### --feedback--

Both support that.

---

A `body.touching` flag.

### --feedback--

`touching` is a property on dynamic bodies; the *terrain* type is irrelevant.

---

Loading a sprite sheet.

### --feedback--

Sprite sheets are a separate asset type that both can use.
