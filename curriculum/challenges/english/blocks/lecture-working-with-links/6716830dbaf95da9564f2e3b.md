# --interactive--

A Recurrent Neural Network (RNN) solves the memory problem by maintaining a hidden state — a summary of everything it has seen so far — and updating it at every time step. This gives the network a form of memory that persists across the sequence.

### The Core Idea: A Loop in Time

In a standard neural network, data flows forward once. An RNN has a loop: at each time step, the network takes two inputs — the current element of the sequence and the hidden state from the previous time step — and produces a new hidden state.

```
h_t = tanh(W_x * x_t + W_h * h_{t-1} + b)
```

Where:
- `x_t` is the input at time step t
- `h_{t-1}` is the hidden state from the previous time step
- `W_x` and `W_h` are weight matrices
- `b` is the bias
- `h_t` is the new hidden state

The same weights (`W_x`, `W_h`, `b`) are used at every time step. This is weight sharing over time — analogous to how CNNs share weights over space.

### Unrolling the RNN

It helps to visualise an RNN "unrolled" through time. For a sequence of 4 words:

```
x_1 → [RNN] → h_1
x_2 → [RNN] → h_2   (also takes h_1 as input)
x_3 → [RNN] → h_3   (also takes h_2 as input)
x_4 → [RNN] → h_4   (also takes h_3 as input)
         ↓
      output
```

Each box represents the same set of weights applied at each time step. The hidden state carries information forward from each step to the next.

### A Simple RNN in TensorFlow

```python
import tensorflow as tf

model = tf.keras.Sequential([
    # Embedding converts integer word IDs to dense vectors
    tf.keras.layers.Embedding(input_dim=10000, output_dim=64),

    # SimpleRNN processes the sequence
    tf.keras.layers.SimpleRNN(128, return_sequences=False),

    # Output layer for binary classification
    tf.keras.layers.Dense(1, activation='sigmoid')
])

model.compile(optimizer='adam',
              loss='binary_crossentropy',
              metrics=['accuracy'])
model.summary()
```

`return_sequences=False` means the RNN outputs only the final hidden state `h_T` — a summary of the entire sequence. Setting it to `True` returns the hidden state at every time step, which is needed when stacking RNN layers.

### Many-to-One vs Many-to-Many

RNNs are flexible in what they input and output:

| Mode | Input | Output | Example use |
|---|---|---|---|
| Many-to-one | Full sequence | Single value | Sentiment classification |
| Many-to-many (equal) | Sequence | Same-length sequence | Part-of-speech tagging |
| Many-to-many (unequal) | Sequence | Different-length sequence | Machine translation |
| One-to-many | Single input | Sequence | Image captioning |

### The Hidden State as Memory

The hidden state at the last time step contains a compressed representation of the entire input sequence. For a sentiment classifier reading a movie review, `h_T` encodes "this review is mostly positive" after reading all the words. The Dense layer then uses this compressed summary to make the final prediction.

### Key Points

- An RNN maintains a hidden state that is updated at every time step, giving it memory across the sequence
- The same weights are reused at every time step — weight sharing over time
- `return_sequences=False` returns only the final hidden state; `True` returns all hidden states
- RNNs can operate in many-to-one, many-to-many, or one-to-many modes depending on the task
- The final hidden state is a compressed summary of the entire input sequence

## --text--

What two inputs does an RNN receive at each time step?

## --answers--

The current input element and the output from the previous layer

### --feedback--

The output from a previous layer is how information flows in feedforward networks. An RNN has a different mechanism for carrying information forward within the same layer.

---

The current input element and the hidden state from the previous time step

---

The entire sequence and a randomly initialised hidden state

### --feedback--

The RNN processes one time step at a time, not the entire sequence at once. The hidden state is carried over from the previous step, not randomly reinitialised.

---

The loss value and the gradient from the previous time step

### --feedback--

Loss and gradients are used in backpropagation during training, not as inputs to the forward pass of an RNN.

## --text--

What does setting `return_sequences=True` on an RNN layer do?

## --answers--

It causes the layer to process the input sequence in reverse order

### --feedback--

Reversing the sequence is a separate option. `return_sequences` controls how many hidden states are returned as output, not the processing direction.

---

It returns the hidden state at every time step, not just the final one

---

It returns only the hidden state from the first time step

### --feedback--

`return_sequences=True` returns all hidden states. `return_sequences=False` (the default) returns only the final one.

---

It doubles the number of parameters in the RNN layer

### --feedback--

`return_sequences` only changes the output shape of the layer. The number of parameters (weights) is the same either way.
