# --interactive--

Images are grids where every pixel is independent of time. Sequential data is fundamentally different — the order in which values appear carries meaning. Understanding this distinction is the key to understanding why a new family of neural networks is needed.

### What Makes Data Sequential?

Data is sequential when each element has a meaningful relationship to the elements that came before and after it. Changing the order destroys the meaning.

Consider the sentence "The cat sat on the mat." Rearranging the words to "mat the on sat cat The" produces nonsense. The meaning lives in the sequence, not just the individual words.

Examples of sequential data:

- **Text** — sentences, paragraphs, documents
- **Speech** — audio waveforms sampled over time
- **Time series** — stock prices, temperature readings, sensor data
- **Video** — frames that must be understood in temporal order
- **DNA sequences** — strings of nucleotides where position matters
- **Music** — notes that form melodies through their order and timing

### Why Standard Networks Struggle with Sequences

A fully connected network and a CNN both have a fixed input size. They process each input independently and have no memory of previous inputs. This creates two problems for sequential data:

1. **Variable length** — sentences have different numbers of words. Time series can span days or years. A fixed-size input cannot accommodate this naturally.

2. **No memory across time steps** — if you process a sentence one word at a time with a Dense layer, the network forgets each word the moment it moves to the next. Context — knowing that "bank" refers to a river bank because the word "river" appeared three words ago — is lost.

### Time Steps

When working with sequential data, each element in the sequence is called a **time step**. A sentence with 10 words has 10 time steps. A time series with 30 daily readings has 30 time steps. A 5-second audio clip sampled at 16,000 Hz has 80,000 time steps.

In TensorFlow, sequential data is typically represented as a 3D tensor with shape:

```
(batch_size, time_steps, features)
```

For example, a batch of 32 sentences, each 20 words long, where each word is represented by 50 numbers, has shape `(32, 20, 50)`.

### Fixed-Length vs Variable-Length Sequences

Most deep learning frameworks process sequences in fixed-length batches. Variable-length sequences are handled by either:

- **Padding** — adding zeros to shorter sequences so all sequences in a batch have the same length
- **Truncating** — cutting longer sequences to a maximum length

```python
import tensorflow as tf

sequences = [[1, 2, 3], [4, 5], [6, 7, 8, 9]]

padded = tf.keras.preprocessing.sequence.pad_sequences(
    sequences, maxlen=4, padding='post'
)
# [[1 2 3 0]
#  [4 5 0 0]
#  [6 7 8 9]]
```

### Key Points

- Sequential data is data where order carries meaning — rearranging elements destroys information
- Examples include text, speech, time series, video, and DNA sequences
- Standard networks have no memory between inputs and cannot handle variable-length sequences naturally
- Each element in a sequence is called a time step
- Sequential data in TensorFlow has shape `(batch_size, time_steps, features)`

The last two dimensions are swapped. Time steps come before features in TensorFlow's convention for sequential data.

This shape is missing the features dimension and places batch_size in an unusual position. The standard convention is batch first, then time steps, then features.

### --feedback--

Fully connected networks process numbers — text and audio must first be converted to numbers, but the conversion is not the fundamental problem with sequences.

### --feedback--

Hardware compatibility is not the issue. The problem is architectural — how the network processes information over time.

---

### --feedback--

Filters are a feature of convolutional layers, not fully connected layers. The issue with Dense layers on sequences is the lack of memory between time steps.
