# --interactive--

Image classification answers one question: "What is in this image?" Object detection goes further: "What objects are in this image, where exactly are they, and how many are there?" This lesson introduces the key concepts behind object detection without requiring you to build a full detector from scratch.

### Classification vs Detection vs Segmentation

Computer vision tasks form a spectrum of increasing difficulty:

| Task | What it outputs | Example |
|---|---|---|
| Classification | One label per image | "This image contains a cat" |
| Object detection | Labels + bounding boxes | "Cat at (x=45, y=12, w=80, h=90)" |
| Semantic segmentation | A class label per pixel | Every pixel labelled as cat, background, etc. |
| Instance segmentation | Pixel masks per object | Two separate pixel masks for two different cats |

### Bounding Boxes

In object detection, each detected object is described by a **bounding box** — a rectangle defined by four numbers:

- `(x, y)` — the coordinates of the top-left corner
- `width, height` — the dimensions of the box

Or alternatively as `(x_min, y_min, x_max, y_max)` — the corners of the rectangle.

The model must predict both the class of the object and the coordinates of its bounding box. This makes object detection significantly harder than classification.

### Key Metrics: IoU and mAP

**Intersection over Union (IoU)** measures how well a predicted bounding box overlaps the true bounding box:

```
IoU = Area of Overlap / Area of Union
```

An IoU of 1.0 is a perfect match. A prediction is usually considered correct if its IoU exceeds 0.5.

**Mean Average Precision (mAP)** is the standard metric for evaluating object detectors. It averages the precision of the detector across all classes and across different IoU thresholds. Higher is better; state-of-the-art detectors on COCO achieve mAP > 55.

### Major Object Detection Architectures

**Two-stage detectors** (e.g., Faster R-CNN): First propose candidate regions that might contain objects, then classify each region. More accurate but slower.

**One-stage detectors** (e.g., YOLO, SSD): Predict all bounding boxes and class probabilities in a single forward pass. Faster and suitable for real-time applications.

**YOLO** (You Only Look Once) is one of the most widely used detectors. It divides the image into a grid and predicts bounding boxes and class probabilities for each grid cell simultaneously. YOLO versions have progressed from v1 to v9, with each version improving speed and accuracy.

### Using a Pre-trained Detector

You can use pre-trained object detectors directly without training from scratch:

```python
# Using TensorFlow Hub for object detection
import tensorflow_hub as hub
import tensorflow as tf

detector = hub.load("https://tfhub.dev/tensorflow/ssd_mobilenet_v2/2")

# Run detection on an image tensor
image = tf.image.decode_jpeg(tf.io.read_file("photo.jpg"))
image = tf.expand_dims(image, 0)  # add batch dimension

results = detector(image)
print(results['detection_classes'])    # class IDs
print(results['detection_scores'])     # confidence scores
print(results['detection_boxes'])      # bounding box coordinates
```

### Non-Maximum Suppression

A common issue in object detection is that the model predicts multiple overlapping bounding boxes for the same object. **Non-maximum suppression (NMS)** solves this by keeping only the highest-confidence box and removing any overlapping boxes that exceed an IoU threshold with it.

### Key Points

- Object detection outputs class labels and bounding box coordinates for every object in an image
- IoU measures the overlap between a predicted and true bounding box; mAP evaluates overall detector performance
- Two-stage detectors (Faster R-CNN) are more accurate; one-stage detectors (YOLO, SSD) are faster
- Non-maximum suppression removes duplicate overlapping predictions
- Pre-trained detectors from TensorFlow Hub can be used directly on new images

Boxes outside image boundaries are handled by clipping, not NMS. NMS addresses a different type of redundancy in predictions.

It removes duplicate overlapping bounding box predictions for the same object

NMS filters predictions — it does not boost confidence scores. It removes redundant boxes that overlap significantly with a higher-confidence prediction.

Coordinate normalisation is a preprocessing step unrelated to NMS. NMS is a post-processing step that removes redundant detections.
