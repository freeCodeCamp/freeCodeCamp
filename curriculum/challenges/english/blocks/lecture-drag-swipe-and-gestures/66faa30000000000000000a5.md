# --interactive--

A **swipe** is a press, a quick move, and a release in one motion. There is no `pointerswipe` event in Phaser — you measure it yourself from `pointerdown` and `pointerup`. Three numbers are all you need: how far the pointer moved, how long it took, and which direction it ended up going.

The pattern fits in twelve lines:

```js
let start = null;

this.input.on("pointerdown", pointer => {
  start = { x: pointer.x, y: pointer.y, t: this.time.now };
});

this.input.on("pointerup", pointer => {
  if (!start) return;
  const dx = pointer.x - start.x;
  const dy = pointer.y - start.y;
  const dt = this.time.now - start.t;
  if (Math.abs(dx) > 80 && dt < 250) {
    this.dashTo(dx > 0 ? "right" : "left");
  }
  start = null;
});
```

The two thresholds are the entire design surface:

- **Distance threshold** (80 px) — too low and a tap registers as a swipe; too high and the gesture feels stiff.
- **Time threshold** (250 ms) — too low and slow swipes get rejected; too high and a long press counts.

A good starting point on a 480-wide canvas is `> 80 px in < 250 ms`. Tune from there.

Direction is just sign + dominance:

```js
const horizontal = Math.abs(dx) > Math.abs(dy);
const dir = horizontal
  ? (dx > 0 ? "right" : "left")
  : (dy > 0 ? "down" : "up");
```

Whichever axis moved more wins. Tie-break in whichever direction your game cares about.

For pinpoint UI gestures (a card you flick off-screen), measure **velocity** rather than distance: `velocity = dx / dt`. A small move that happened in 30 ms can still be a confident flick. For most arcade controls, distance + time is plenty.

A common bug: forgetting to clear `start` on `pointerup`. The next `pointerdown` overwrites it, but if the user releases outside the canvas, you can listen to the wrong release later. Always reset to `null` on every `pointerup`.

If you only care about left/right swipes (paddle controls, inventory carousels), drop the vertical math entirely:

```js
if (Math.abs(dx) > 80 && dt < 250) {
  this.player.setVelocityX(dx > 0 ? 400 : -400);
}
```

That's the whole "flick to dash" mechanic in one block.

# --questions--

## --text--

Which set of measurements is enough to detect a horizontal swipe?

## --answers--

The pointer's horizontal distance and the time elapsed between `pointerdown` and `pointerup`.

---

The pointer's velocity in pixels per millisecond, ignoring distance.

### --feedback--

Velocity alone gives the rate but not the total travel; a tiny twitch can clear a velocity threshold without being a swipe.

---

The pointer's pressure and the angle of the move.

### --feedback--

Pressure isn't reliably exposed across browsers and isn't part of the swipe definition.

---

Just the direction (left or right); duration and distance don't matter.

### --feedback--

Without thresholds you'd treat every micro-jitter as a swipe.

## --text--

For a 480-pixel-wide canvas, which thresholds are a sensible starting point for "this counts as a swipe"?

## --answers--

`Math.abs(dx) > 80 && dt < 250` — at least 80 px in under 250 ms.

---

`Math.abs(dx) > 4 && dt < 50` — any quick wiggle.

### --feedback--

That fires on every accidental jitter; players will trigger swipes they didn't mean.

---

`Math.abs(dx) > 200 && dt < 1000` — you must drag a full second.

### --feedback--

That's a drag, not a swipe — by the time the user finishes, the moment is gone.

---

`dx > 0 && dt > 0` — anything moving right counts.

### --feedback--

That fires on a single rightward pixel of motion.

## --text--

In the swipe pattern shown in the lecture, what should you do with the `start` object after every `pointerup` to avoid stale data?

## --answers--

Reset it to `null` so the next `pointerdown` starts fresh.

---

Push it onto a queue for replay analytics.

### --feedback--

That's a separate concern; functionally, you still need to clear it for the next gesture.

---

Leave it alone — Phaser cleans up pointer state between events.

### --feedback--

Phaser does *not* clean up your closure state. If you don't reset, an outside-canvas release leaks a stale start into the next swipe.

---

Set its timestamp to zero but keep the coordinates.

### --feedback--

Half-resetting leaves enough state for the next swipe to misfire.
