# --interactive--

Self-attention processes all positions simultaneously — but this creates a problem. Unlike an RNN that inherently knows position 1 comes before position 2 (because it processes them in order), self-attention treats a sequence as an unordered set. The sentences "dog bites man" and "man bites dog" would produce identical attention patterns without some way to inject position information.

Positional encoding solves this by adding a position-dependent signal to each token's embedding before it enters the transformer.

### The Requirement

A good positional encoding must:

1. Produce a unique representation for each position
2. Encode relative distances — position 5 should be "closer" to position 6 than to position 50
3. Generalise to sequence lengths not seen during training
4. Not overpower the token's semantic meaning

### Sinusoidal Positional Encoding

The original transformer paper used sinusoidal functions to create positional encodings:

```
PE(pos, 2i)   = sin(pos / 10000^(2i/d_model))
PE(pos, 2i+1) = cos(pos / 10000^(2i/d_model))
```

Where `pos` is the position in the sequence and `i` is the dimension index. Even dimensions use sine; odd dimensions use cosine. Different dimensions use different frequencies — some oscillate rapidly (encoding fine-grained position), others oscillate slowly (encoding coarse position, like whether you are in the first or second half of the sequence).

The positional encoding is added directly to the token embedding:

```
input to transformer = token_embedding + positional_encoding
```

### A Visual Intuition

Think of a clock with multiple hands moving at different speeds. The combination of all hands' positions uniquely identifies the time, even though each individual hand is periodic. Sinusoidal positional encoding works similarly — the combination of many sine/cosine waves at different frequencies produces a unique "fingerprint" for each position.

### Implementing Sinusoidal Encoding

```python
import numpy as np
import tensorflow as tf

def get_positional_encoding(max_seq_len, d_model):
    positions = np.arange(max_seq_len)[:, np.newaxis]   # (max_seq_len, 1)
    dims      = np.arange(d_model)[np.newaxis, :]        # (1, d_model)

    angles = positions / np.power(10000, (2 * (dims // 2)) / d_model)

    # Apply sin to even indices, cos to odd indices
    angles[:, 0::2] = np.sin(angles[:, 0::2])
    angles[:, 1::2] = np.cos(angles[:, 1::2])

    return tf.cast(angles, dtype=tf.float32)  # (max_seq_len, d_model)

pe = get_positional_encoding(max_seq_len=100, d_model=512)
print(pe.shape)   # (100, 512)
```

### Learned Positional Embeddings

An alternative to sinusoidal encoding is to learn position embeddings during training — just like word embeddings. Each position gets a learnable vector that is trained alongside the rest of the model.

```python
# Learned positional embedding
pos_embedding = tf.keras.layers.Embedding(
    input_dim=max_seq_len,
    output_dim=d_model
)
positions = tf.range(start=0, limit=seq_len, delta=1)
pos_enc   = pos_embedding(positions)
```

Modern models like BERT and GPT use learned positional embeddings. They are simpler to implement and often perform similarly to sinusoidal encoding on standard benchmarks.

### Key Points

- Self-attention has no inherent sense of order, so position information must be explicitly added
- Positional encodings are added to token embeddings before the first transformer layer
- Sinusoidal encoding uses sine and cosine functions at different frequencies to create unique position signatures
- Learned positional embeddings are an alternative where position vectors are trained like word embeddings
- Modern models like BERT and GPT typically use learned positional embeddings

## --text--

What is the advantage of learned positional embeddings over sinusoidal positional encoding?

## --answers--

Learned embeddings generalise better to sequences longer than those seen during training

### --feedback--

This is actually an advantage of sinusoidal encoding — it can extrapolate to longer sequences. Learned embeddings only work well up to the maximum length seen during training.

---

Learned embeddings require fewer parameters than sinusoidal encoding

### --feedback--

Sinusoidal encoding has zero parameters — it is a fixed mathematical function. Learned embeddings add parameters to the model. Simplicity is a different kind of advantage.

---

Learned embeddings are simpler to implement and often perform similarly on standard benchmarks

---

Learned embeddings eliminate the need for the multi-head attention layer

### --feedback--

Positional embeddings and multi-head attention are separate components. Positional encoding informs the model of order; attention computes relationships between positions.
