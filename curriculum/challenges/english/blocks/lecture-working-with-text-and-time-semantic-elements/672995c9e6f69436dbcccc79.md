# --interactive--

A single self-attention operation captures one type of relationship between positions. Multi-head attention runs several self-attention operations in parallel, each learning to focus on different aspects of the sequence simultaneously.

### The Limitation of Single-Head Attention

Consider the sentence: "John gave Mary the book that she had been wanting."

A single attention head might focus on the subject-verb relationship (John → gave). But at the same time, the model also needs to track the coreference (she → Mary), the object relationship (gave → book), and the relative clause (book → wanting). A single attention head cannot capture all of these simultaneously.

### How Multi-Head Attention Works

Multi-head attention splits the model's representation into H separate "heads". Each head has its own set of learned projection matrices (W_q, W_k, W_v) and computes its own attention independently:

```
Head_1 = Attention(Q·W_q1, K·W_k1, V·W_v1)
Head_2 = Attention(Q·W_q2, K·W_k2, V·W_v2)
...
Head_H = Attention(Q·W_qH, K·W_kH, V·W_vH)
```

Each head projects into a lower-dimensional space (`d_k = d_model / H`), so the total computation is similar to a single full-dimensional attention.

After all heads compute their outputs, they are concatenated and projected back to the original model dimension:

```
MultiHead(Q, K, V) = Concat(Head_1, ..., Head_H) · W_o
```

### What Each Head Learns

Different heads learn to specialise in different relationship types. Research visualising transformer attention has found heads that consistently:

- Track subject-verb agreement across sentences
- Connect pronouns to their antecedents
- Attend to the immediately preceding token
- Focus on the end-of-sentence token for classification tasks
- Capture syntactic dependency relationships

This specialisation emerges from training — no head is told what to focus on. The diversity of learned patterns is what makes multi-head attention so expressive.

### Multi-Head Attention in TensorFlow

```python
import tensorflow as tf

# Multi-head attention layer
mha = tf.keras.layers.MultiHeadAttention(
    num_heads=8,
    key_dim=64       # dimension per head (d_k)
)

# Example: sequence of 10 tokens, each represented as 512-dimensional vector
query = tf.random.normal((1, 10, 512))
key   = tf.random.normal((1, 10, 512))
value = tf.random.normal((1, 10, 512))

output = mha(query, key, value)
print(output.shape)   # (1, 10, 512) — same shape as input
```

For self-attention, query, key, and value are all the same tensor:

```python
output = mha(query, query, query)   # self-attention
```

### Parameter Count

For 8 heads with `d_model = 512`:
- Each head: `d_k = 512 / 8 = 64`
- Per head matrices: 3 × (512 × 64) = 98,304 weights
- All 8 heads: 8 × 98,304 = 786,432 weights
- Output projection W_o: 512 × 512 = 262,144 weights
- **Total: ~1 million parameters** per multi-head attention layer

### Key Points

- Multi-head attention runs H independent attention operations in parallel, each in a lower-dimensional subspace
- Each head learns to specialise in different relationship types — syntactic, semantic, coreference, etc.
- Outputs from all heads are concatenated and projected back to the model dimension
- In TensorFlow, `MultiHeadAttention` handles all of this automatically
- The total computation is similar to single-head attention because each head works in a reduced dimension

## --text--

Why is multi-head attention used instead of a single attention head?

## --answers--

Because a single head cannot process sequences longer than 512 tokens

### --feedback--

Sequence length limits are a hardware and memory concern, not a fundamental limitation of single-head attention. Multi-head attention is used for a different reason related to what can be learned.

---

Because multiple heads allow the model to capture different types of relationships simultaneously

---

Because multi-head attention uses fewer parameters than single-head attention

### --feedback--

Multi-head attention uses a similar number of parameters to single-head attention — the heads work in lower dimensions, but there are more of them. Parameter efficiency is not the motivation.

---

Because single-head attention cannot compute gradients during backpropagation

### --feedback--

Single-head attention is fully differentiable and can be trained with backpropagation. The motivation for multiple heads is about representation diversity, not gradient computation.

512

### --feedback--

Using the full model dimension for each head would multiply the computation by the number of heads. Each head works in a reduced dimension.

---

8

### --feedback--

8 is the number of heads, not the dimension per head. The per-head dimension is computed by dividing the model dimension by the number of heads.

---

64

---

4096

### --feedback--

4096 would result from multiplying rather than dividing. The per-head dimension is `d_model / num_heads = 512 / 8 = 64`.
