# --interactive--

The original transformer was designed for sequence-to-sequence tasks like machine translation. It consists of two stacks of layers — an encoder that reads the input and a decoder that generates the output. Understanding this architecture gives you the foundation for understanding BERT (encoder-only) and GPT (decoder-only).

### The Encoder Stack

The encoder takes the input sequence (e.g., an English sentence) and transforms it into a sequence of rich contextual representations. It consists of N identical layers stacked on top of each other (the original paper used N=6).

Each encoder layer has two sub-layers:

1. **Multi-Head Self-Attention** — each position attends to all positions in the input
2. **Feed-Forward Network** — a simple two-layer fully connected network applied independently to each position

Each sub-layer uses a **residual connection** followed by **layer normalisation**:

```
output = LayerNorm(x + SubLayer(x))
```

The residual connection (adding the input `x` back to the sub-layer's output) helps gradients flow during training and prevents the vanishing gradient problem.

### The Decoder Stack

The decoder generates the output sequence one token at a time, using the encoder's output at every step. Each decoder layer has three sub-layers:

1. **Masked Multi-Head Self-Attention** — the decoder attends to its own previous outputs, but with masking so it cannot see future tokens (it must predict the next token without cheating)
2. **Cross-Attention** — the decoder attends to the encoder's output (this is where the translation happens — the decoder looks at the full source sequence to inform each output token)
3. **Feed-Forward Network** — same as in the encoder

### Why Masking in the Decoder?

The decoder is autoregressive — it generates tokens one at a time, left to right. When generating token at position 5, it should only see tokens 1 through 4. The masking sets attention scores for future positions to -∞ before softmax, which makes their attention weights effectively zero.

```
Position 1 can attend to: [1]
Position 2 can attend to: [1, 2]
Position 3 can attend to: [1, 2, 3]
Position 4 can attend to: [1, 2, 3, 4]
```

### The Full Architecture

```
Input sequence
      ↓
[Token Embedding + Positional Encoding]
      ↓
[Encoder Layer 1] → Self-Attention → Feed-Forward
[Encoder Layer 2] → Self-Attention → Feed-Forward
...
[Encoder Layer 6] → Self-Attention → Feed-Forward
      ↓
   Encoder Output (used by all decoder layers)

Output sequence (shifted right)
      ↓
[Token Embedding + Positional Encoding]
      ↓
[Decoder Layer 1] → Masked Self-Attention → Cross-Attention → Feed-Forward
[Decoder Layer 2] → Masked Self-Attention → Cross-Attention → Feed-Forward
...
[Decoder Layer 6] → Masked Self-Attention → Cross-Attention → Feed-Forward
      ↓
[Linear + Softmax] → Probability over vocabulary
```

### Encoder-Only and Decoder-Only Variants

The full encoder-decoder is used for tasks requiring both reading and generation (translation, summarisation). Many tasks only need one half:

- **Encoder-only** (BERT) — reads the full input with bidirectional attention; good for classification and understanding tasks
- **Decoder-only** (GPT) — generates text autoregressively; good for language modelling and generation tasks

### Key Points

- The encoder processes the full input sequence using bidirectional self-attention
- The decoder generates output autoregressively using masked self-attention and cross-attention to the encoder
- Residual connections and layer normalisation appear after every sub-layer
- Masking prevents the decoder from attending to future tokens during training
- BERT is encoder-only; GPT is decoder-only; T5 and the original transformer use the full encoder-decoder

## --text--

What is the purpose of the residual connection in each transformer sub-layer?

## --answers--

It stores a copy of the input for use during inference

### --feedback--

Residual connections are active during both training and inference, not just inference. Their purpose is related to how gradients flow during backpropagation.

---

It helps gradients flow during training and prevents the vanishing gradient problem in deep networks

---

It doubles the model's capacity by adding an extra copy of each layer

### --feedback--

A residual connection adds the input back to the output — it does not duplicate layers or double capacity. It is a skip connection that aids training stability.

---

It forces each layer to learn the same representation as the layer before it

### --feedback--

The opposite is true — residual connections allow each layer to learn only the residual (the difference from the input), making it easier to learn incremental improvements.

## --text--

Why does the decoder use masked self-attention?

## --answers--

To prevent the model from attending to padding tokens in the input

### --feedback--

Padding masks are a separate type of masking. The decoder's masked self-attention addresses a different issue related to the generation process.

---

To reduce the computational cost of attention by only processing nearby tokens

### --feedback--

The masking in the decoder does not reduce computation for efficiency — all positions are still computed. The masking serves a different purpose related to the training objective.

---

To prevent the decoder from seeing future tokens when predicting the current token

---

To block the decoder from attending to the encoder output more than once

### --feedback--

The decoder attends to the encoder output at every layer via cross-attention. The masking applies to the decoder's self-attention over its own outputs, not the cross-attention to the encoder.

## --text--

What is the difference between self-attention in the encoder and cross-attention in the decoder?

## --answers--

Self-attention uses sine functions; cross-attention uses cosine functions

### --feedback--

Sine and cosine functions are part of positional encoding, not the difference between attention types. Both self-attention and cross-attention use the same QKV formula.

---

In self-attention, Q, K, and V come from the same sequence; in cross-attention, Q comes from the decoder and K, V come from the encoder output

---

Self-attention is applied once per layer; cross-attention is applied once per training epoch

### --feedback--

Both self-attention and cross-attention are applied at every forward pass through every layer. The difference is about where the queries, keys, and values originate.

---

Cross-attention does not use the softmax function, making it faster than self-attention

### --feedback--

Cross-attention uses the same softmax-based formula as self-attention. The only difference is the source of the query, key, and value vectors.
