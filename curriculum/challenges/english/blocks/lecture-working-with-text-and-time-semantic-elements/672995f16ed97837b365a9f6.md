# --interactive--

BERT (Bidirectional Encoder Representations from Transformers) was introduced by Google in 2018 and transformed the field of Natural Language Processing. It showed that a transformer encoder pre-trained on massive text data could be fine-tuned to achieve state-of-the-art performance on almost any NLP task with minimal task-specific architecture.

### The Key Insight: Pre-training and Fine-tuning

Before BERT, each NLP task required training a model from scratch with a task-specific architecture. BERT introduced the idea of a two-phase approach:

1. **Pre-training** — train a large transformer encoder on vast amounts of unlabelled text (BERT used the entirety of Wikipedia and Books Corpus — over 3 billion words) using self-supervised objectives
2. **Fine-tuning** — take the pre-trained BERT and train it for a few more epochs on a small labelled dataset for the specific task

Fine-tuning BERT on a new task requires only a few thousand examples and a few hours of training, because BERT has already learned a rich understanding of language.

### Pre-training Objectives

BERT uses two pre-training tasks:

**Masked Language Modelling (MLM)** — 15% of tokens are randomly masked, and the model must predict the masked tokens using context from both sides. This forces BERT to build deep bidirectional representations.

```
Input:  "The [MASK] sat on the mat"
Target: predict "cat"
```

**Next Sentence Prediction (NSP)** — given two sentences, predict whether the second sentence follows the first in the original text. This trains BERT to understand relationships between sentences.

### Why Bidirectional Matters

GPT (a decoder-only transformer) processes text left to right — when encoding position 5, it can only see positions 1–4. BERT's encoder processes the full sequence in both directions simultaneously. This means every token's representation is influenced by all other tokens, giving a much richer contextual understanding.

```
"I went to the bank to deposit money"
"I sat by the river bank"
```

In both sentences, BERT's representation of "bank" is different because it sees the full context from both directions. GPT (left-to-right only) would not fully capture the right-side context when encoding "bank."

### Fine-tuning BERT for Different Tasks

The same pre-trained BERT model is fine-tuned differently depending on the task:

```python
from transformers import BertTokenizer, TFBertForSequenceClassification
import tensorflow as tf

# Load pre-trained BERT
tokenizer = BertTokenizer.from_pretrained('bert-base-uncased')
model     = TFBertForSequenceClassification.from_pretrained(
                'bert-base-uncased', num_labels=2)

# Tokenize input
text    = "This movie was absolutely brilliant!"
inputs  = tokenizer(text, return_tensors='tf', padding=True, truncation=True)
```

### BERT Variants

| Model | Parameters | Notes |
|---|---|---|
| BERT-base | 110M | 12 layers, 12 heads |
| BERT-large | 340M | 24 layers, 16 heads |
| DistilBERT | 66M | 40% smaller, 97% of BERT performance |
| RoBERTa | 125M | Improved training procedure |
| ALBERT | 12M | Parameter sharing, much smaller |

### Key Points

- BERT is a transformer encoder pre-trained on massive text using masked language modelling and next sentence prediction
- Pre-training on unlabelled text followed by fine-tuning on small labelled datasets became the dominant NLP paradigm
- Bidirectional context means every token sees all other tokens, producing richer representations than left-to-right models
- The same pre-trained BERT can be fine-tuned for classification, question answering, named entity recognition, and more
- DistilBERT and RoBERTa are popular BERT variants offering different speed/accuracy tradeoffs
