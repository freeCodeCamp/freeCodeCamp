# --interactive--

Training a GAN is unlike training a standard neural network. Instead of minimising a single loss function, you are simultaneously training two networks with opposing goals. This creates unique challenges and requires a carefully balanced training loop.

### The Training Loop

GAN training alternates between two steps in each iteration:

**Step 1 — Train the discriminator:**
1. Sample a batch of real images from the training set (label = 1)
2. Generate a batch of fake images using the current generator (label = 0)
3. Compute discriminator loss on both batches
4. Update discriminator weights only

**Step 2 — Train the generator:**
1. Generate a new batch of fake images
2. Pass them through the discriminator
3. Compute generator loss — the generator wants the discriminator to output 1 (real) for its fakes
4. Update generator weights only — the discriminator weights are frozen during this step

```python
import tensorflow as tf
import numpy as np

@tf.function
def train_step(real_images, generator, discriminator,
               g_optimizer, d_optimizer, latent_dim, batch_size):

    noise = tf.random.normal([batch_size, latent_dim])

    # ── Step 1: Train discriminator ──
    with tf.GradientTape() as d_tape:
        fake_images   = generator(noise, training=True)
        real_output   = discriminator(real_images, training=True)
        fake_output   = discriminator(fake_images, training=True)

        real_loss = cross_entropy(tf.ones_like(real_output),  real_output)
        fake_loss = cross_entropy(tf.zeros_like(fake_output), fake_output)
        d_loss    = real_loss + fake_loss

    d_grads = d_tape.gradient(d_loss, discriminator.trainable_variables)
    d_optimizer.apply_gradients(zip(d_grads, discriminator.trainable_variables))

    # ── Step 2: Train generator ──
    noise = tf.random.normal([batch_size, latent_dim])
    with tf.GradientTape() as g_tape:
        fake_images = generator(noise, training=True)
        fake_output = discriminator(fake_images, training=True)
        # Generator wants discriminator to output 1 (real) for its fakes
        g_loss = cross_entropy(tf.ones_like(fake_output), fake_output)

    g_grads = g_tape.gradient(g_loss, generator.trainable_variables)
    g_optimizer.apply_gradients(zip(g_grads, generator.trainable_variables))

    return d_loss, g_loss

cross_entropy = tf.keras.losses.BinaryCrossentropy(from_logits=False)
```

### The Nash Equilibrium

The ideal end state of GAN training is a **Nash equilibrium** — a game theory concept where neither player can improve by changing only their own strategy. In a GAN, this corresponds to:

- The generator producing outputs perfectly matching the real data distribution
- The discriminator outputting 0.5 for every image (it genuinely cannot tell real from fake)

In practice, this ideal is rarely fully reached, but good GANs get close.

### Monitoring Training Progress

Unlike standard networks, GAN training cannot be monitored with a single loss curve. You need to watch both losses:

| Pattern | Meaning |
|---|---|
| Both losses stable and moderate | Healthy training |
| D loss → 0, G loss → high | Discriminator too strong — generator cannot learn |
| G loss → 0, D loss → high | Generator collapsed to a single output (mode collapse) |
| Both losses oscillate wildly | Learning rate too high or training unstable |

The most reliable way to monitor GAN training is to periodically save and visually inspect generated samples.

### The Training Imbalance Problem

If the discriminator becomes too strong too quickly, it can perfectly distinguish real from fake before the generator has learned anything useful. The generator receives only near-zero gradients and cannot improve — this is the **vanishing gradient problem for GANs**.

Common remedies:
- Train the discriminator fewer steps per generator step (e.g., 1 discriminator step per 1 generator step)
- Use label smoothing — instead of labels 0 and 1, use 0.1 and 0.9
- Add noise to the discriminator's inputs early in training

### Key Points

- GAN training alternates between updating the discriminator and the generator
- Discriminator and generator weights are updated separately — one is frozen while the other trains
- The ideal end state is a Nash equilibrium where the discriminator outputs 0.5 for all images
- Training should be monitored by watching both losses and visually inspecting generated samples
- If the discriminator becomes too strong too early, the generator receives vanishing gradients and cannot learn

Parameter parity is an architectural choice, not what Nash equilibrium means. The equilibrium is about the strategies (learned distributions) of the two networks.

Reaching the maximum epoch count is just stopping training, not a Nash equilibrium. A true equilibrium is about the relationship between the two networks' capabilities.
