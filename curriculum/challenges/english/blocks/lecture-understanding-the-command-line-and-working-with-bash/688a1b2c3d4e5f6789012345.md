# --description--

Some keyboard layouts use non-English language settings that can cause unexpected errors in Bash and PostgreSQL workshops. For example, with certain layouts, psql may not recognize semicolons, preventing commands like `CREATE DATABASE;` from executing.

To avoid these issues, set your keyboard layout to **English (US)** before starting the workshops.

**On Windows:** Go to Settings → Time & Language → Language & Region → add English (United States) → set keyboard to US.

**On macOS:** Go to System Settings → Keyboard → Input Sources → add English (U.S.) and make it active.

**On Linux (command line):** Run `setxkbmap us` to switch to the US layout.

To verify your layout is correct, open a terminal and confirm these characters type as shown:

\- _ / \ | ' " : ; $ ( ) [ ] { }

If any character appears differently, your layout is still set to a non-English option.
