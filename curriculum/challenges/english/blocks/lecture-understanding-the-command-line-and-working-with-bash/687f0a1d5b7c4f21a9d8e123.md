# --description--

Let's learn how your system language and keyboard settings can affect Bash and PostgreSQL workshops.

Many programming tools expect commands to use standard English keyboard characters. In Bash and PostgreSQL workshops, this is especially important for symbols like semicolons (`;`), quotation marks, slashes, and other special characters.

If your operating system or keyboard layout uses a non-English language configuration, some characters may behave differently than expected. This can lead to confusing errors when entering SQL commands or terminal commands.

For the best experience while working through these workshops, it is recommended to temporarily switch to the English (US) keyboard layout while completing the course. This is especially helpful if commands in Bash or PostgreSQL are producing unexpected characters or syntax errors when typing symbols such as `;` or `\`.

GitHub Codespaces uses your local keyboard layout, so changing the layout on your own computer will also affect typing inside Codespaces.

On Windows:

1. Open **Settings -> Time & Language -> Language & Region**
2. Add **English (United States)** if it is not already installed
3. Under keyboard layouts, select the **US** keyboard
4. Move the US keyboard layout to the top of the list if needed

On macOS:

1. Open **System Settings -> Keyboard -> Input Sources**
2. Add the **English (US)** keyboard layout
3. Make sure the US layout is selected as the active keyboard

On Linux distributions such as Ubuntu, Debian, or Mint:

1. Open your system keyboard settings
2. Add the **English (US)** keyboard layout
3. Move it to the top of the keyboard layout list if possible

Some Linux systems also allow changing the keyboard layout from the terminal with commands like:

```bash
setxkbmap us
```

After switching layouts, test the following characters in a text box or terminal:

```md
; : / \ | ' " $ { } [ ] ( )
```

If the symbols appear correctly, your keyboard layout is likely configured properly.

If you use a non-US physical keyboard layout such as AZERTY, QWERTZ, or regional QWERTY variants, the symbol keys may be located in different physical positions. In that case, use your operating system's on-screen keyboard or keyboard viewer to locate keys such as `;`, `/`, `\`, `|`, `'`, and `"` after switching to the US layout.

Using the English (US) keyboard layout helps ensure that Bash and PostgreSQL commands behave exactly as expected throughout the workshop.
