# --description--

Let's learn how your system language settings can affect Bash and PostgreSQL workshops.

Many programming tools expect commands to use standard English keyboard characters. In Bash and PostgreSQL workshops, this is especially important for symbols like semicolons (`;`), quotation marks, and other special characters.

If your operating system or keyboard layout uses a non-English language configuration, some characters may behave differently than expected. This can lead to confusing errors when entering SQL commands or terminal commands.

For the best experience while working through these workshops, it is recommended to use an English keyboard layout and English system language settings whenever possible.

On Windows, you can manage your language settings through the "Language & Region" settings page.

On macOS, you can manage keyboard and language preferences through the System Settings application.

On Linux, the process depends on your distribution and desktop environment, but most systems allow language and keyboard configuration through their system settings applications.

If you continue to experience issues entering commands correctly, double-check that your keyboard layout is set to English before continuing with the workshop.
