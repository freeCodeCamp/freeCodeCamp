# --interactive--

Backpropagation tells the network which direction to adjust each weight. Gradient descent is the algorithm that actually moves the weights in that direction. Together they are the engine of neural network training.

### The Intuition

Imagine you are standing on a hilly landscape in thick fog. Your goal is to reach the lowest valley. You cannot see far ahead, but you can feel which way the ground slopes beneath your feet. Your strategy: take a small step in the downhill direction, stop, check the slope again, and repeat.

This is exactly what gradient descent does. The "landscape" is the loss surface — a high-dimensional space where each dimension corresponds to a weight. The "valley" is the set of weight values that produces the lowest loss. The gradient tells you which direction is uphill; gradient descent steps in the opposite direction.

### The Update Rule

After backpropagation computes the gradient, gradient descent updates each weight using this rule:

```
new_weight = old_weight - learning_rate * gradient
```

The **learning rate** is a small positive number (typically between 0.0001 and 0.1) that controls how large each step is.

- If the learning rate is **too large**, the steps overshoot the valley and the loss bounces around without converging
- If the learning rate is **too small**, training takes an extremely long time to converge
- A **well-chosen learning rate** leads to smooth, steady decrease in loss

### Variants of Gradient Descent

**Batch Gradient Descent** computes the gradient using the entire dataset before taking one step. This gives an accurate gradient but is very slow for large datasets.

**Stochastic Gradient Descent (SGD)** computes the gradient using a single random training example before each step. This is fast but noisy — the loss bounces up and down rather than decreasing smoothly.

**Mini-batch Gradient Descent** is the standard in practice. It splits the dataset into small batches (typically 32, 64, or 128 examples) and takes one step per batch. This balances accuracy and speed.

When people say "SGD" in the context of deep learning libraries, they usually mean mini-batch gradient descent.

### Modern Optimisers

Plain gradient descent has been improved by several modern optimisers that adapt the learning rate automatically:

- **Adam** (Adaptive Moment Estimation) — the most widely used optimiser. It maintains per-parameter learning rates and adapts them based on the history of gradients. A good default for most problems.
- **RMSprop** — adapts learning rates based on recent gradient magnitudes. Works well for recurrent networks.
- **AdaGrad** — reduces the learning rate for frequently updated parameters. Good for sparse data.

In TensorFlow:

```python
model.compile(
    optimizer='adam',
    loss='sparse_categorical_crossentropy',
    metrics=['accuracy']
)
```

### An Epoch

One full pass through the entire training dataset is called an **epoch**. Training typically runs for many epochs — from a few dozen to several thousand — until the loss stops improving.

### Key Points

- Gradient descent updates weights by stepping in the direction that reduces the loss
- The learning rate controls step size — too large overshoots, too small is very slow
- Mini-batch gradient descent is the standard variant used in practice
- Adam is the most widely used optimiser and a good default for most problems
- One full pass through the training data is called an epoch

What happens if the learning rate is set too high?

## --answers--

Training slows down significantly and may never converge

### --feedback--

A very slow convergence is the result of a learning rate that is too small, not too high.

---

The network ignores the gradient and keeps the same weights

### --feedback--

The gradient is always used in the update rule. The learning rate scales how much the gradient moves the weights, not whether it is used.

---

The weight updates overshoot the optimal values and the loss fails to converge

---

The activation functions stop working correctly

### --feedback--

Activation functions are independent of the learning rate. An excessively high learning rate causes unstable weight updates, not activation function failures.

What is an "epoch" in neural network training?

## --answers--

One update step using a single training example

### --feedback--

One update step using a single example describes one iteration of stochastic gradient descent, not an epoch.

---

One full pass through the entire training dataset

---

The point at which the loss stops decreasing

### --feedback--

The point where loss stops improving is called convergence. An epoch is simply one complete pass through the training data, regardless of whether the model has converged.

---

One forward propagation step

### --feedback--

A forward propagation step processes one batch or example. An epoch involves processing all training data, which requires many forward propagation steps.
