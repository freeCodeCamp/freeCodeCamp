# --interactive--

A neural network that performs perfectly on training data but fails on new data is useless in the real world. Understanding overfitting and underfitting is essential for building models that generalise well.

### The Goal: Generalisation

The real measure of a model's quality is not how well it performs on data it has already seen — it is how well it performs on **new, unseen data**. This ability is called generalisation.

### Underfitting

Underfitting occurs when a model is too simple to capture the patterns in the data. The model performs poorly on both training data and new data.

Common causes:
- The network is too small (too few neurons or layers)
- Training stopped too early
- The learning rate is too small
- The features provided are not informative enough

A model that predicts the same average value for every input, regardless of the features, is an extreme example of underfitting.

### Overfitting

Overfitting occurs when a model learns the training data too well — including the noise and random quirks that do not represent the true underlying pattern. The model performs very well on training data but poorly on new data.

Signs of overfitting:
- Training loss is low but validation loss is high
- Training accuracy is much higher than validation accuracy
- Performance degrades on every new example it has not seen before

A useful analogy: a student who memorises every question and answer from past exams without understanding the material will fail new questions. The student "overfit" to the training data.

### The Bias-Variance Tradeoff

This is often described as the **bias-variance tradeoff**:

- **High bias** (underfitting) — the model makes systematic errors because it is too simple
- **High variance** (overfitting) — the model is too sensitive to the specific training examples and fails to generalise
- The goal is to find a model with **low bias and low variance**

### Regularization Techniques

Regularization refers to techniques that reduce overfitting without making the model simpler.

**Dropout** randomly sets a fraction of neurons to zero during each training step. This prevents the network from relying too heavily on any single neuron and forces it to learn redundant representations.

```python
tf.keras.layers.Dropout(rate=0.5)  # randomly zeroes 50% of neurons
```

**L2 Regularization** (weight decay) adds a penalty to the loss function proportional to the square of the weights. This discourages large weights and keeps the model from becoming overly complex.

```python
tf.keras.layers.Dense(64, kernel_regularizer=tf.keras.regularizers.l2(0.01))
```

**Early Stopping** monitors validation loss during training and stops training when it starts to increase — even if training loss is still decreasing.

```python
callback = tf.keras.callbacks.EarlyStopping(monitor='val_loss', patience=5)
```

**Data Augmentation** artificially increases the training dataset by applying transformations (rotation, flipping, cropping) to existing examples. More varied data makes it harder for the model to memorise.

### Key Points

- Underfitting: model too simple, performs poorly on both training and new data
- Overfitting: model too complex, performs well on training data but poorly on new data
- Generalisation is the ability to perform well on unseen data — the true goal of training
- Dropout, L2 regularization, early stopping, and data augmentation are common ways to reduce overfitting
