# --description--

Before you begin this course, there is an important setup step for learners who use a non-English keyboard layout.

Tools like `psql` and Bash rely on specific characters such as `\`, `;`, `|`, `'`, `"`, `$`, `{`, and `}`. On international keyboards (for example, AZERTY, QWERTZ, or Latin American layouts), these characters may be located in different places, may require unusual key combinations, or may behave as "dead keys" that do not produce the expected character.

This can cause confusing, hard-to-debug errors. For example, typing a semicolon (`;`) at the end of a `psql` statement may produce no response at all if your keyboard sends a different character code.

To avoid these problems, you should temporarily switch your system keyboard layout to **English (US)** for the duration of this course.

## Windows

1. Open **Settings → Time & Language → Language & Region**.
2. If "English (United States)" is not listed, click **Add a language** and add it.
3. Under **Keyboards**, select **US**.
4. Move **US** to the top of the list or remove other layouts.

## macOS

1. Open **System Settings → Keyboard → Input Sources**.
2. Click **+** and add **English → U.S.**.
3. Ensure **U.S.** is the active layout.
4. Remove other layouts or disable automatic switching.

## Linux (Ubuntu, Debian, Mint, etc.)

**GUI method:**

1. Open **Settings → Keyboard**.
2. Add **English (US)** and move it to the top of the list.
