# --interactive--

Attention is the mechanism that allows a neural network to focus on the most relevant parts of an input when producing each part of an output. It is the core idea behind transformers and one of the most important innovations in modern deep learning.

### The Human Analogy

When you read the sentence "The animal didn't cross the street because it was too tired", you intuitively know that "it" refers to "animal" rather than "street". You resolved this ambiguity by paying attention to context — specifically, the word "tired" connects more naturally to an animal than to a street.

Attention mechanisms give neural networks a similar ability: when processing each word, the model can look at all other words and decide which ones are most relevant to understanding the current one.

### Attention in Sequence-to-Sequence Models

The original attention mechanism (Bahdanau attention, 2015) was introduced for machine translation. In a traditional encoder-decoder RNN, the decoder only sees the final encoder hidden state. With attention, the decoder can look at all encoder hidden states at each decoding step and compute a weighted sum — paying more attention to the most relevant source words.

```
Translating: "The cat sat on the mat" → French

When generating "chat" (cat):
  Attention weights: the=0.05, cat=0.85, sat=0.05, on=0.02, the=0.01, mat=0.02
  → Model focuses heavily on "cat"

When generating "assis" (sat):
  Attention weights: the=0.02, cat=0.10, sat=0.80, on=0.05, the=0.01, mat=0.02
  → Model focuses heavily on "sat"
```

Each decoder step produces a different set of attention weights, allowing it to focus on different parts of the source sentence.

### Queries, Keys, and Values

Modern attention is framed as a retrieval problem using three components:

- **Query (Q)** — what you are looking for ("what information do I need right now?")
- **Key (K)** — what each position has to offer ("what information do I contain?")
- **Value (V)** — the actual content to retrieve ("if you select me, here is what you get")

The attention score between a query and a key measures how relevant that position is. High score → high attention weight → more of that position's value is included in the output.

```
Attention(Q, K, V) = softmax(QK^T / √d_k) × V
```

The division by `√d_k` (the square root of the key dimension) prevents the dot products from becoming too large, which would push the softmax into regions with very small gradients.

### Attention Weights as a Heatmap

A useful way to visualise attention is as a heatmap where rows are queries (output positions) and columns are keys (input positions). A bright cell at (row i, column j) means "output position i paid a lot of attention to input position j."

These heatmaps reveal what the model has learned about relationships between words — and they often match human intuition about which words are related.

### Key Points

- Attention allows a model to focus on the most relevant parts of the input when producing each output
- The original attention mechanism let the decoder look at all encoder states instead of just the final one
- Attention is framed as a query-key-value retrieval: queries ask, keys describe, values deliver
- The formula `softmax(QK^T / √d_k) × V` computes a weighted sum of values based on query-key similarity
- Attention weights can be visualised as heatmaps showing which input positions influenced which output positions

## --text--

What problem did the original attention mechanism solve in sequence-to-sequence models?

## --answers--

It allowed the model to process sequences in parallel instead of sequentially

### --feedback--

Parallelisation is a benefit of the transformer's self-attention, not the original Bahdanau attention mechanism. The original attention still used RNNs.

---

It allowed the decoder to focus on all encoder hidden states at each step, instead of only the final one

---

It eliminated the need for an encoder entirely

### --feedback--

The encoder is still present with attention — the decoder gains the ability to selectively focus on all encoder outputs, not bypass the encoder.

---

It reduced the number of parameters needed for translation models

### --feedback--

Attention actually adds parameters to the model. Its purpose is to improve information access, not reduce model size.

The stored content that will be retrieved if a position is selected

### --feedback--

Stored content that gets retrieved describes the value. The query has a different role in the attention computation.

### --feedback--

Describing what each position contains is the role of the key. The query asks a question; the key provides the description to match against.

What the current position is looking for — the information it needs

---

The final weighted output after attention is applied

### --feedback--

The final weighted output is computed by multiplying attention weights with values. The query is an input to the computation, not the output.

## --text--

Why is the dot product divided by `√d_k` in the attention formula?

## --answers--

To normalise the attention weights so they sum to exactly 1

### --feedback--

The softmax function is what normalises weights to sum to 1. The division by `√d_k` addresses a different problem that occurs before softmax is applied.

---

To reduce the number of parameters in the attention layer

### --feedback--

`d_k` is a fixed dimension value, not a parameter. The scaling does not reduce parameters — it stabilises the numerical values going into softmax.

---

To prevent dot products from becoming too large, which would cause softmax to produce near-zero gradients

---

To convert the attention scores from floating point to integers

### --feedback--

No integer conversion happens in attention. The scaling keeps floating point values in a range where softmax produces useful gradients.
