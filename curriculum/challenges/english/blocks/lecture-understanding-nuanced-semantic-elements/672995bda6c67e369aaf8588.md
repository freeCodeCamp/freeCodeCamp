# --interactive--

The original attention mechanism connected two different sequences — an encoder and a decoder. Self-attention applies the same idea within a single sequence: every position attends to every other position in the same sequence to build a richer understanding of each element in context.

### What Makes it "Self" Attention?

In standard attention, queries come from the decoder and keys/values come from the encoder — two different sequences interact.

In self-attention, queries, keys, and values all come from the same sequence. Every word simultaneously acts as a query ("what do I need to know?"), a key ("what do I have to offer?"), and a value ("here is my content").

This allows each word to incorporate context from every other word in the sentence before being passed to the next layer.

### A Step-by-Step Example

Consider the sentence: **"The trophy didn't fit because it was too big."**

When self-attention processes the word "it":
- "it" sends out a query: "who or what am I?"
- Every word in the sentence returns a key
- The dot product between "it"s query and each word's key produces attention scores
- After softmax, the scores become attention weights
- "trophy" gets a high weight (0.72), "fit" gets moderate weight (0.15), others get low weights
- The output for "it" is a weighted combination of all words' values — strongly influenced by "trophy"

The resulting representation of "it" now encodes the information that it most likely refers to "trophy." This is coreference resolution happening inside the attention mechanism.

### Computing Self-Attention

For each position, three learned weight matrices project the input into query, key, and value spaces:

```python
import numpy as np

def self_attention(X, W_q, W_k, W_v):
    """
    X: input matrix of shape (seq_len, d_model)
    W_q, W_k, W_v: learned projection matrices of shape (d_model, d_k)
    """
    Q = X @ W_q    # queries: (seq_len, d_k)
    K = X @ W_k    # keys:    (seq_len, d_k)
    V = X @ W_v    # values:  (seq_len, d_k)

    d_k = K.shape[-1]
    scores = Q @ K.T / np.sqrt(d_k)   # (seq_len, seq_len)
    weights = softmax(scores, axis=-1) # attention weights
    output = weights @ V               # (seq_len, d_k)
    return output, weights
```

The output for each position is a weighted sum of all value vectors. Positions with high attention weights contribute more to the output.

### The Attention Matrix

The attention weights form a square matrix of shape `(seq_len, seq_len)`. Each row represents one position and sums to 1. The value at row i, column j tells you "how much did position i attend to position j?"

For a 6-word sentence, this is a 6×6 matrix. For a 512-token document, it is a 512×512 matrix. This quadratic scaling with sequence length is one of the computational challenges of transformers on very long sequences.

### What Self-Attention Learns

Through training, different self-attention heads learn to capture different types of relationships:

- **Syntactic relationships** — subject-verb agreement, modifier-noun connections
- **Coreference** — pronouns connecting to their antecedents
- **Semantic roles** — who did what to whom
- **Long-range dependencies** — connecting a word to a relevant word hundreds of positions away

### Key Points

- Self-attention lets every position in a sequence attend to every other position simultaneously
- Queries, keys, and values all come from the same input sequence
- Three learned projection matrices (W_q, W_k, W_v) create the query, key, and value representations
- The attention weight matrix is square — shape `(seq_len, seq_len)` — which scales quadratically with sequence length
- Self-attention can capture syntactic, semantic, and long-range relationships within a sequence

## --text--

What makes self-attention different from the original encoder-decoder attention?

## --answers--

Self-attention uses three matrices instead of one

### --feedback--

Both attention types use query, key, and value projections. The key distinction is about where the queries, keys, and values come from.

---

In self-attention, the queries, keys, and values all come from the same sequence

---

Self-attention only attends to adjacent positions, not all positions

### --feedback--

Self-attention attends to all positions simultaneously — that is its power. Attending only to adjacent positions would be more like a convolution.

---

Self-attention is used only in the decoder, not the encoder

### --feedback--

Self-attention is used in both encoder and decoder layers of the transformer. The distinction is not about which component uses it.

## --text--

Why does the attention weight matrix scale quadratically with sequence length?

## --answers--

Because each layer of the transformer doubles the sequence length

### --feedback--

Transformers do not change sequence length between layers. The quadratic scaling comes from the structure of the attention computation itself.

---

Because the weight matrices W_q, W_k, W_v grow with the square of the sequence length

### --feedback--

The projection matrices W_q, W_k, W_v have shape `(d_model, d_k)` — they depend on the model dimension, not the sequence length. The quadratic scaling comes from a different part of the computation.

---

Because every position must compute an attention score with every other position, producing a (seq_len × seq_len) matrix

---

Because softmax is applied twice — once per position and once per head

### --feedback--

Softmax is applied once per row of the attention matrix. The quadratic scaling is about the size of the attention matrix itself, not how many times softmax is applied.
