# --interactive--

In this lesson you will build, train, and evaluate a complete neural network using TensorFlow and Keras. Every concept from the previous lessons — layers, weights, activation functions, loss functions, and gradient descent — comes together here.

### The Dataset: MNIST

MNIST is a dataset of 70,000 handwritten digit images, each 28×28 pixels in greyscale, labelled 0 through 9. It is the "hello world" of deep learning and is built directly into TensorFlow.

```python
import tensorflow as tf

(x_train, y_train), (x_test, y_test) = tf.keras.datasets.mnist.load_data()
print(x_train.shape)  # (60000, 28, 28)
print(y_train.shape)  # (60000,)
```

### Step 1: Preprocess the Data

Neural networks work best with input values between 0 and 1. Pixel values range from 0 to 255, so we divide by 255 to normalise them. We also flatten each 28×28 image into a 784-element vector so it can be fed into a Dense layer.

```python
x_train = x_train / 255.0
x_test  = x_test  / 255.0
```

### Step 2: Build the Model

A `Sequential` model is a linear stack of layers. We build a network with:
- A `Flatten` layer to reshape each 28×28 image into 784 numbers
- A `Dense` hidden layer with 128 neurons and ReLU activation
- A `Dropout` layer to reduce overfitting
- A `Dense` output layer with 10 neurons (one per digit) and Softmax activation

```python
model = tf.keras.Sequential([
    tf.keras.layers.Flatten(input_shape=(28, 28)),
    tf.keras.layers.Dense(128, activation='relu'),
    tf.keras.layers.Dropout(0.2),
    tf.keras.layers.Dense(10, activation='softmax')
])
```

### Step 3: Compile the Model

Compiling configures three things: the optimiser (Adam), the loss function (sparse categorical cross-entropy for integer labels), and the metric to monitor (accuracy).

```python
model.compile(
    optimizer='adam',
    loss='sparse_categorical_crossentropy',
    metrics=['accuracy']
)
```

### Step 4: Train the Model

`model.fit()` runs the training loop — forward propagation, loss computation, backpropagation, and weight updates — for the specified number of epochs.

```python
model.fit(x_train, y_train, epochs=5, validation_split=0.1)
```

Typical output after 5 epochs:
```
Epoch 5/5 - loss: 0.0821 - accuracy: 0.9751 - val_accuracy: 0.9741
```

### Step 5: Evaluate on Test Data

After training, evaluate performance on the held-out test set — data the model has never seen.

```python
test_loss, test_accuracy = model.evaluate(x_test, y_test)
print(f"Test accuracy: {test_accuracy:.4f}")
# Output: Test accuracy: 0.9762
```

Over 97% accuracy on handwritten digit classification — with fewer than 20 lines of code.

### Step 6: Make Predictions

```python
import numpy as np

predictions = model.predict(x_test[:5])
predicted_classes = np.argmax(predictions, axis=1)
print(predicted_classes)   # e.g., [7 2 1 0 4]
print(y_test[:5])          # e.g., [7 2 1 0 4]
```

`np.argmax` returns the index of the highest probability — the class the model is most confident about.

### Key Points

- MNIST is a standard benchmark dataset of 70,000 handwritten digit images
- Normalising inputs to 0–1 and using Dropout are simple but important practices
- `model.compile()` sets the optimiser, loss function, and metrics
- `model.fit()` runs the full training loop automatically
- `model.evaluate()` measures performance on unseen test data

What does `np.argmax(predictions, axis=1)` return?

## --answers--

The highest probability value for each prediction

### --feedback--

`np.argmax` returns the index of the highest value, not the value itself. The index corresponds to the predicted class.

---

The average probability across all classes

### --feedback--

`np.argmax` finds the position of the maximum, not an average. Averaging probabilities would not give a class prediction.

---

The index of the class with the highest predicted probability

---

The sum of all output probabilities for each example

### --feedback--

The sum of Softmax outputs is always 1 by definition. `np.argmax` returns the index of the single highest value.
