# --description--

CSP (Content Security Policy) and Permissions-Policy are two powerful HTTP security headers that help protect web applications by restricting what resources can load and what browser features a page can access.

Content Security Policy focuses primarily on resource loading. It acts as a safety filter for your frontend scripts, styles, and images.

Malicious actors often try to inject harmful scripts into vulnerable websites. A strong CSP stops these Cross-Site Scripting (XSS) attacks by establishing a strict whitelist of trusted source domains.

When a server sends the CSP header, the browser reads the specific directives. For instance, the `script-src` directive tells the browser which domains can safely run JavaScript on the page.

If a hacker attempts to inject a malicious script from an unapproved external server, the browser detects the policy violation, drops the script instantly, and logs a security error.

CSP can also restrict image sources, form destinations, and stylesheet origins.

Permissions-Policy focuses on browser features and device hardware. It replaced the older Feature-Policy header.

This security mechanism allows site owners to explicitly enable or disable specific browser features for their own page and any embedded iframes, and prevents third-party snippets from abusing user privacy.

Using a Permissions-Policy header, you can control access to hardware like the webcam, microphone, and geolocation tracking.

For example, if your website does not need to know a user location, you can set the policy to block the geolocation API entirely. Even if an attacker compromises a script on your page, the browser will refuse to activate the tracking feature.

The implementation of Permissions-Policy uses a clear, structured header format that names the feature and defines its allowed origins.

To restrict a hardware feature strictly to your own domain, you use the `self` keyword enclosed in parentheses, such as `camera=(self)`. And if you want to disable a feature across the entire page for every user, you use an empty list instead, which looks like `geolocation=()`.

Combining both headers provides a solid defense for web applications. CSP locks down data delivery and prevents code injection, and Permissions-Policy guards the physical device and ensures privacy compliance.

Modern web developers use both tools to create a heavily fortified browsing environment.
