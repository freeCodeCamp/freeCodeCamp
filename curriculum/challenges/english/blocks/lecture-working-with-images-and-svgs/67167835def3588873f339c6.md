# --interactive--

The convolutional layer is the core building block of a CNN. Understanding how it works — what a filter is, how it slides across an image, and what it detects — gives you the intuition for everything else in computer vision.

### What is a Filter?

A filter (also called a kernel) is a small grid of numbers, typically 3×3 or 5×5. It acts as a pattern detector. When a filter is applied to an image, it produces a new image (called a **feature map**) that highlights where that pattern appears.

A simple 3×3 edge detection filter might look like this:

```
-1  -1  -1
 0   0   0
 1   1   1
```

This filter responds strongly to horizontal edges — places where bright pixels above transition to dark pixels below.

### How Convolution Works

Convolution is the process of sliding the filter across the image and computing a dot product at each position:

1. Place the filter over a 3×3 region of the image
2. Multiply each filter value by the corresponding pixel value
3. Sum all the products — this gives one number in the feature map
4. Slide the filter one step to the right and repeat
5. After reaching the end of a row, move down one step and repeat from the left
6. Continue until the filter has covered the entire image

The result is a feature map — a new grid of numbers showing where and how strongly the filter's pattern was detected.

### A Concrete Example

```
Image patch:      Filter:        Products:
10  20  30       -1  -1  -1     -10  -20  -30
40  50  60    ×   0   0   0  =    0    0    0
70  80  90        1   1   1      70   80   90

Sum = -10 - 20 - 30 + 0 + 0 + 0 + 70 + 80 + 90 = 180
```

A large positive value (180) means the filter detected a strong horizontal edge at this location.

### Multiple Filters

A single convolutional layer does not apply just one filter — it applies many. Each filter learns to detect a different feature: one might detect horizontal edges, another vertical edges, another corners, another colour gradients.

The number of filters is a hyperparameter you choose. Common choices are 32, 64, or 128 filters per layer.

```python
import tensorflow as tf

model = tf.keras.Sequential([
    # 32 filters, each 3x3, applied to a 28x28 greyscale image
    tf.keras.layers.Conv2D(32, (3, 3), activation='relu', input_shape=(28, 28, 1)),
])
```

If an image is 28×28 and you apply 32 filters, the output has shape `(26, 26, 32)` — 32 feature maps, each slightly smaller than the input.

### Strides and Padding

**Stride** controls how many pixels the filter moves at each step. A stride of 1 moves one pixel at a time (default). A stride of 2 skips every other position, producing a smaller feature map and reducing computation.

**Padding** adds a border of zeros around the image before convolving. Using `padding='same'` keeps the output the same size as the input. Using `padding='valid'` (the default) allows the output to shrink.

```python
tf.keras.layers.Conv2D(32, (3, 3), activation='relu', padding='same')
```

### What Filters Learn

In the early layers of a CNN, filters learn simple patterns: edges, colours, and gradients. In deeper layers, filters combine those simple patterns into more complex detectors: textures, shapes, and eventually object parts. This hierarchical feature learning is what makes CNNs so powerful for vision tasks.

### Key Points

- A filter is a small grid of learnable numbers that detects a specific pattern
- Convolution slides the filter across the image, computing a dot product at each position
- The result is a feature map showing where the filter's pattern was detected
- A Conv2D layer learns many filters simultaneously, each detecting a different feature
- Stride controls step size; padding controls whether the output size matches the input
