# --interactive--

After a convolutional layer produces feature maps, the next step is usually a pooling layer. Pooling reduces the spatial size of the feature maps, making the network faster and more robust to small shifts in the input.

### Why Pooling?

Convolutional layers produce large feature maps. A 224×224 image passed through a Conv2D layer with 64 filters might produce 64 feature maps of size 222×222. This is a huge amount of data to process in the next layer.

Pooling solves three problems at once:

1. **Reduces computation** — smaller feature maps mean fewer calculations in subsequent layers
2. **Controls overfitting** — fewer values means fewer parameters to memorise
3. **Adds translation invariance** — if a feature shifts slightly in the image (e.g., a cat's eye moves a few pixels), pooling helps the network still detect it

### Max Pooling

Max pooling is the most common type. It divides the feature map into non-overlapping windows and keeps only the maximum value from each window.

Using a 2×2 window with stride 2:

```
Feature map:          After 2×2 max pooling:
1   3   2   4         3   4
5   6   1   2    →    6   3
3   2   1   0
1   2   3   4
```

The output is half the size in each spatial dimension. The maximum value represents "this feature was detected somewhere in this region" — it does not matter exactly where.

```python
import tensorflow as tf

model = tf.keras.Sequential([
    tf.keras.layers.Conv2D(32, (3, 3), activation='relu', input_shape=(28, 28, 1)),
    tf.keras.layers.MaxPooling2D(pool_size=(2, 2)),
])

# Input:  (28, 28, 1)
# After Conv2D:      (26, 26, 32)
# After MaxPooling:  (13, 13, 32)
```

### Average Pooling

Average pooling replaces the maximum with the average of all values in the window. It is used less frequently than max pooling in classification networks but appears in some architectures, particularly in the final layers before the output.

```
Feature map:          After 2×2 average pooling:
1   3   2   4         3.75  2.25
5   6   1   2    →    2.0   2.0
3   2   1   0
1   2   3   4
```

### Global Average Pooling

Global Average Pooling (GAP) is a special case where the pool window covers the entire feature map, producing a single number per feature map. It replaces the `Flatten` layer in modern CNN architectures and dramatically reduces the number of parameters.

```python
tf.keras.layers.GlobalAveragePooling2D()
# Input:  (7, 7, 512)  → Output: (512,)
```

### Translation Invariance

One of the most important properties that pooling provides is translation invariance. Consider detecting a cat's eye in an image. If the cat moves slightly, the eye's exact pixel position changes. Max pooling means the network responds to "the eye feature appeared somewhere in this 2×2 region" rather than "the eye feature appeared at exactly pixel (42, 67)". This makes the model more robust to small variations in position.

### Key Points

- Pooling reduces the spatial size of feature maps, reducing computation and overfitting
- Max pooling keeps the largest value in each window, representing the strongest feature detection in that region
- A 2×2 max pool with stride 2 halves the height and width of the feature map
- Global Average Pooling reduces each entire feature map to a single number
- Pooling adds translation invariance — the network becomes less sensitive to exact feature positions

Max pooling reduces spatial dimensions — it does not increase them. Each 2×2 region is replaced by a single value.

`padding='same'` on a Conv2D layer would preserve size. A 2×2 max pool with stride 2 reduces each dimension by half.

### --feedback--

A 2×2 pool with stride 2 divides each dimension by 2. 26 ÷ 2 = 13, not 6.

Which value does max pooling select from each pooling window?

The average of all values in the window

### --feedback--

Taking the average is what average pooling does. Max pooling specifically selects the single largest value.

---

### --feedback--

Max pooling selects the largest value, not the one nearest zero. Selecting near-zero values would discard the strongest feature detections.

---

The largest value in the window

---

### --feedback--

Random selection would discard information unpredictably. Max pooling deterministically selects the highest value, which represents the strongest feature detection.

The ability to process images of different sizes without resizing

### --feedback--

Handling different image sizes is a different property. Translation invariance specifically refers to robustness to small positional shifts of features within an image.

The ability to detect features regardless of which colour channel they appear in

### --feedback--

Colour channel handling is a different concern. Translation invariance is about position — the same feature at slightly different locations should still be detected.

---

The network becoming less sensitive to small positional shifts of features in the image

The ability to classify images that have been rotated or flipped

### --feedback--

Rotation and flip invariance require data augmentation or specific architectural choices. Max pooling provides invariance to small translations (shifts), not rotations.
