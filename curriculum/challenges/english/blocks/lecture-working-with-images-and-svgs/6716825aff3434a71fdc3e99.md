# --interactive--

Individual convolutional layers and pooling layers become powerful when combined into a complete architecture. This lesson shows how these pieces fit together and how data flows from raw pixels to a final class prediction.

### The Overall Pattern

A typical CNN follows this repeating pattern:

```
[Conv2D → ReLU → MaxPooling] × N  →  Flatten  →  Dense  →  Output
```

The convolutional blocks handle feature extraction. The dense layers at the end perform classification using the extracted features.

### Feature Extraction: The Convolutional Base

The first part of a CNN is the convolutional base — a series of Conv2D and pooling layers that progressively transform the raw image into abstract feature representations.

As you go deeper into the network:

- **Spatial dimensions decrease** — pooling shrinks the width and height
- **Number of feature maps increases** — more filters detect more complex patterns
- **Features become more abstract** — early layers detect edges; later layers detect shapes and object parts

A common pattern doubles the number of filters after each pooling step: 32 → 64 → 128.

### Classification Head: Dense Layers

After the convolutional base, the 3D feature maps are flattened into a 1D vector and passed through one or more Dense layers. These layers combine the detected features to make the final prediction.

```python
import tensorflow as tf

model = tf.keras.Sequential([
    # Convolutional base
    tf.keras.layers.Conv2D(32, (3, 3), activation='relu', input_shape=(32, 32, 3)),
    tf.keras.layers.MaxPooling2D((2, 2)),

    tf.keras.layers.Conv2D(64, (3, 3), activation='relu'),
    tf.keras.layers.MaxPooling2D((2, 2)),

    tf.keras.layers.Conv2D(128, (3, 3), activation='relu'),

    # Classification head
    tf.keras.layers.Flatten(),
    tf.keras.layers.Dense(128, activation='relu'),
    tf.keras.layers.Dropout(0.5),
    tf.keras.layers.Dense(10, activation='softmax')  # 10 classes
])

model.summary()
```

### Reading a Model Summary

`model.summary()` prints the shape of the output at every layer. This is invaluable for debugging:

```
Layer                    Output Shape         Param #
conv2d (Conv2D)          (None, 30, 30, 32)   896
max_pooling2d            (None, 15, 15, 32)   0
conv2d_1 (Conv2D)        (None, 13, 13, 64)   18,496
max_pooling2d_1          (None, 6, 6, 64)     0
conv2d_2 (Conv2D)        (None, 4, 4, 128)    73,856
flatten                  (None, 2048)         0
dense (Dense)            (None, 128)          262,272
dropout                  (None, 128)          0
dense_1 (Dense)          (None, 10)           1,290
```

Notice that the Dense layers at the end contain the most parameters. The convolutional layers are surprisingly efficient because their weights are shared across the entire image.

### Why Convolutional Layers are Parameter-Efficient

A 3×3 Conv2D layer with 32 filters has only `3 × 3 × input_channels × 32` weights — the same filter is applied at every position in the image. This weight sharing is what makes CNNs so much more efficient than Dense layers for images.

Compare:
- A Dense layer connecting a 32×32×3 image to 128 neurons: `3,072 × 128 = 393,216` weights
- A Conv2D layer with 32 filters of size 3×3 on the same image: `3 × 3 × 3 × 32 = 864` weights

### Key Points

- A CNN has two main parts: a convolutional base (feature extraction) and a classification head (Dense layers)
- Spatial dimensions decrease through the network; the number of feature maps typically increases
- Early layers detect simple features (edges); deeper layers detect complex features (shapes, objects)
- Weight sharing in convolutional layers makes them far more parameter-efficient than Dense layers
- `model.summary()` shows output shapes and parameter counts at every layer

Why are convolutional layers more parameter-efficient than Dense layers for processing images?

## --answers--

They use fewer activation functions, which reduces the total parameter count

### --feedback--

Activation functions do not contribute parameters — they are fixed mathematical operations. The efficiency of Conv layers comes from a different property.

---

They only process the centre of the image, ignoring the borders

### --feedback--

Convolutional filters process the entire image (with or without padding). The efficiency comes from how the weights are shared, not from ignoring any part of the image.

---

The same filter weights are shared and reused at every position in the image

---

They skip every other pixel to reduce computation

### --feedback--

Skipping pixels describes strided convolution and is about reducing computation, not the fundamental reason for parameter efficiency. Weight sharing is the key reason.
