# --interactive--

The GRU (Gated Recurrent Unit), introduced in 2014 by Kyunghyun Cho and colleagues, is a streamlined alternative to the LSTM. It achieves similar performance on many tasks while using fewer parameters and training faster.

### How the GRU Simplifies the LSTM

The LSTM has three gates and two states (hidden state + cell state). The GRU makes two simplifications:

1. **Merges the cell state and hidden state into a single state** — the GRU has only one state `h_t`
2. **Uses only two gates** instead of three — the reset gate and the update gate

This reduces the number of weight matrices from four (in an LSTM) to three, making GRUs about 25% faster to train for the same hidden size.

### The Two Gates

**Update gate** — decides how much of the previous hidden state to keep versus how much to replace with new information:

```
z_t = sigmoid(W_z * [h_{t-1}, x_t] + b_z)
```

When `z_t` is close to 1, the hidden state is carried forward mostly unchanged — similar to the LSTM's forget gate letting memory persist. When close to 0, the new input dominates.

**Reset gate** — decides how much of the previous hidden state to use when computing the candidate new state:

```
r_t = sigmoid(W_r * [h_{t-1}, x_t] + b_r)
h̃_t = tanh(W_h * [r_t ⊙ h_{t-1}, x_t] + b_h)
```

When the reset gate is close to 0, the candidate state is computed almost entirely from the current input, allowing the network to drop irrelevant history.

**Final update:**
```
h_t = (1 - z_t) ⊙ h̃_t + z_t ⊙ h_{t-1}
```

### GRU in TensorFlow

Swapping an LSTM for a GRU requires changing only one word:

```python
import tensorflow as tf

# LSTM version
lstm_model = tf.keras.Sequential([
    tf.keras.layers.Embedding(10000, 128),
    tf.keras.layers.LSTM(256),
    tf.keras.layers.Dense(1, activation='sigmoid')
])

# GRU version — almost identical
gru_model = tf.keras.Sequential([
    tf.keras.layers.Embedding(10000, 128),
    tf.keras.layers.GRU(256),
    tf.keras.layers.Dense(1, activation='sigmoid')
])
```

### LSTM vs GRU: When to Use Which

| | LSTM | GRU |
|---|---|---|
| Parameters | More (~4× SimpleRNN) | Fewer (~3× SimpleRNN) |
| Training speed | Slower | Faster |
| Performance on long sequences | Slightly better | Comparable |
| Performance on shorter sequences | Comparable | Comparable |
| Best for | Tasks with very long dependencies | Most practical applications |

In practice, the difference in performance between GRU and LSTM is often small. The GRU is a good first choice because it trains faster and requires less tuning. If performance is insufficient, switch to an LSTM.

### Bidirectional RNNs

Both GRUs and LSTMs can be made **bidirectional** — processing the sequence in both forward and backward directions and concatenating the results. This is useful when future context matters as much as past context (e.g., understanding a word in the middle of a sentence benefits from seeing both what came before and after it).

```python
tf.keras.layers.Bidirectional(tf.keras.layers.GRU(128))
# Output size: 256 (128 forward + 128 backward)
```

### Key Points

- The GRU simplifies the LSTM by merging the cell state and hidden state into one, and using two gates instead of three
- The update gate controls how much of the previous state to preserve
- The reset gate controls how much history to use when computing new candidate values
- GRUs train faster than LSTMs and perform comparably on most tasks
- Bidirectional wrappers process sequences in both directions, doubling the output size

What does the update gate in a GRU control?

## --answers--

Whether to apply dropout to the hidden state

### --feedback--

Dropout is a separate regularization technique applied outside the GRU cell. The update gate is a learnable part of the GRU's internal computation.

---

How much of the previous hidden state to keep versus how much to replace with new information

---

How much of the current input to send to the output layer

### --feedback--

The update gate operates between the previous hidden state and the new candidate state. Sending information to the output layer is controlled by the final hidden state `h_t`.

---

The learning rate for the GRU layer during training

### --feedback--

Learning rate is a training hyperparameter set during compilation. The update gate is a learned value computed dynamically from the input and previous state.
