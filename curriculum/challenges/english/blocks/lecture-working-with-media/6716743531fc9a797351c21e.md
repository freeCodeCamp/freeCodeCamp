# --interactive--

In this lesson you will build and train a complete CNN on CIFAR-10 — a dataset of 60,000 colour images across 10 object categories. This is a more realistic task than MNIST and demonstrates how CNNs handle real-world visual data.

### The CIFAR-10 Dataset

CIFAR-10 contains 60,000 colour images (32×32 pixels, 3 channels) across 10 classes:

```
0: airplane    1: automobile    2: bird       3: cat     4: deer
5: dog         6: frog          7: horse      8: ship    9: truck
```

```python
import tensorflow as tf
import numpy as np

(x_train, y_train), (x_test, y_test) = tf.keras.datasets.cifar10.load_data()

print(x_train.shape)  # (50000, 32, 32, 3)
print(x_test.shape)   # (10000, 32, 32, 3)
```

### Step 1: Preprocess

```python
# Normalise pixel values to 0-1
x_train = x_train.astype('float32') / 255.0
x_test  = x_test.astype('float32')  / 255.0

# Labels are shape (50000, 1) — squeeze to (50000,)
y_train = y_train.squeeze()
y_test  = y_test.squeeze()
```

### Step 2: Build the CNN

```python
model = tf.keras.Sequential([
    # Block 1
    tf.keras.layers.Conv2D(32, (3, 3), activation='relu',
                           padding='same', input_shape=(32, 32, 3)),
    tf.keras.layers.Conv2D(32, (3, 3), activation='relu', padding='same'),
    tf.keras.layers.MaxPooling2D((2, 2)),
    tf.keras.layers.Dropout(0.25),

    # Block 2
    tf.keras.layers.Conv2D(64, (3, 3), activation='relu', padding='same'),
    tf.keras.layers.Conv2D(64, (3, 3), activation='relu', padding='same'),
    tf.keras.layers.MaxPooling2D((2, 2)),
    tf.keras.layers.Dropout(0.25),

    # Classification head
    tf.keras.layers.Flatten(),
    tf.keras.layers.Dense(512, activation='relu'),
    tf.keras.layers.Dropout(0.5),
    tf.keras.layers.Dense(10, activation='softmax')
])
```

Using two Conv2D layers before each pooling step allows the network to build more complex features before reducing the spatial resolution.

### Step 3: Compile

```python
model.compile(
    optimizer='adam',
    loss='sparse_categorical_crossentropy',
    metrics=['accuracy']
)
```

### Step 4: Train with Early Stopping

```python
early_stop = tf.keras.callbacks.EarlyStopping(
    monitor='val_loss',
    patience=5,
    restore_best_weights=True
)

history = model.fit(
    x_train, y_train,
    epochs=50,
    batch_size=64,
    validation_split=0.1,
    callbacks=[early_stop]
)
```

`restore_best_weights=True` ensures the model reverts to the weights from the epoch with the best validation loss, not the last epoch.

### Step 5: Evaluate

```python
test_loss, test_accuracy = model.evaluate(x_test, y_test)
print(f"Test accuracy: {test_accuracy:.4f}")
# Typically around 75-80% with this architecture
```

75–80% accuracy on CIFAR-10 is a strong result for a network trained from scratch. State-of-the-art models exceed 99%, but they use much deeper architectures and more data.

### Inspecting Predictions

```python
class_names = ['airplane','automobile','bird','cat','deer',
               'dog','frog','horse','ship','truck']

predictions = model.predict(x_test[:5])
for i, pred in enumerate(predictions):
    predicted = class_names[np.argmax(pred)]
    actual    = class_names[y_test[i]]
    confidence = np.max(pred) * 100
    print(f"Predicted: {predicted} ({confidence:.1f}%)  Actual: {actual}")
```

### Key Points

- CIFAR-10 is a standard 10-class colour image classification benchmark
- Two consecutive Conv2D layers before pooling builds richer features before downsampling
- `EarlyStopping` with `restore_best_weights=True` prevents overfitting and saves the best model automatically
- A CNN trained from scratch on CIFAR-10 typically reaches 75–80% accuracy
- State-of-the-art performance requires much deeper architectures or transfer learning

`restore_best_weights` does not restart training. It operates on the model weights at the end of training.

Saving to disk after each epoch is the job of the `ModelCheckpoint` callback. `restore_best_weights` keeps the best weights in memory during training.

### --feedback--

Weight updates continue throughout training. `restore_best_weights` only acts at the end — it restores the best checkpoint rather than keeping the final (possibly worse) weights.
