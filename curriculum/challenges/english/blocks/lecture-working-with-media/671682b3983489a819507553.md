# --interactive--

One of the most effective ways to improve a CNN's generalisation is to give it more data. Data augmentation artificially increases the diversity of the training set by applying random transformations to existing images — without collecting a single new photo.

### The Core Idea

Consider training a cat classifier. Your dataset has cats photographed from the front. In the real world, cats appear from the side, slightly blurred, in different lighting, or partially cropped. A model trained only on front-facing cat photos may fail on these variations.

Data augmentation randomly applies realistic transformations during training so the model sees a different version of each image every epoch. The label stays the same — a flipped or slightly rotated cat is still a cat.

### Common Augmentation Techniques

**Horizontal flip** — mirrors the image left to right. Useful for most natural images (faces, animals, vehicles). Not useful for tasks where direction matters (reading text, detecting left vs right turns).

**Rotation** — rotates the image by a random angle. A small rotation (±15°) is usually safe. Large rotations may make the image unrecognisable.

**Zoom** — zooms in or out randomly, simulating different distances from the subject.

**Width and height shift** — shifts the image horizontally or vertically by a fraction of its size, simulating off-centre framing.

**Brightness and contrast adjustment** — simulates different lighting conditions.

**Cutout / Random erasing** — randomly masks a rectangular region of the image, forcing the model to rely on multiple parts of the image rather than one obvious feature.

### Augmentation in TensorFlow

The cleanest way to add augmentation in modern TensorFlow is with a `Sequential` preprocessing block added to the start of the model:

```python
import tensorflow as tf

data_augmentation = tf.keras.Sequential([
    tf.keras.layers.RandomFlip('horizontal'),
    tf.keras.layers.RandomRotation(0.1),      # ±10% of 360°
    tf.keras.layers.RandomZoom(0.1),
    tf.keras.layers.RandomTranslation(0.1, 0.1),
])

model = tf.keras.Sequential([
    data_augmentation,
    tf.keras.layers.Conv2D(32, (3, 3), activation='relu',
                           padding='same', input_shape=(32, 32, 3)),
    tf.keras.layers.MaxPooling2D((2, 2)),
    tf.keras.layers.Conv2D(64, (3, 3), activation='relu', padding='same'),
    tf.keras.layers.MaxPooling2D((2, 2)),
    tf.keras.layers.Flatten(),
    tf.keras.layers.Dense(128, activation='relu'),
    tf.keras.layers.Dropout(0.5),
    tf.keras.layers.Dense(10, activation='softmax')
])
```

When augmentation layers are part of the model, they are **automatically disabled during inference** — `model.predict()` and `model.evaluate()` always receive the original, unmodified images.

### How Much Augmentation?

More is not always better. Overly aggressive augmentation can destroy the meaningful content of an image. A few guidelines:

- Start with horizontal flips and small rotations — these are almost always safe
- Avoid transformations that change the label (e.g., rotating a "6" by 180° makes it look like a "9")
- Monitor validation accuracy — if it stops improving with augmentation, the transformations may be too aggressive

### Key Points

- Data augmentation applies random realistic transformations to training images to increase dataset diversity
- It helps models generalise by exposing them to variations they might encounter in the real world
- Augmentation layers in TensorFlow are automatically disabled during evaluation and inference
- Common transforms include flipping, rotation, zoom, and translation
- The label does not change — a transformed cat image is still labelled as a cat

### --feedback--

Augmentation does not remove examples. It adds variety by transforming existing ones, which makes memorisation harder.

### --feedback--

Augmentation does not change model architecture or parameter count. It changes the effective diversity of the training data.

Augmentation does not affect the learning rate. It works by increasing the effective diversity of training examples so the model cannot simply memorise them.
