# --interactive--

In this lesson you will build a complete GAN from scratch to generate handwritten digits using the MNIST dataset. Every component from the previous lessons comes together here.

### Step 1: Load and Preprocess the Data

GANs work best when the generator's output range matches the data range. We normalise pixel values to [-1, 1] to match the tanh activation in the generator's output layer.

```python
import tensorflow as tf
import numpy as np

(x_train, _), (_, _) = tf.keras.datasets.mnist.load_data()

# Normalise to [-1, 1] and add channel dimension
x_train = (x_train.astype('float32') - 127.5) / 127.5
x_train = x_train[..., tf.newaxis]  # shape: (60000, 28, 28, 1)

BUFFER_SIZE = 60000
BATCH_SIZE  = 256
LATENT_DIM  = 100

dataset = tf.data.Dataset.from_tensor_slices(x_train)
dataset = dataset.shuffle(BUFFER_SIZE).batch(BATCH_SIZE)
```

### Step 2: Build the Generator

```python
def build_generator():
    return tf.keras.Sequential([
        tf.keras.layers.Dense(7 * 7 * 256, use_bias=False,
                               input_shape=(LATENT_DIM,)),
        tf.keras.layers.BatchNormalization(),
        tf.keras.layers.LeakyReLU(),
        tf.keras.layers.Reshape((7, 7, 256)),

        # Upsample to 14×14
        tf.keras.layers.Conv2DTranspose(128, (5, 5), strides=(1, 1),
                                         padding='same', use_bias=False),
        tf.keras.layers.BatchNormalization(),
        tf.keras.layers.LeakyReLU(),

        # Upsample to 28×28
        tf.keras.layers.Conv2DTranspose(64, (5, 5), strides=(2, 2),
                                         padding='same', use_bias=False),
        tf.keras.layers.BatchNormalization(),
        tf.keras.layers.LeakyReLU(),

        tf.keras.layers.Conv2DTranspose(1, (5, 5), strides=(2, 2),
                                         padding='same', use_bias=False,
                                         activation='tanh')
        # Output: (28, 28, 1) — values in [-1, 1]
    ])
```

`Conv2DTranspose` (also called a transposed convolution or deconvolution) is the opposite of a regular convolution — it upsamples spatial dimensions rather than reducing them.

### Step 3: Build the Discriminator

```python
def build_discriminator():
    return tf.keras.Sequential([
        tf.keras.layers.Conv2D(64, (5, 5), strides=(2, 2), padding='same',
                                input_shape=(28, 28, 1)),
        tf.keras.layers.LeakyReLU(),
        tf.keras.layers.Dropout(0.3),

        tf.keras.layers.Conv2D(128, (5, 5), strides=(2, 2), padding='same'),
        tf.keras.layers.LeakyReLU(),
        tf.keras.layers.Dropout(0.3),

        tf.keras.layers.Flatten(),
        tf.keras.layers.Dense(1, activation='sigmoid')
    ])
```

### Step 4: Define Losses and Optimisers

```python
cross_entropy = tf.keras.losses.BinaryCrossentropy()

def discriminator_loss(real_output, fake_output):
    real_loss = cross_entropy(tf.ones_like(real_output)  * 0.9, real_output)
    fake_loss = cross_entropy(tf.zeros_like(fake_output) + 0.1, fake_output)
    return real_loss + fake_loss

def generator_loss(fake_output):
    return cross_entropy(tf.ones_like(fake_output), fake_output)

g_optimizer = tf.keras.optimizers.Adam(learning_rate=2e-4, beta_1=0.5)
d_optimizer = tf.keras.optimizers.Adam(learning_rate=1e-4, beta_1=0.5)

generator     = build_generator()
discriminator = build_discriminator()
```

### Step 5: Training Loop

```python
EPOCHS    = 50
seed_noise = tf.random.normal([16, LATENT_DIM])  # fixed noise for monitoring

@tf.function
def train_step(images):
    noise = tf.random.normal([BATCH_SIZE, LATENT_DIM])

    with tf.GradientTape() as g_tape, tf.GradientTape() as d_tape:
        fake_images  = generator(noise, training=True)
        real_output  = discriminator(images,      training=True)
        fake_output  = discriminator(fake_images, training=True)

        g_loss = generator_loss(fake_output)
        d_loss = discriminator_loss(real_output, fake_output)

    g_grads = g_tape.gradient(g_loss, generator.trainable_variables)
    d_grads = d_tape.gradient(d_loss, discriminator.trainable_variables)

    g_optimizer.apply_gradients(zip(g_grads, generator.trainable_variables))
    d_optimizer.apply_gradients(zip(d_grads, discriminator.trainable_variables))
    return g_loss, d_loss

for epoch in range(EPOCHS):
    for image_batch in dataset:
        g_loss, d_loss = train_step(image_batch)

    if (epoch + 1) % 10 == 0:
        # Generate sample images using the fixed seed noise
        samples = generator(seed_noise, training=False)
        print(f"Epoch {epoch+1}: G loss={g_loss:.4f}, D loss={d_loss:.4f}")
```

### Step 6: Generating New Images

```python
# After training: generate new digit images
noise       = tf.random.normal([25, LATENT_DIM])
generated   = generator(noise, training=False)

# Rescale from [-1, 1] back to [0, 255]
generated_images = (generated * 127.5 + 127.5).numpy().astype('uint8')
```

### Key Points

- MNIST pixels are normalised to [-1, 1] to match the generator's tanh output
- `Conv2DTranspose` upsamples spatial dimensions in the generator
- Label smoothing (0.9 and 0.1 instead of 1 and 0) stabilises discriminator training
- A fixed seed noise vector lets you track visual progress across epochs
- After training, new images are generated by passing fresh random noise through the generator

Why are MNIST pixel values normalised to the range [-1, 1] for GAN training, rather than [0, 1]?

### --feedback--

Negative input values are not a discriminator requirement. The normalisation range is chosen to match a specific layer's output range.

---

Because values in [-1, 1] train faster than values in [0, 1] for all neural networks

### --feedback--

The [-1, 1] range does not universally train faster. The specific reason here is about matching the generator's output range so that real and fake images occupy the same numerical range.

---

Conv2DTranspose layers work with any numerical range. The normalisation choice is driven by the generator's output activation, not the transpose convolution layers.

## --text--

What does `Conv2DTranspose` do in the generator?

## --answers--

It applies a convolution to reduce the spatial dimensions of the feature maps

### --feedback--

Reducing spatial dimensions describes a regular Conv2D with strides or a pooling layer. The generator needs to increase spatial size to go from a small dense representation to a full image.

---

It transposes the weight matrix of a standard Conv2D layer to halve the parameter count

### --feedback--

`Conv2DTranspose` does not simply transpose the weight matrix. It performs a learnable upsampling operation that increases spatial dimensions.

---

It upsamples the spatial dimensions, increasing the width and height of feature maps

---

It converts feature maps from 3D to 2D by removing the channel dimension

### --feedback--

Removing the channel dimension is done by Flatten or Reshape layers. Conv2DTranspose maintains the spatial structure while increasing its resolution.

Why is a fixed seed noise vector saved before training begins?

## --answers--

To ensure the generator always produces the same output on every forward pass

### --feedback--

Using a fixed seed for all generation would eliminate diversity. The fixed seed is used only for a specific monitoring purpose during training.

---

To provide a consistent reference point so you can visually track how the generator improves across epochs

---

To prevent the discriminator from memorising the generator's outputs

### --feedback--

Memorisation prevention is handled by training dynamics and regularisation. The fixed seed serves a different purpose related to monitoring progress.

---

Because TensorFlow requires a fixed random seed to enable GPU acceleration

### --feedback--

GPU acceleration works regardless of whether a fixed seed is used. The fixed seed is a monitoring tool, not a technical requirement.
