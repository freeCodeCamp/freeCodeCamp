# --interactive--

The original GAN used fully connected layers. DCGAN (Deep Convolutional GAN), introduced in 2015 by Radford, Metz, and Chintala, replaced fully connected layers with convolutional layers throughout, and established a set of architectural guidelines that became the standard for image-generating GANs.

### Why Convolutions?

Fully connected generators and discriminators treat images as flat vectors, ignoring spatial structure. Convolutional layers are designed specifically for spatial data — they learn local patterns, preserve spatial relationships, and share weights across positions. This makes them far better suited to generating and evaluating images.

The DCGAN paper also identified specific architectural choices that dramatically stabilise GAN training.

### DCGAN Architecture Guidelines

1. **Replace pooling layers with strided convolutions** — let the network learn its own downsampling rather than using fixed max pooling
2. **Use batch normalisation in both generator and discriminator** — except in the generator's output layer and the discriminator's input layer
3. **Remove fully connected hidden layers** — use global average pooling or go directly from convolutional features to output
4. **Use ReLU in the generator** (except the output layer, which uses tanh)
5. **Use Leaky ReLU in the discriminator**

### DCGAN Generator for 64×64 Images

```python
import tensorflow as tf

def build_dcgan_generator(latent_dim=100):
    return tf.keras.Sequential([
        # Project and reshape: latent → 4×4×1024
        tf.keras.layers.Dense(4 * 4 * 1024, use_bias=False,
                               input_shape=(latent_dim,)),
        tf.keras.layers.BatchNormalization(),
        tf.keras.layers.ReLU(),
        tf.keras.layers.Reshape((4, 4, 1024)),

        # 4×4 → 8×8
        tf.keras.layers.Conv2DTranspose(512, (5, 5), strides=(2, 2),
                                         padding='same', use_bias=False),
        tf.keras.layers.BatchNormalization(),
        tf.keras.layers.ReLU(),

        # 8×8 → 16×16
        tf.keras.layers.Conv2DTranspose(256, (5, 5), strides=(2, 2),
                                         padding='same', use_bias=False),
        tf.keras.layers.BatchNormalization(),
        tf.keras.layers.ReLU(),

        # 16×16 → 32×32
        tf.keras.layers.Conv2DTranspose(128, (5, 5), strides=(2, 2),
                                         padding='same', use_bias=False),
        tf.keras.layers.BatchNormalization(),
        tf.keras.layers.ReLU(),

        # 32×32 → 64×64 — output layer: no BN, tanh activation
        tf.keras.layers.Conv2DTranspose(3, (5, 5), strides=(2, 2),
                                         padding='same', use_bias=False,
                                         activation='tanh'),
    ])
```

Note the pattern: each `Conv2DTranspose` with `strides=(2,2)` doubles the spatial dimensions. The number of filters halves as the resolution increases — this is the standard DCGAN structure.

### DCGAN Discriminator for 64×64 Images

```python
def build_dcgan_discriminator():
    return tf.keras.Sequential([
        # Input: 64×64×3 — no batch norm on first layer
        tf.keras.layers.Conv2D(128, (5, 5), strides=(2, 2), padding='same',
                                input_shape=(64, 64, 3)),
        tf.keras.layers.LeakyReLU(0.2),

        # 32×32
        tf.keras.layers.Conv2D(256, (5, 5), strides=(2, 2), padding='same'),
        tf.keras.layers.BatchNormalization(),
        tf.keras.layers.LeakyReLU(0.2),

        # 16×16
        tf.keras.layers.Conv2D(512, (5, 5), strides=(2, 2), padding='same'),
        tf.keras.layers.BatchNormalization(),
        tf.keras.layers.LeakyReLU(0.2),

        # 8×8
        tf.keras.layers.Conv2D(1024, (5, 5), strides=(2, 2), padding='same'),
        tf.keras.layers.BatchNormalization(),
        tf.keras.layers.LeakyReLU(0.2),

        tf.keras.layers.Flatten(),
        tf.keras.layers.Dense(1, activation='sigmoid')
    ])
```

The discriminator mirrors the generator: each strided convolution halves the spatial dimensions, while the number of filters doubles.

### Visualising the Latent Space

One of the most striking properties of a trained DCGAN is the smooth interpolation between images in latent space:

```python
def interpolate(z1, z2, steps=10):
    """Generate images along a straight line between two latent vectors"""
    alphas  = tf.linspace(0.0, 1.0, steps)
    vectors = [z1 * (1 - a) + z2 * a for a in alphas]
    images  = [generator(v[tf.newaxis], training=False) for v in vectors]
    return images
```

Interpolating between two noise vectors produces a smooth visual transition — the generated images morph gradually from one realistic output to another, passing through other realistic images along the way.

### Key Points

- DCGAN replaced fully connected layers with strided convolutions for image generation
- Key guidelines: strided convolutions instead of pooling, batch norm in hidden layers, ReLU in generator, Leaky ReLU in discriminator
- The generator uses `Conv2DTranspose` to progressively upsample from a small dense representation to a full image
- The discriminator mirrors the generator, using strided convolutions to progressively downsample
- A trained DCGAN produces smooth interpolations between images in latent space

What is the main advantage of DCGAN over the original fully connected GAN for image generation?

## --answers--

DCGAN trains faster because it uses fewer parameters overall

### --feedback--

DCGAN typically uses more parameters than a simple fully connected GAN on small images. The advantage is about the quality and nature of what is learned.

---

DCGAN uses convolutional layers that preserve spatial structure, producing sharper and more realistic images

---

DCGAN does not require a discriminator, making training more stable

### --feedback--

DCGAN still uses both a generator and a discriminator. Removing the discriminator would fundamentally change how a GAN works.

---

DCGAN can only generate greyscale images, which require less computation than colour images

### --feedback--

DCGAN works well for both greyscale and colour images. The architecture outputs colour images by setting the final layer to 3 channels.

In the DCGAN generator, what does each `Conv2DTranspose` layer with `strides=(2,2)` do to the spatial dimensions?

## --answers--

It halves the width and height of the feature maps

### --feedback--

Halving spatial dimensions is what strided convolutions do in the discriminator. In the generator, `Conv2DTranspose` with stride 2 does the opposite.

---

It keeps the spatial dimensions the same while increasing the number of channels

### --feedback--

`padding='same'` without strides would preserve dimensions. With `strides=(2,2)`, `Conv2DTranspose` changes the spatial size significantly.

---

It doubles the width and height of the feature maps

---

It removes every other row and column of the feature map

### --feedback--

Removing rows and columns is a downsampling operation. `Conv2DTranspose` is an upsampling operation — it inserts new values between existing ones.

According to DCGAN guidelines, where should batch normalisation NOT be applied?

## --answers--

In the middle layers of both the generator and discriminator

### --feedback--

Batch normalisation is recommended in the middle (hidden) layers of both networks. The guidelines specify exceptions at the boundaries.

---

In the generator's output layer and the discriminator's input layer

---

In any layer that uses Leaky ReLU activation

### --feedback--

Leaky ReLU and batch normalisation are commonly used together in the discriminator's hidden layers. The restriction is about specific boundary layers, not activation functions.

### --feedback--

Batch normalisation is recommended in the discriminator's hidden layers. Only the first layer (input) is excluded.
