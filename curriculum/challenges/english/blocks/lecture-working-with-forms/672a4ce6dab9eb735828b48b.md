# --interactive--

A standard GAN generates random samples with no control over what is produced. A Conditional GAN (cGAN) adds a conditioning signal — such as a class label — so you can control exactly what the generator produces.

### The Problem with Unconditional GANs

A GAN trained on MNIST will generate digit images, but you cannot specify which digit. The output is random. If you want to generate the digit "7", you have no mechanism to request it — you can only keep sampling until you happen to get a 7.

Conditional GANs solve this by feeding label information into both the generator and the discriminator, allowing explicit control over the output.

### How Conditioning Works

The condition (e.g., a class label) is provided to both networks:

- **Generator**: receives both the random noise vector and the condition label. It must learn to generate images that match the requested class.
- **Discriminator**: receives both the image and the condition label. It must learn to detect not just whether an image is real, but whether the image matches its label.

This second point is critical. If the discriminator only checked whether images were realistic (without checking the label), the generator could learn to produce realistic images of any digit regardless of the requested label. The discriminator must also verify that the image matches the condition.

### Implementing a Conditional GAN

The most common approach is to embed the label and concatenate it with the noise (for the generator) or the image features (for the discriminator):

```python
import tensorflow as tf

NUM_CLASSES = 10
LATENT_DIM  = 100
EMBED_DIM   = 50

def build_conditional_generator():
    # Noise input
    noise_input = tf.keras.Input(shape=(LATENT_DIM,))

    # Label input — embed the integer class label into a dense vector
    label_input    = tf.keras.Input(shape=(1,))
    label_embedded = tf.keras.layers.Embedding(NUM_CLASSES, EMBED_DIM)(label_input)
    label_flat     = tf.keras.layers.Flatten()(label_embedded)

    # Concatenate noise and label embedding
    combined = tf.keras.layers.Concatenate()([noise_input, label_flat])

    # Generator body
    x = tf.keras.layers.Dense(7 * 7 * 256, use_bias=False)(combined)
    x = tf.keras.layers.BatchNormalization()(x)
    x = tf.keras.layers.LeakyReLU()(x)
    x = tf.keras.layers.Reshape((7, 7, 256))(x)

    x = tf.keras.layers.Conv2DTranspose(128, (5,5), strides=(1,1),
                                          padding='same', use_bias=False)(x)
    x = tf.keras.layers.BatchNormalization()(x)
    x = tf.keras.layers.LeakyReLU()(x)

    x = tf.keras.layers.Conv2DTranspose(64, (5,5), strides=(2,2),
                                          padding='same', use_bias=False)(x)
    x = tf.keras.layers.BatchNormalization()(x)
    x = tf.keras.layers.LeakyReLU()(x)

    output = tf.keras.layers.Conv2DTranspose(1, (5,5), strides=(2,2),
                                               padding='same', use_bias=False,
                                               activation='tanh')(x)

    return tf.keras.Model([noise_input, label_input], output)
```

### Generating Specific Digits

After training, you can request any digit:

```python
def generate_digit(generator, digit, n_samples=5):
    noise  = tf.random.normal([n_samples, LATENT_DIM])
    labels = tf.fill([n_samples, 1], digit)   # all samples request same digit

    generated = generator([noise, labels], training=False)
    return generated

# Generate five examples of the digit "7"
sevens = generate_digit(generator, digit=7)
```

### Beyond Class Labels: Other Conditioning Signals

Class labels are just one type of conditioning signal. cGANs have been applied with many other conditions:

- **Text descriptions** — "a red sports car on a sunny road" → generate matching image (the basis of text-to-image models)
- **Another image** — translate a sketch into a photorealistic image, or a daytime scene into a nighttime scene (pix2pix)
- **Attributes** — control specific face attributes like age, glasses, hair colour
- **Segmentation maps** — generate photorealistic scenes from a semantic layout diagram

### Key Points

- A Conditional GAN adds a conditioning signal (such as a class label) to control what the generator produces
- Both the generator and discriminator receive the condition — the discriminator must verify that images match their labels
- Labels are embedded into dense vectors and concatenated with the noise input in the generator
- After training, specific outputs can be requested by providing the desired label alongside the noise
- Conditioning signals can be class labels, text, images, or any other structured input

The noise vector is still used — it provides the randomness that creates diversity within each class. The label is added alongside the noise, not instead of it.

Learning rate is a training hyperparameter set before training begins. The label is part of the model's input, not a training configuration.
