# --interactive--

GANs are notoriously difficult to train. Two failure modes appear repeatedly across different datasets and architectures: mode collapse and training instability. Understanding them — and how to address them — is essential for anyone building GANs.

### Mode Collapse

In statistics, a "mode" is a peak in a probability distribution. The real data distribution typically has many modes — for a digit dataset, there is a mode for each digit (0 through 9), each looking slightly different across many examples.

**Mode collapse** occurs when the generator learns to produce only a narrow set of outputs — sometimes just a single output — regardless of the input noise. It has discovered that producing one type of fake reliably fools the current discriminator, so it exploits this rather than learning the full variety of the real data.

Example with MNIST:
- Healthy generator: produces diverse digits 0–9 with natural variation
- Mode-collapsed generator: produces only the digit "1" in various orientations, ignoring all other digits

Mode collapse is problematic because:
1. The generated samples lack diversity, making them useless for data augmentation
2. The discriminator will eventually learn to reject this single mode, and the generator must then jump to a new mode — causing oscillation rather than convergence

### Detecting Mode Collapse

```python
# Generate a large batch and check for diversity
noise    = tf.random.normal([100, latent_dim])
fakes    = generator(noise, training=False)

# For image data: if all generated images look nearly identical,
# mode collapse has occurred
# For quantitative detection: compute the standard deviation
# across the batch — low std suggests collapse
std = tf.math.reduce_std(fakes)
print(f"Batch std: {std:.4f}")   # very low → likely mode collapse
```

### Training Instability

Even without mode collapse, GAN training can be unstable. The losses oscillate, the generated quality fluctuates wildly between epochs, or training diverges entirely.

Common causes:
- **Learning rate too high** — both networks overshoot optimal weights
- **Imbalanced capacity** — one network is much larger and more powerful than the other
- **Batch normalisation misuse** — applying batch norm incorrectly in the discriminator can destabilise training

### Solutions and Best Practices

**Use Leaky ReLU in the discriminator.** Standard ReLU kills gradients for negative inputs. Leaky ReLU (`f(x) = max(0.2x, x)`) allows small gradients to flow for negative values, keeping training more stable.

```python
tf.keras.layers.LeakyReLU(alpha=0.2)
```

**Use label smoothing.** Instead of hard labels (0 and 1), use soft labels (0.1 and 0.9). This prevents the discriminator from becoming overconfident.

```python
real_labels = tf.ones_like(real_output)  * 0.9   # soft real label
fake_labels = tf.zeros_like(fake_output) + 0.1   # soft fake label
```

**Use separate optimisers with different learning rates.** The generator often benefits from a slightly higher learning rate than the discriminator.

```python
g_optimizer = tf.keras.optimizers.Adam(learning_rate=0.0002, beta_1=0.5)
d_optimizer = tf.keras.optimizers.Adam(learning_rate=0.0001, beta_1=0.5)
```

**Use batch normalisation in the generator.** This stabilises the generator's outputs and makes training smoother.

```python
tf.keras.layers.BatchNormalization()
```

**Monitor with visual inspection.** Save sample outputs every N steps. Visual inspection is often the most reliable indicator of training health.

### The Wasserstein GAN (WGAN)

The Wasserstein GAN is an architecture variant specifically designed to address mode collapse and instability. It replaces the standard binary cross-entropy loss with the **Wasserstein distance** (also called Earth Mover's Distance), which provides smoother, more informative gradients throughout training.

WGANs use a **critic** instead of a discriminator — the critic outputs an unbounded score (not a probability) and its weights are clipped to a limited range after each update to satisfy the mathematical requirements of the Wasserstein distance.

### Key Points

- Mode collapse occurs when the generator produces only a narrow set of outputs instead of learning the full data distribution
- Training instability causes oscillating losses and inconsistent output quality
- Leaky ReLU, label smoothing, and batch normalisation are standard practices for more stable GAN training
- The Wasserstein GAN replaces cross-entropy loss with Wasserstein distance for smoother gradients
- Visual inspection of generated samples is the most reliable way to detect both mode collapse and instability

Why is Leaky ReLU preferred over standard ReLU in the GAN discriminator?

## --answers--

Because Leaky ReLU produces outputs in the range 0 to 1, which is required for the discriminator's binary output

### --feedback--

The 0-to-1 output for the discriminator comes from the final sigmoid activation, not Leaky ReLU. Leaky ReLU is used in the hidden layers for a different reason.

---

Because Leaky ReLU allows small gradients to flow for negative inputs, keeping training more stable

---

Because Leaky ReLU trains faster than standard ReLU on GPU hardware

### --feedback--

Speed is not the primary motivation for using Leaky ReLU in GANs. The reason is about gradient behaviour for negative input values.

---

Because standard ReLU causes the discriminator to output only 0 or 1

### --feedback--

Standard ReLU does not force binary outputs — it zeroes out negative activations in hidden layers. The issue with standard ReLU in GANs is about gradient flow, not the output range.

What makes the Wasserstein GAN different from a standard GAN?

## --answers--

It uses a convolutional generator and discriminator instead of fully connected layers

### --feedback--

Convolutional architectures are used in DCGANs, not specifically in WGANs. The WGAN's innovation is in the loss function and training procedure.

---

It trains the generator and discriminator simultaneously in the same gradient tape

### --feedback--

Standard GANs already have variations in how steps are coordinated. The key difference in WGANs is the mathematical objective used to measure the distance between real and generated distributions.

---

It replaces binary cross-entropy with the Wasserstein distance for smoother, more informative gradients

---

It removes the discriminator entirely and replaces it with a pre-trained classifier

### --feedback--

The WGAN keeps both networks but modifies the training objective. The network evaluating generated samples is renamed "critic" but is still structurally similar to a discriminator.
