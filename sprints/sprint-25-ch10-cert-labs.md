### 2. Lab — `lab-roguelike-dungeon`

≥18 user stories. Examples:

1. `MainScene` extends `Phaser.Scene` with the standard methods.
2. Movement is turn-based, one tile per arrow-key press.
3. Dungeon generation uses a seeded RNG (`Phaser.Math.RandomDataGenerator`).
4. Dungeon is generated by binary space partition with at least 5 rooms per floor.
5. All rooms are reachable (no isolated rooms).
6. Player has fog-of-war / FOV that reveals only nearby tiles.
7. Explored tiles render dimmed.
8. Enemies spawn in non-entry rooms.
9. Enemy AI uses a finite state machine (chase / wander).
10. Combat: bumping an enemy attacks; HP drops on hit.
11. Game over on player HP zero with a run-summary screen.
12. Player advances floors via an exit tile.
13. Each floor uses a fresh BSP layout.
14. At least 4 item rarities (common / uncommon / rare / epic).
15. Loot tables vary per floor depth.
16. A boss room with multi-stage HP appears every 5 floors.
17. Defeating a boss yields meta-currency persisted in `localStorage`.
18. Title screen offers meta-purchases that apply at run start.
19. Same seed reliably produces the same dungeon (determinism check).
20. Game accepts mobile touch input (tap-to-move).

Hints: regex assertions targeting the Sprint 22 reference solution. Solution: that reference solution.

### 3. Lab — `lab-multiplayer-party-game`

≥14 user stories. Examples:

1. `MainScene` extends `Phaser.Scene` with the standard methods.
2. Two player tanks spawn on opposite sides of an arena.
3. Player 1 input: WASD for movement, Q for fire.
4. Player 2 input: arrow keys for movement, slash for fire.
5. Tanks rotate with left/right and move with up.
6. Firing spawns a shell from the tank barrel.
7. A shell hitting an opponent reduces their HP.
8. HP bars render per tank.
9. A round ends when one tank reaches 0 HP.
10. Best-of-5 rounds decides match winner.
11. Round-break with a countdown between rounds.
12. At least one power-up spawns mid-round (e.g. shield, multi-shot).
13. Best win streak persists in `localStorage`.
14. Game accepts mobile touch with two virtual joysticks (one per player).

Hints + solution: Sprint 23's reference.
